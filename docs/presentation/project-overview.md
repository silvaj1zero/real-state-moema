## SLIDE 2 — O Problema

### Hoje: Esforço Desordenado

| O que a Luciana faz hoje | A dor |
|--------------------------|-------|
| Planilha Excel para controlar leads | Sem integração com mapa, sem funil visual |
| Captei para buscar proprietários | Dados não cruzam com território |
| Ligações manuais (5/dia) | Sem scripts acessíveis, sem tracking de conversão |
| WhatsApp para referrals | Sem rastreamento de reciprocidade |
| Memória para lembrar de zeladores | Sem registro de Marketing de Gentileza |
| PDF genérico como ACM | Sem dados reais da micro-região |

**Resultado:** O funil "morre por asfixia" — sem volume no topo, sem dados no meio, sem controle no fim.

> *"A alta performance imobiliária não é sobre sorte; é sobre a gestão de um negócio baseado em processos rigorosos, análise de dados e construção de autoridade."*

## SLIDE 3 — A Solução

### Um app que fala a linguagem RE/MAX

O sistema digitaliza a **Metodologia de Alta Performance** em 4 fases progressivas:

```
FASE 1                FASE 2              FASE 3              FASE 4
Mapear                Prospectar           Automatizar          Conectar
Território            com Método           Inteligência         a Rede

Mapa + Edifícios      Funil V1→V2→Venda   ACM + Dossiê PDF     Referrals
Raios Concêntricos    Informantes          Agentes de Busca     Clubes RE/MAX
Cadastro em Campo     Scripts              Feed Diário          Multi-Consultor
Offline               FROG + KPIs          Alertas FISBO        Comissões
```

Cada fase entrega **valor real e independente** — a Luciana já usa o app desde a Fase 1.

## SLIDE 4 — Como Funciona: O Conceito de Micro-Territórios

### Expansão como mancha de óleo

```
                    ┌─────────────────────┐
                    │     Raio 2km        │  Bloqueado até
                    │  ┌───────────────┐  │  dominar 1km
                    │  │   Raio 1km    │  │
                    │  │  ┌─────────┐  │  │  Desbloqueia quando
                    │  │  │  500m   │  │  │  80% do 500m mapeado
                    │  │  │  [★]    │  │  │
                    │  │  │ Epicentro│  │  │  ★ = Rua Alvorada
                    │  │  └─────────┘  │  │     Moema, SP
                    │  └───────────────┘  │
                    └─────────────────────┘
```

- **Epicentro:** Rua Alvorada, CEP 04550-000
- **Expansão:** 500m → 1km → 2km (Moema → Vila Olímpia → Itaim Bibi)
- **Regra:** Dominar 80% do raio atual antes de expandir
- **KPI:** Densidade de carteira (imóveis sob assessoria por km²)

> *"O consultor profissional não espera o cliente; ele mapeia o mercado e se posiciona como a solução técnica."*

## SLIDE 5 — Fase 1: Mapear o Território (Epic 1)

### O que a Luciana ganha no dia 1

| Funcionalidade | Benefício |
|---------------|-----------|
| Mapa com edifícios pré-carregados | Abre o app e já vê dezenas de prédios — foca em qualificar, não cadastrar |
| Cadastro rápido (<30 segundos) | Registra prédio enquanto caminha, com GPS automático |
| Cores por status no mapa | Sabe onde já passou (verde), onde falta ir (cinza) |
| Funciona offline | Cadastra em subsolos e garagens sem perder dados |
| % de cobertura do raio | Vê progresso: "Raio 500m: 72% mapeado" |

### Cenário: Tarde típica da Luciana

> Luciana sai do escritório da Galeria, abre o app no celular. Vê 47 edifícios no raio de 500m (pré-carregados). Caminha pela Al. dos Maracatins, toca no Ed. Maracatins 400 — edita: "Alto Padrão, 3+ dorms, zelador amigável". **15 segundos.** Em 1 tarde, qualifica 20 prédios. No fim do dia: "Raio 500m: 42% mapeado".

**Validação:** Teste de campo de 5 dias com a Luciana antes de avançar para Fase 2.

## SLIDE 6 — Fase 2: Prospectar com Método (Epic 2)

### A metodologia RE/MAX digitalizada

```
┌──────────────────────────────────────────────────────────┐
│                  FUNIL DE VENDAS RE/MAX                   │
│                                                          │
│  Contato → V1 (Rapport) → V2 (ACM) → Exclusividade → Venda │
│    📞         🤝            📊           ✍️          🎉    │
│                                                          │
│  Scripts    Informantes    Checklist      Marketing      │
│  de Objeção  + Gentileza   Preparação     Ativo          │
│                                                          │
│  FROG       Meta 5        Diagnóstico    Dashboard       │
│  Mapping    V1s/dia       de Gargalos    KPIs            │
└──────────────────────────────────────────────────────────┘
```

| Feature | O que resolve |
|---------|--------------|
| Funil visual (5 etapas) | Vê todos os leads em cada etapa, swipe para avançar |
| Cadastro de informantes | Zeladores/porteiros como "Oficiais de Inteligência" + regra dos 5% |
| Marketing de Gentileza | Registra cafés, brindes, agradecimentos — com lembretes |
| Scripts de objeção | "Não trabalho com imobiliárias" → resposta pronta no card do lead |
| Diagnóstico de gargalos | "Muitas V1s, poucas V2s → Falha de rapport" — com ação sugerida |
| Expansão 80% | Ao dominar o raio de 500m, desbloqueia o de 1km automaticamente |

### Cenário: Manhã de prospecção

> Luciana abre o app às 8h. Dashboard: 4 leads parados em "Contato" há 3 dias (alerta). Meta: 0/5 V1s agendadas. Consulta script "Fecho de Duas Opções", liga: *"Sr. João, prefere que eu passe amanhã às 10h ou quinta às 16h?"* João aceita quinta. Swipe: lead vai para "V1 Agendada". Notificação lembrará na quinta de manhã.

> Luciana gera ACM do Ed. Alvorada 200: 8 comparáveis — 3 vendas reais (R$ 11.200/m²), 5 anúncios ativos (R$ 12.800/m²). Gera Dossiê PDF com branding RE/MAX. Na V2, mostra ao proprietário: *"Veja — os anúncios pedem R$ 12.800, mas as vendas reais fecham a R$ 11.200. Eu tenho os dados que o Google não tem."* Proprietária impressionada. Fecha exclusividade.

## SLIDE 11 — Princípios de Design

### 3 regras que guiam todo o desenvolvimento

| Princípio | Significado | Exemplo |
|-----------|------------|---------|
| **Funciona sozinho** | Nenhuma feature depende de sistema externo para ter valor | ACM funciona com dados manuais. Referrals funcionam sem o parceiro ter conta |
| **Impossibilita o caminho errado** | O app não permite que você faça a coisa errada | Checklist impede V2 sem preparação. Mapa já vem populado no dia 1 |
| **Automação sugere, humano decide** | Especialmente com dinheiro e dados | Comissão: sistema sugere split, Luciana confirma. Nunca paga sozinho |
