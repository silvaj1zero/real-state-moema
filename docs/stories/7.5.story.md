# Story 7.5: Container CNPJ ETL (rictom/cnpj-sqlite) + Tabela cnpj_enrichment

**Epic:** 7 — Inteligencia de Prospeccao Automatizada Multi-Fonte
**Wave:** A (P0 enriquecimento)
**Status:** InReview
**Priority:** Must
**Estimated Complexity:** L (8 pts) — requer PoC per `story-prototyping`

## User Story

Como sistema de classificacao,
Quero ter base RFB CNPJ atualizada mensalmente em tabela `cnpj_enrichment` filtrada por CNAEs imobiliarios SP,
Para que `classifyAdvertiser` (Story 7.3) consiga identificar Builder (Construtora), Broker (Imobiliaria), Administradora e Holding via lookup CNAE rapido (<50ms).

## Technical Notes

- **Reference primaria:** ADR-EPIC7-005 + `docs/code-anatomy/rictom-cnpj-sqlite/extraction-notes.md`
- **Container runtime:** preferir GitHub Actions self-hosted runner (free tier suficiente p/ mensal) OR Hetzner CPX21 dedicado se Apify nao for adequado (Apify nao roda Python container nativamente)
- **share_token RFB:** rictom pipeline detecta via PROPFIND WebDAV; pode mudar; alerta no AC7
- **CNAE codes verbatim do oficial:** https://www.gov.br/receitafederal/dados/cnae.csv
  - 6822500 — Gestao e administracao da propriedade imobiliaria
  - 6831700 — Corretagem na compra e venda e avaliacao de imoveis
  - 6810201 — Compra e venda de imoveis proprios
  - 6810202 — Aluguel de imoveis proprios
  - 4110700 — Incorporacao de empreendimentos imobiliarios
  - 4120400 — Construcao de edificios
  - 4399105 — Outros servicos especializados para construcao
  - 8112500 — Condominios prediais
  - 8121400 — Limpeza em predios e em domicilios
  - 8130300 — Atividades paisagisticas
- **DASK vs pandas:** usar variante pandas (DASK tem comentario "database locked" do mantenedor)
- **CNPJ format:** RFB armazena cnpj_basico (8 digits) + cnpj_ordem (4 digits) + cnpj_dv (2 digits). Concatenar = 14 digits CNPJ completo. Validar via algoritmo CNPJ check digit antes de inserir.

### Decisions

1. **Upstream pin strategy:** Dockerfile carries `ARG RICTOM_SHA=4e1b9c5b...` (placeholder pinning convention). Actual rictom commit SHA materialized via `curl` at setup time (documented in README §Setup). Rationale: vendoring the MIT upstream into our git history would couple histories and inflate PR diffs. `scripts-upstream/*.py` are STUBS that fail loudly (exit 70) if invoked unmaterialized — protects against silent no-op runs.
2. **DASK discarded:** per `docs/code-anatomy/rictom-cnpj-sqlite/extraction-notes.md` — upstream comment notes "database locked" on DASK variant. pandas variant kept.
3. **CNAE list order:** verbatim from Story 7.5 AC2 (6822500 first, 8130300 last). Tests lock the order via `test_target_list_order_locked` — ADR-EPIC7-005 must be bumped before any reorder.
4. **Migration 011 RLS:** column-level GRANT used for `authenticated` (no socios/raw_data) instead of FGAC view. Reason: Story 7.10 (encryption) is the right layer for granular policies — keeping baseline simple here per Constitution Article V (Quality First > premature abstraction).
5. **`docker-compose` v3.9 + persistent named volume:** keeps the multi-GB RFB downloads outside the image, makes incremental runs (re-filter without re-download) trivial.
6. **GH Actions workflow disabled by default:** `if: false` guard at job level. Will be enabled by @devops after (a) upstream scripts materialized, (b) self-hosted runner provisioned, (c) secrets configured.
7. **DV validator added:** `_validate_cnpj_dv` is overkill for trusted RFB data but cheap insurance — 4 lines, tested with 7 edge cases. Documented in code header.
