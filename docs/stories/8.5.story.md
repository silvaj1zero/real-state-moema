# Story 8.5: Orquestração app ↔ engine `acm-imobiliario` (pipeline de 1 clique)

**Epic:** 8 — Geração de Entregáveis ACM
**Camada:** Integração
**Status:** Descoped (superseded by ADR-EPIC8-001)
**Priority:** Could
**Estimated Complexity:** L (8-13 pts)

> **PO Validation (2026-06-15, Pax):** Contingente (6/10).
> **ADR-EPIC8-001 resolvida (2026-06-15, Aria):** a ADR escolheu **render nativo** — o app **não chama o engine para renderizar**, então a orquestração app↔engine para geração de entregáveis **deixa de existir**. O que restaria (gatilho de atualização dos dados ITBI) **já é** a decisão de hospedagem do `epic7_itbi_monthly` (Epic 7 / handoff Sec. 4). **Esta story é descopada do Epic 8**; o gatilho de dados segue rastreado no Epic 7. Mantida no repo como registro da decisão.

## User Story

Como Luciana (consultora),
Quero, a partir de um lead/imóvel no app, disparar a geração completa do ACM (laudo + resumo + deck + didático) em um clique,
Para ir da pesquisa ao pacote de entrega sem passos manuais — a "fábrica de ACM" da Fase 3 do Roadmap.

## Contexto

Operacionaliza o **Passo 3.1 do Roadmap Metodológico** ("um comando `aiox acm <endereço>` que encadeia geocodificar → ITBI → filtrar/dedup → ranquear → calcular → gerar laudo + resumo + deck + material"). Só faz sentido se a ADR-EPIC8-001 (Story 8.3) escolher o caminho (A) — app orquestra o engine — ou um híbrido.

## Acceptance Criteria

- [ ] AC1: Endpoint/ação no app que recebe imóvel-alvo (endereço/coordenadas + `consultant_id`) e dispara o pipeline do engine `acm-imobiliario` para aquele alvo.
- [ ] AC2: Estado de progresso visível na UI (geocode → ITBI → ranking → cálculo → render), com tratamento de falha por etapa.
- [ ] AC3: Saídas (4 PDFs) retornadas/armazenadas e anexadas ao **dossiê** do imóvel (integra com "Incluir no Dossiê" existente).
- [ ] AC4: Atualização incremental — reusar cache de geocode/ITBI quando o alvo/raio já foi processado (Roadmap Passo 3.3).
- [ ] AC5: Hospedagem do disparo decidida (GitHub Action / job Python / edge function) — alinhar com a decisão pendente do `epic7_itbi_monthly` no `docs/HANDOFF-SESSION-20260615.md` Sec. 4.
- [ ] AC6: Revisão adversarial opcional (Roadmap Passo 3.4) antes de fechar o pacote — flag de "segundo avaliador".
