# Story 4.2: Tracking de Comissões

**Epic:** 4 — Parcerias Ganha/Ganha & Escala
**Status:** InProgress
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero registrar comissões por venda fechada com splits sugeridos e confirmação manual,
Para que eu tenha controle financeiro completo do meu negócio e rastreie pagamentos a informantes e parceiros de referral.

## Acceptance Criteria

- [x] AC1: Ao fechar uma venda (lead com `etapa_funil = 'venda'`), botão "Registrar Comissão" aparece no card do lead (azul #003DA5, ícone cifrão). Formulário com: valor do imóvel (input numérico R$, obrigatório — `comissoes.valor_imovel` NUMERIC(14,2)), percentual de comissão (input %, default 6% — `comissoes.percentual_comissao` NUMERIC(5,2)), cálculo automático do valor bruto (`valor_bruto = valor_imovel * percentual_comissao / 100`). INSERT em `comissoes` com `consultant_id = auth.uid()`, `lead_id` FK.
- [x] AC2: Sistema sugere splits automaticamente ao salvar: consultora (50% do bruto — `split_consultora`), franquia (50% do bruto — `split_franquia`). Se lead tem `informante_id`: adiciona split informante 5% do bruto (`split_informante`, `informante_id` FK). Se lead tem `referral_id`: adiciona split referral (`split_referral`, `referral_id` FK) com percentual configurável. Cálculos exibidos em cards visuais com valores em R$. Percentuais de consultora/franquia são configuráveis no formulário (default 50/50).
- [x] AC3: Cláusula de Relacionamento: se comprador é conhecido do proprietário (checkbox "Cláusula de Relacionamento" no form), campo adicional `percentual_clausula` (slider 3-4%, default from `consultant_settings.percentual_relacionamento_default`). Tipo split muda para `tipo_split = 'clausula_relacionamento'`. Valor deduzido do bruto antes dos demais splits. Tooltip explicativo: "Quando o comprador já tem relação com o proprietário, aplica-se desconto de 3-4% sobre a comissão".
- [x] AC4: Confirmação manual OBRIGATÓRIA antes de salvar splits. Tela de review com todos os valores calculados em tabela: beneficiário | % | R$ valor. Botão "Confirmar Splits" (verde #28A745) e "Editar" (outline). Somente após confirmação o INSERT final acontece. **NUNCA auto-pagar — princípio PV inegociável.** `status_pagamento` inicia como 'pendente'.
- [x] AC5: Status de pagamento rastreável por comissão: pendente → recebido → pago_informante → pago_parceiro → completo. Transição manual via chips no card da comissão. Ao marcar "recebido" (`data_recebimento = now()`), alerta visual (badge verde pulsante) no dashboard. Ao marcar "pago_informante" (`data_pagamento_informante = now()`) e "pago_parceiro" (`data_pagamento_referral = now()`), registra datas. Status "completo" só quando todos os pagamentos aplicáveis estão feitos.
- [x] AC6: Dashboard financeiro acessível via menu lateral → "Comissões" (ícone gráfico barras #003DA5). Cards resumo no topo: Total Bruto (soma `valor_bruto`), Total Líquido Consultora (soma `split_consultora`), Pendentes de Recebimento (count WHERE `status_pagamento = 'pendente'`), Recebidas no Mês (sum WHERE `data_recebimento` no mês atual). Gráfico de barras mensal (últimos 6 meses) com valor bruto por mês. Lista de comissões abaixo com filtros: status, período, parceiro/informante.
- [x] AC7: Exportação CSV e PDF do dashboard. Botão "Exportar" (ícone download) no topo do dashboard. CSV: todas as colunas da tabela `comissoes` + nome do lead + nome do parceiro/informante. PDF: relatório formatado com logo RE/MAX, período selecionado, totais, lista detalhada. Geração client-side (jsPDF + autoTable ou similar).
- [x] AC8: Integração com VGV para Clubes RE/MAX: ao registrar comissão com status 'recebido', UPDATE em `consultant_settings.vgv_acumulado += valor_bruto`. Esse valor alimenta a barra de progresso da Story 4.3. Reversão se comissão for cancelada/editada.
- [x] AC9: Ao salvar comissão, toast "Comissão R$ [valor] registrada!" com haptic. Se lead veio de referral, referral atualizado para status 'comissao_paga' automaticamente (após confirmar pagamento parceiro). Se lead veio de informante, informante ganha +1 no tracking de conversões (Story 2.3).

## Technical Notes

- **Database:** Tabela `comissoes` (migration 004). Colunas: id, consultant_id, lead_id, valor_imovel, percentual_comissao, valor_bruto, split_consultora, split_franquia, split_informante, split_referral, tipo_split (check: padrao/referral/informante/clausula_relacionamento), percentual_clausula (default 3.0), informante_id (FK), referral_id (FK), status_pagamento (check: pendente/recebido/pago_informante/pago_parceiro/completo), data_recebimento, data_pagamento_informante, data_pagamento_referral, notas, created_at, updated_at. Índice: `idx_comissoes_consultant`.
- **API:** Supabase client SDK. TanStack Query mutations com invalidação de `['comissoes']` e `['consultant_settings']` (VGV). Cálculo de splits no client (lógica pura, testável).
- **Components:** ComissaoForm (formulário com cálculo dinâmico), SplitReview (tabela confirmação), ComissaoDashboard (cards + gráfico), ComissaoList (filtros + paginação), ExportButton (CSV/PDF).
- **Gráficos:** Recharts ou Chart.js para barras mensais. Dados agregados via query SQL com `date_trunc('month', created_at)`.
- **PDF:** jsPDF + jspdf-autotable para geração client-side. Template com header RE/MAX.

## Notes

- **Princípio PV — NUNCA auto-pagar:** O sistema SUGERE splits, mas a consultora CONFIRMA manualmente cada valor. Automação calcula, humano decide. Isso é inegociável.
- **Princípio PV — Cláusula de Relacionamento:** Gap resolvido na auditoria PV. Quando o comprador já tem relação com o proprietário, aplica-se desconto de 3-4% configurável. Campo `percentual_relacionamento_default` em `consultant_settings` define o padrão.
- **Princípio PV — Split consultora/franquia:** O default 50/50 é o modelo RE/MAX padrão, mas varia por contrato. Manter editável.
- VGV acumulado deve ser calculado a partir de comissões com `status_pagamento IN ('recebido','pago_informante','pago_parceiro','completo')`, não de todas as comissões.
