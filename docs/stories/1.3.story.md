# Story 1.3: Cadastro Rápido de Edifícios em Campo

**Epic:** 1 — Foundation, Mapa & Registro de Campo
**Status:** Done
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero cadastrar um edifício em menos de 30 segundos enquanto caminho,
Para que eu possa mapear minha região de atuação eficientemente sem parar.

## Acceptance Criteria

- [x] AC1: FAB (Floating Action Button) no mapa conforme frontend-spec seção 2.2 — canto inferior direito, 16px acima da tab bar, 56px diâmetro, cor vermelho RE/MAX #DC1431, ícone "+" branco 24px, sombra elevation 8. Tap abre formulário de Cadastro Rápido.
- [x] AC2: GPS preenche coordenadas automaticamente via `navigator.geolocation.getCurrentPosition()`. Indicador de precisão GPS no header: verde = boa precisão (<20m), amarelo = baixa (20-100m), vermelho = sem sinal. Coordenadas salvas como `geography(Point, 4326)` na tabela `edificios`.
- [x] AC3: Endereço sugerido via reverse geocoding (Mapbox Geocoding API). Campo pré-preenchido com endereço mais provável. Botão "edit" para correção manual. Se GPS indisponível: campo vazio com placeholder "Digite o endereço".
- [x] AC4: Campo obrigatório: nome (auto-focus ao abrir, prefixo "Ed." como placeholder sugerido, font 16px, altura 48px). Campo endereço pré-preenchido (obrigatório, editável). Dados inseridos na tabela `edificios` com campos: nome, endereco, coordinates, bairro (extraído do geocoding), cep, cidade, estado, origem = 'manual', created_by = auth.uid().
- [x] AC5: Campos opcionais: tipologia (chips select: Residencial, Comercial, Misto — enum `tipologia_edificio`) e padrão (chips select: Baixo, Médio, Alto, Luxo — enum `padrao_edificio`). Chips com altura 40px, outlined quando inativo, filled azul quando selecionado. Se preenchidos, cria automaticamente registro em `edificios_qualificacoes` com consultant_id = auth.uid(), status_varredura = 'mapeado'.
- [x] AC6: Máximo 3 taps obrigatórios + teclado para nome: (1) tap FAB, (2) digita nome no teclado, (3) tap "Salvar". Tipologia e padrão são taps opcionais. Botão Salvar full-width 56px verde #22C55E, disabled até nome preenchido, loading spinner durante salvamento.
- [x] AC7: Pin aparece no mapa imediatamente após salvar com animação bounce + 3 piscadas em azul (status "Mapeado"). Toast "Ed. [Nome] cadastrado!" por 2s, dismissável. Haptic feedback (vibração curta) ao salvar com sucesso.
- [x] AC8: Schema separado: INSERT em `edificios` (dados base) + INSERT em `edificios_qualificacoes` (qualificação privada do consultor com status_varredura = 'mapeado' e campos opcionais tipologia/padrao). Constraint UNIQUE(edificio_id, consultant_id) na qualificacoes.
- [x] AC9: Tap "X" ou swipe down no header: confirma descarte se houver dado preenchido ("Descartar cadastro?" com Cancelar/Descartar). Se nenhum dado preenchido, fecha sem confirmação.

## Technical Notes

- **Database:** INSERT em `edificios` (nome, endereco, endereco_normalizado, coordinates geography, bairro, cep, cidade, estado, origem='manual', verificado=true, created_by) + INSERT condicional em `edificios_qualificacoes` (edificio_id, consultant_id, tipologia, padrao, status_varredura='mapeado'). RLS: edificios permite insert para authenticated, qualificacoes filtra por consultant_id.
- **API:** Supabase client SDK para inserts. Mapbox Geocoding API (reverse) para endereço a partir de coordenadas: `https://api.mapbox.com/geocoding/v5/mapbox.places/{lng},{lat}.json`. Proxy via Next.js API Route para proteger token Mapbox.
- **UI Components:** FAB (56px, vermelho #DC1431), QuickRegisterForm (seção 2.4 do frontend-spec), GPSIndicator, ChipSelect (tipologia/padrão), SaveButton (56px, verde #22C55E), ConfirmDiscard dialog.
- **PostGIS:** Coordenadas salvas como `ST_GeogFromText('SRID=4326;POINT({lng} {lat})')`. UUID gerado no client para suporte offline futuro.
- **Offline:** Nesta story, cadastro requer conexão. Suporte offline será adicionado na Story 1.6 (IndexedDB mutation queue + background sync).

## Notes

- **Performance crítica (<30s):** Este é o formulário mais importante do app. A consultora está andando, com uma mão, sob sol forte. Cada segundo conta. Auto-focus no campo nome, GPS automático, mínimo de taps.
- O reverse geocoding do Mapbox pode ser lento em áreas com sinal fraco. Implementar timeout de 3s com fallback para campo manual.
- UUIDs gerados no client (via `crypto.randomUUID()`) para preparar suporte offline na Story 1.6 — o ID é gerado antes do envio ao servidor.
- Se tipologia e padrão não forem preenchidos, o registro em `edificios_qualificacoes` ainda é criado com status_varredura = 'mapeado' e campos opcionais como null. Isso garante que o edifício aparece como "Mapeado" (azul) no mapa.
