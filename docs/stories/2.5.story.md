# Story 2.5: Biblioteca de Scripts de Objeção

**Epic:** 2 — Leads, Funil de Vendas & Metodologia RE/MAX
**Status:** Ready
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero acessar scripts de contorno de objeções pré-carregados e personalizáveis,
Para que eu esteja preparada para responder objeções comuns durante ligações e visitas.

## Acceptance Criteria

- [ ] AC1: Tela "Biblioteca de Scripts" acessível via: (a) tab "Mais" → "Scripts", (b) card do lead → botão "Scripts" (acesso contextual). Campo de busca no topo (input 48px, placeholder "Buscar objeção...", ícone lupa). Busca em tempo real com debounce 300ms. Query: `SELECT * FROM scripts WHERE (consultant_id = auth.uid() OR consultant_id IS NULL) AND (objecao ILIKE '%' || $1 || '%' OR resposta ILIKE '%' || $1 || '%' OR titulo ILIKE '%' || $1 || '%') ORDER BY is_default DESC, titulo ASC`.
- [ ] AC2: 5 scripts pré-carregados (seed data, `scripts.is_default = true`, `consultant_id = NULL`). Já inseridos na migration 002: (1) "Não trabalho com imobiliárias" — categoria `objecao_imobiliaria`, etapa `contato`, técnica "Reframe"; (2) "Experiência ruim com corretores" — categoria `objecao_experiencia`, etapa `contato`, técnica "Diferenciação"; (3) "Fecho de Duas Opções — Agendamento" — categoria `abordagem_inicial`, etapa `contato`, técnica "Neurologica"; (4) "Já tenho comprador interessado" — categoria `objecao_exclusividade`, etapa `v1_realizada`, técnica "Cláusula de Relacionamento"; (5) "Central Hub — Defesa da Exclusividade" — categoria `objecao_exclusividade`, etapa `v2_realizada`, técnica "Central Hub". Scripts default não são editáveis nem deletáveis pelo usuário (readonly).
- [ ] AC3: Filtro por categorias via chips toggleáveis abaixo da busca. Categorias baseadas no enum `categoria_script`: Objeção Imobiliária, Objeção Experiência, Objeção Exclusividade, Objeção Comissão, Objeção Preço, Abordagem Inicial, Fechamento, Follow-up. Chips 36px altura, outlined quando inativo, filled azul #003DA5 quando ativo. Múltiplos chips selecionáveis (filtro OR). Filtro por etapa do funil também disponível (chips secundários: Contato, V1, V2, Exclusividade).
- [ ] AC4: Script card expandível (accordion). Estado colapsado: título da objeção (font 16px bold, entre aspas), categoria (chip 24px), etapa do funil (chip cor correspondente). Estado expandido: contexto completo da objeção (`scripts.objecao`), resposta/script (`scripts.resposta` — font 15px, fundo cinza claro #F3F4F6, padding 16px, border-radius 12px), técnica utilizada (`scripts.tecnica` — chip destaque verde #22C55E). Botões: "Copiar" (copia resposta para clipboard + toast "Copiado!"), "Usar agora" (se acessado do lead, abre discador com telefone do lead).
- [ ] AC5: Acesso contextual do card do lead. Quando acessado via botão "Scripts" no lead card, a biblioteca abre com filtro pré-aplicado pela etapa do funil do lead. Ex: lead em `contato` → filtra scripts com `etapa_funil = 'contato'` ou `etapa_funil IS NULL`. Scripts da etapa do lead aparecem primeiro, demais abaixo separados por divisor "Outros scripts".
- [ ] AC6: Scripts personalizáveis. Botão "+ Novo Script" (outlined azul, 36px) no topo da lista. Formulário: título/objeção (input 48px, obrigatório), categoria (dropdown, obrigatório — enum `categoria_script`), etapa do funil (dropdown, opcional — enum `etapa_funil`), resposta/script (textarea, obrigatório, mín 20 caracteres), técnica (input, opcional). INSERT em `scripts` com `consultant_id = auth.uid()`, `is_default = false`. Scripts do usuário editáveis e deletáveis (swipe left → "Editar" / "Excluir" com confirmação).
- [ ] AC7: Scripts default não editáveis, mas duplicáveis. Ao tocar "Editar" em script default, prompt: "Scripts padrão não podem ser editados. Deseja criar uma cópia personalizada?". Se sim, duplica com `consultant_id = auth.uid()` e `is_default = false`, abre para edição. Original permanece intacto.

## Technical Notes

- **Database:** SELECT/INSERT/UPDATE/DELETE em `scripts` (id, consultant_id, titulo, categoria, etapa_funil, objecao, resposta, tecnica, is_default). Seed data: 5 scripts na migration 002 com `consultant_id = NULL` e `is_default = true`. Index: `idx_scripts_consultant` sugerido em `scripts(consultant_id)`.
- **API:** Supabase client SDK. TanStack Query key: `['scripts', consultant_id, filters]`. Busca client-side com filtro local (lista pequena, <100 scripts) — não precisa de busca server-side complexa. Mutation para criar/editar/excluir scripts do usuário.
- **Components:** ScriptLibrary (tela principal), ScriptSearchBar (busca com debounce), ScriptCategoryFilter (chips), ScriptCard (accordion expandível), ScriptForm (criar/editar), ScriptActions (Copiar, Usar agora).
- **Clipboard API:** `navigator.clipboard.writeText(resposta)` para botão "Copiar". Fallback: `document.execCommand('copy')` para browsers antigos.
- **Deep link telefone:** Botão "Usar agora" usa `tel:` URL scheme para abrir discador nativo.
