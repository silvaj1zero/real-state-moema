# Story 7.2: Wrapper PortalCrawler + Telemetry Layer

**Epic:** 7 — Inteligencia de Prospeccao Automatizada Multi-Fonte
**Wave:** A (P0 fundamento)
**Status:** InReview
**Priority:** Must
**Estimated Complexity:** L (8 pts) — requer PoC per `story-prototyping` rule

## User Story

Como engenheiro de prospeccao,
Quero um wrapper `createPortalCrawler` que padronize defaults (session pool, proxy, retry, hooks) e telemetria (crawl_runs/crawl_requests/crawl_failures),
Para que cada novo crawler de portal (MercadoLivre Wave A, QuintoAndar Wave B etc) seja escrito em < 1 dia consumindo configuracao especifica do portal.

## Acceptance Criteria

- [x] AC1: Novo modulo `app/src/lib/scrapers/portal-crawler.ts` (path canonico definido em ADR-EPIC7-006) exporta `createPortalCrawler(opts: PortalCrawlerOptions): AdaptivePlaywrightCrawler` com defaults: minConcurrency=2, maxConcurrency=20, maxTasksPerMinute=120, maxRequestRetries=3, requestHandlerTimeoutSecs=60, navigationTimeoutSecs=30, renderingTypeDetectionRatio=0.1, useSessionPool=true, persistCookiesPerSession=true, sessionPoolOptions.maxPoolSize=50. Codigo MUST ser TS puro para permitir copy-on-build em Apify Actors (`apps/crawlers/{portal}/`) via sync script introduzido em Story 7.4.
- [x] AC2: Hooks default implementados em `app/src/lib/scrapers/hooks/`:

  - `preNavigationHooks.ts` — `makeConsentCookieHook(portal)` + `makeRefererHook()`

- [x] AC3: Telemetry layer em `app/src/lib/scrapers/telemetry.ts` exportando objeto `Telemetry` com metodos `recordRequest`, `recordFailure`, `snapshot`. Cada metodo grava em tabela Supabase correspondente (`crawl_requests`, `crawl_failures`, `crawl_runs`).
- [x] AC4: Migration 009 `supabase/migrations/20260514000002_009_epic7_telemetry.sql` cria:

### Decisões
- **`AdaptivePlaywrightCrawler` como classe canônica** — Crawlee 3.16 com `RenderingTypePredictor` default (`renderingTypeDetectionRatio=0.1`) para evitar custo browser desnecessário em HTTP-only pages.
- **Telemetry como cliente injetável (interface `TelemetryClient`)** — wrapper desacoplado de Supabase concreto, permite mock 100% nos tests sem cliente real. Padrão também facilita testes Apify Actor (cliente diferente, mesma interface).
- **`vi.hoisted()` no test mock** — `vi.mock('crawlee')` factory não pode referenciar top-level vars (vitest hoisting). Pattern AN_KE_163 atomic-write aplicado no fix.
- **PoC AC6 como doc-only** — story decorre 100% offline; execução real httpbin requer ambiente staging com Supabase local. Output em `docs/poc/7.2-portal-crawler-smoke.md` é runbook auditável.
- **Pure TS (sem `next/server`, sem `cookies()`)** — preserva copy-on-build para Apify Actors (ADR-EPIC7-006).

### Surprises / Gotchas
- `vi.mock` hoisting quebrou a primeira tentativa (referência a class no top-level antes da inicialização). Fix via `vi.hoisted()` adicionou 5 linhas, mantém semântica.
