## Dev Agent Record

### Cardinalidade dos agentes — 2, não 4 (AC2, gates KISS)

A falha real citável (anti-lista 3 + gate-determinism) exige UMA fronteira: **executor ≠ auditor** — o auditor precisa de contexto independente e instrumento único (checklist), senão vira self-review com recálculo. `@acm-data`/`@acm-verifier`/`@acm-writer` são fases sequenciais do MESMO fluxo (SOP pós-H3), mesmo toolset (Bash + CLI), zero handoff de estado entre elas que justifique arquivos separados — nenhuma falha real citada para o split 4-way (gate KISS 1). Resultado: `acm-operator` (3 papéis como seções do mesmo agente) + `acm-auditor` (opus pinado, tools read-only — não pode rodar CLI nem "recalcular").
