## Story

Como operador, quero rodar `/acm-validate <endereço|pasta-do-caso>` e receber o pacote completo (dataset R5 + laudo + planilha Fase 1 + resumo de avisos) com um auditor automático dando veredito ancorado, para que um caso novo não exija copiar pasta de scripts nem depender de mim como herói.

## Contexto

A fundação está pronta: CLI `acm-validate` offline (P-1, `app/scripts/acm/acm-validate.tsx`), merge-back (P-2), pipeline canônico (`validatePipeline.ts`), R5 canônico (9.17), guard-rails (9.8/9.22). O que falta é a camada de ORQUESTRAÇÃO: skill + agentes com papéis separados e um gate de auditoria que — pela anti-lista — só pode nascer depois do checklist ancorado (9.28).

## Acceptance Criteria

1. **AC1 — Skill `/acm-validate`** em `.claude/skills/acm-validate/SKILL.md` DESTE repo (rule skill-squad-colocation: skill viaja com o squad do projeto, nunca no repo multi-tenant): recebe endereço ou pasta de caso, orquestra o fluxo e produz o pacote + relatório.
2. **AC2 — Agentes (personas enxutas, KISS):** `@acm-data` (dataset/R5/guias), `@acm-verifier` (Fase 1 humana assistida + cross-checks), `@acm-auditor` (aplica `CHECKLIST-ACM-AUDITOR-v1.md` da 9.28 — veredito ancorado), `@acm-writer` (emissão laudo/lite conforme política H-3). Avaliar se 4 arquivos são necessários ou se 2 bastam (gates KISS: estrutura só com falha real citada — justificar no Dev Record).
3. **AC3 — Zero lógica nova de cálculo:** a skill chama a CLI/lib existente; qualquer número vem do `computeLaudo`. Proibido o auditor "recalcular" (anti-lista 3: CLI que empacota script sujo).
4. **AC4 — Veredito ancorado:** o `@acm-auditor` usa EXCLUSIVAMENTE o checklist da 9.28 (condição→banda); saída PASS/CONCERNS/FAIL com evidência campo a campo do computation.json.
5. **AC5 — Prova ponta a ponta:** rodar a skill num endereço NOVO (não Honduras/AP) até o pacote sair; DoD do veredito ("Lite ou CLI gera pacote para 1 endereço novo sem copiar pasta") cumprido via skill.
6. **AC6 — Validação de skill:** `validate-skill`/lint de skill do framework passa; descrição da skill ≤ 180 chars, trigger-first.
