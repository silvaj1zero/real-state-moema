# Story 6.6: Lookup de Proprietario via Cartorio (Infosimples/ARISP)

**Epic:** 6 — Busca Parametrica On-Demand com Enriquecimento de Contatos
**Status:** Done
**Priority:** Should
**Estimated Complexity:** M

> **Fronteira da API paga (plano 2026-07-08):** story implementada ATE a fronteira
> da Infosimples — schema, endpoint, guardas, client, hook e testes completos, mas
> a consulta real so roda com `OWNER_LOOKUP_ENABLED=true` + `INFOSIMPLES_TOKEN`
> (decisao de custo do founder). Flag OFF (default) → endpoint responde 503 sem
> consumir credito.

## User Story

Como consultora RE/MAX,
Quero descobrir o nome do proprietario de um edificio especifico a partir do endereco,
Para que eu possa abordar o dono com uma oferta personalizada mesmo quando nao ha anuncio FISBO ativo.

## Acceptance Criteria

- [x] AC1: Novo endpoint `POST /api/owners/lookup` protegido por sessao Supabase (authenticated). Body: `{ edificio_id: uuid }` ou `{ sql_lote: string, endereco: string }`. Orquestra o pipeline em 3 etapas: (a) obter SQL do lote via GeoSampa (reutiliza Story 3.5 se ja houver `edificios.sql_lote`, caso contrario consulta GeoSampa WFS on-the-fly por endereco), (b) consulta Infosimples/ARISP API com o SQL para obter matricula e nome do proprietario, (c) grava resultado em nova tabela `owner_lookups` e responde com payload consolidado.
- [x] AC2: Nova tabela `owner_lookups` (schema em migration 007): `id UUID PK, consultant_id UUID FK auth.users, edificio_id UUID FK edificios ON DELETE CASCADE, sql_lote TEXT, matricula TEXT, nome_proprietario TEXT, cpf_cnpj_masked TEXT, cartorio TEXT, data_matricula DATE, ultima_transacao DATE, fonte TEXT DEFAULT 'infosimples', custo_brl NUMERIC(6,2), raw_response JSONB, status TEXT CHECK (status IN ('pending','success','failed','not_found')), error_message TEXT, created_at TIMESTAMPTZ, updated_at TIMESTAMPTZ`. UNIQUE (edificio_id, matricula). Indices em `consultant_id`, `edificio_id`, `created_at DESC`, `(edificio_id, status) WHERE status='success'`.
- [x] AC3: RLS completo em `owner_lookups`: SELECT/INSERT/UPDATE/DELETE com `consultant_id = auth.uid()`. GRANTs para authenticated e service_role.
- [x] AC4: Rate limit por consultor: maximo 30 lookups / hora via funcao `fn_check_owner_lookup_rate_limit(p_consultant_id uuid)`. Retorna 429 com `Retry-After` se excedido. Limite configuravel via env var `OWNER_LOOKUP_RATE_LIMIT` (default 30).
- [x] AC5: Budget guard: rejeitar request se custo mensal (SUM `custo_brl` do consultor no mes corrente) + custo estimado (0.28) exceder `OWNER_LOOKUP_BUDGET_BRL` (default 60.00). Retorna 402 com mensagem amigavel apontando o consumo atual.
- [x] AC6: Cache de 90 dias: antes de chamar Infosimples, busca em `owner_lookups` onde `edificio_id = $1 AND status='success' AND created_at > now() - interval '90 days'`. Se existir, retorna do cache com flag `cache_hit: true, custo_brl: 0`. Caso contrario consulta API, salva e retorna `cache_hit: false`.
- [x] AC7: Novo modulo `app/src/lib/infosimples.ts` encapsula a chamada HTTPS com token em env var `INFOSIMPLES_TOKEN`. Timeout 20s, 2 retries com backoff 1s/3s. Se API retorna vazio ou 404, grava `status='not_found'` em `owner_lookups` com `custo_brl=0`. Se API devolve erro tecnico, `status='failed'`.
- [x] AC8: Hook `useOwnerLookup(edificio_id)` via React Query: mutation dispara `POST /api/owners/lookup`, query opcional puxa o ultimo lookup bem sucedido para o edificio. Invalida cache local ao completar.
- [x] AC9: Evento `intelligence_feed` tipo `owner_lookup_completo` apos sucesso, com metadata `{ edificio_id, nome_proprietario, cache_hit, custo_brl }`. Rastreabilidade alinhada ao modelo do Epic 3.
- [x] AC10: LGPD compliance: campo `cpf_cnpj_masked` nunca armazena CPF completo, apenas mascara `***.***.***-XX` (apenas ultimos 2 digitos) para permitir desambiguacao sem expor dado sensivel. Endpoint `POST /api/owners/:id/forget` chama `fn_anonimize_owner_lookup(p_id uuid)` zerando nome e matricula. RLS garante que apenas o consultor dono pode esquecer.

## Technical Notes

- **Infosimples/ARISP:** API documentada em `https://api.infosimples.com/api/v2/consultas/arisp/certidao`. Token mensal do cliente em env var privada. Custo por consulta: R$0.28 (valor 2026). Plan `CARTORIO_1000` ofertado em pacotes — avaliar custo efetivo com analyst.
- **GeoSampa fallback:** se `edificios.sql_lote IS NULL`, consultar WFS da camada `CADASTRO_IPTU_LOTE` via endpoint `https://wfs.geosampa.prefeitura.sp.gov.br/geoserver/wfs` com filtro espacial `INTERSECTS(geometry, POINT(lng lat))`.
- **Cache rationale:** matriculas de imoveis mudam raramente (venda/heranca); 90 dias equilibra frescor com custo.
- **Observabilidade:** logs estruturados `{ event: 'owner_lookup', edificio_id, status, elapsed_ms, cost_brl }` via `console.log` — Vercel captura.

## Dev Notes

- **Fronteira da API paga:** `executeOwnerLookup` verifica a flag DEPOIS do cache e das guardas e ANTES de qualquer consulta/persistência — com flag OFF nada é gravado nem cobrado (testado). Cache hit continua funcionando mesmo com flag OFF.
- **Arquitetura testável:** a lógica de decisão mora em `owner-lookup-service.ts` (pura, store fake nos testes); `owner-lookup-store.ts` é I/O fino; a rota é só fiação. 42 testes novos sem tocar DB nem API.
- **Contrato Infosimples assumido:** envelope v2 `{code, code_message, data[]}` (200=sucesso, 600-619=sem resultado) — a confirmar quando a credencial for contratada; ajuste isolado em `parseInfosimplesEnvelope`.
- **`edificio_id` NULLABLE (ajuste consciente ao AC2):** a variante de body `{sql_lote, endereco}` não tem edifício; cache/dedup caem para `sql_lote`.
- **Rate limit (AC4):** `fn_check_owner_lookup_rate_limit` retorna `{count, oldest_at}`; o endpoint compara com `OWNER_LOOKUP_RATE_LIMIT` e deriva `Retry-After` do lookup mais antigo da janela.
- **Pendência de deploy:** migration 023 precisa ser aplicada no Supabase (SQL Editor) antes da rota funcionar em runtime — padrão do projeto.
- **GeoSampa:** fallback não persiste `sql_lote` em `edificios` (RLS de escrita restrita); apenas usa na consulta.

## Risks

|---|---|
| Infosimples fora do ar / indisponivel | Endpoint retorna `503` estruturado, UI mostra retry manual. Cache serve respostas anteriores. |
| Custo acima do orcado | AC5 enforce budget guard; dashboard exibe consumo mensal. |
| CPF do proprietario aparecer em logs | AC10 mascara antes de persistir; log estruturado nao inclui raw_response em texto livre. |
| GeoSampa mudar URL / layer | Isolar em modulo `lib/geosampa.ts` com retry + fallback para IPTU tabular. |
