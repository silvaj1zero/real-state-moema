# Story 6.3: UI de Busca Parametrica

**Epic:** 6 — Busca Parametrica On-Demand com Enriquecimento de Contatos
**Status:** Done
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero uma tela onde eu defina filtros de busca, selecione a area no mapa ou escolha condominios pesquisados, e veja os resultados,
Para que eu encontre oportunidades de captacao de forma direcionada e eficiente.

## Acceptance Criteria

- [x] AC1: Nova tela "Busca Portais" acessivel via menu lateral (rota `/search`).
- [x] AC2: Secao "Area de busca" com 2 modos: "Raio no Mapa" (reutiliza AcmRadiusSelector + mini-mapa) | "Condominios" (multi-select edificios pesquisados).
- [x] AC3: Formulario de filtros: quartos (stepper), suites, banheiros, area m2 (min/max), preco (min/max R$), tipo transacao (chips Venda/Aluguel), portais (multi-select ZAP/OLX/VivaReal), toggle FISBO.
- [x] AC4: Zustand store `useSearchStore` para persistir estado dos filtros entre navegacoes.
- [x] AC5: Hook `useParametricSearch` com trigger mutation, polling de status a cada 5s, query de resultados.
- [x] AC6: Resultados: cards com badges (portal, FISBO, Novo), filtro local rapido, contagem no topo.
- [x] AC7: Busca local primeiro via `fn_scraped_listings_parametric` (instantaneo, gratis). Toggle "Buscar nos portais tambem?" aciona Apify.
- [x] AC8: Historico de buscas colapsavel: ultimas 10, recarrega filtros, "Repetir busca".
