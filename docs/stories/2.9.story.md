# Story 2.9: Dashboard de KPIs

**Epic:** 2 — Leads, Funil de Vendas & Metodologia RE/MAX
**Status:** Done
**Priority:** Must
**Estimated Complexity:** XL

## User Story

Como consultora RE/MAX,
Quero visualizar todos os KPIs do meu negócio em um dashboard unificado e responsivo,
Para que eu acompanhe minha performance diária e saiba se estou no caminho certo para minhas metas.

## Technical Notes

- **Database:** Múltiplas queries agregadas em `leads`, `funnel_transitions`, `informantes`, `frog_contacts`, `agendamentos`, `edificios`/`edificios_qualificacoes`. Nenhuma tabela nova — dashboard é view de dados existentes. Considerar Supabase Edge Function para agregar todas as métricas em uma única chamada (reduzir round trips).
- **API:** Supabase client SDK. Idealmente, uma Edge Function `/api/dashboard` que retorna todas as métricas em um JSON único. TanStack Query key: `['dashboard', consultant_id, periodo]`. StaleTime: 2min (métricas mudam com frequência). Prefetch ao navegar para tab Dashboard.
- **Components:** DashboardScreen (tela principal com scroll), DailyGoal (meta diária com barra), TerritoryKPIs (grid 2x2), FunnelSummary (funil compacto), FrogChips (chips por categoria), InformantMetrics (linha de métricas), PeriodFilter (dropdown), EmptyState (reutilizável por seção), KPICard (card individual 165x80px), GoalConfig (modal de configuração de meta).
- **Performance:** Dashboard agrega dados de 6+ tabelas. Para performance, considerar: (1) Parallel queries com `Promise.all`, (2) Edge Function server-side para agregar, (3) View materializada para métricas que mudam pouco (cobertura). Skeleton loading durante fetch.
- **Offline:** Dashboard não funciona offline (depende de queries complexas). Exibir último snapshot cacheado com aviso "Dados de [data/hora]" quando offline.

## Notes

- **FR-019:** Dashboard com densidade de carteira, velocidade de varredura, taxa de domínio, conversão do funil, meta diária.
- **Princípio PV — Empty states com mensagens motivacionais:** Quando a Luciana tem poucos dados (início), o dashboard NÃO deve parecer vazio e desmotivante. Cada seção tem empty state com mensagem positiva e botão de ação para começar.
- **Princípio PV — Meta diária "5 V1s/dia":** Meta padrão baseada na metodologia RE/MAX de alta performance. Configurável para adaptar ao ritmo da consultora.
- Dashboard é a tela de "reflexão" — Luciana abre no início do dia (o que fazer?) e no final (como fui?). A saudação contextual humaniza.
- Considerar cache agressivo (staleTime 2min) e skeleton loading para UX fluida — dashboard não pode parecer lento.
