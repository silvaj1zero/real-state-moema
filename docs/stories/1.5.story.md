# Story 1.5: Status de Varredura e Visualização por Cores

**Epic:** 1 — Foundation, Mapa & Registro de Campo
**Status:** Ready
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero ver pela cor do pin o status de cada edifício no mapa,
Para que eu saiba rapidamente onde já passei, o que está em prospecção e o que falta visitar.

## Acceptance Criteria

- [ ] AC1: Pins do mapa coloridos por status conforme frontend-spec seção 2.2 (MapPin component seção 3.1): Cinza (#9CA3AF) = Não Visitado (`nao_visitado`), Azul (#003DA5) = Mapeado (`mapeado`), Amarelo (#EAB308) = Em Prospecção (`em_prospeccao`), Verde (#22C55E) = Concluído (`concluido`). Status vem do campo `status_varredura` da tabela `edificios_qualificacoes`.
- [ ] AC2: Status alterável via Card do Edifício (bottom sheet da Story 1.4). Dropdown ou chip select no header do card. UPDATE em `edificios_qualificacoes.status_varredura`. Mudança reflete imediatamente na cor do pin no mapa (optimistic update via TanStack Query invalidation).
- [ ] AC3: Legenda com contagem por status conforme frontend-spec — barra recolhível (36px colapsada) na parte inferior do mapa. Mostra: C:N (cinza), M:N (azul), P:N (amarelo), V:N (verde). Tap expande com nomes completos e ação de filtro. Cores alinhadas com os pins.
- [ ] AC4: Clustering automático em zoom-out via Mapbox GL JS cluster source. Configuração: `cluster: true`, `clusterMaxZoom: 14`, `clusterRadius: 50`. Cluster properties agregadas por status (`nao_visitado`, `mapeado`, `em_prospeccao`, `concluido`). Zoom <13: clusters grandes com contagem total. Zoom 13-14: clusters menores com contagem por status. Zoom >=15: pins individuais.
- [ ] AC5: Filtro rápido por status no overlay de filtro (tap no ícone funil do header). Checkboxes por status (todos ativos por default). Filtros aplicados mostram badge numérico no ícone do filtro. Implementado como filtro no Mapbox GL JS data source (filter expressions).
- [ ] AC6: Percentual de cobertura do raio ativo exibido no centro do header do mapa ("Moema N%"). Calculado via query PostGIS Q2: edifícios com `status_varredura != 'nao_visitado'` / total de edifícios no raio ativo. Atualiza ao mudar status, cadastrar edifício, ou mudar raio ativo.
- [ ] AC7: Badges visuais nos pins: badge "auto" (borda pontilhada + ícone "A") para edifícios com `origem = 'seed'` e `verificado = false`. Badge "verificado" (borda sólida + checkmark) para `verificado = true`. Badge FISBO (estrela vermelha #DC1431) para `is_fisbo_detected = true`. Tamanho badge: 12x12px, posição top-right do pin.
- [ ] AC8: Pin selecionado (com card aberto): borda branca 2px + sombra + escala 1.2x. Animação de transição suave ao selecionar/deselecionar.

## Technical Notes

- **Database:** SELECT de `edificios` JOIN `edificios_qualificacoes` com campos status_varredura, is_fisbo_detected. Query PostGIS Q2 para cálculo de cobertura percentual (ver arquitetura seção 4.3). UPDATE em `edificios_qualificacoes.status_varredura` via card. Edifícios sem registro em `edificios_qualificacoes` (ex: seed data sem qualificação) são tratados como `nao_visitado`.
- **API:** GeoJSON endpoint que retorna edifícios com status para o Mapbox source. TanStack Query com cache key `['buildings-geojson', epicentro_id, raio]`. Invalidation ao mudar status.
- **UI Components:** MapPin (seção 3.1 — custom Mapbox marker com HTML), StatusLegend (barra recolhível com contagens), FilterOverlay (checkboxes por status + "Apenas FISBOs" + "Apenas meu raio"), CoveragePercentage (header center), ClusterMarker (círculo com número + cor predominante).
- **PostGIS:** Query Q2 para cobertura: `COUNT(qualificados) / COUNT(total no raio) * 100`. Usa `ST_DWithin` para filtrar edifícios no raio ativo.
- **Offline:** Status em cache local via TanStack Query. Mudanças de status offline via mutation queue (Story 1.6).

## Notes

- **Mapbox expressions para cores de pin:** Usar `match` expression no layer paint para mapear status → cor. Isso é mais performático do que custom markers individuais para grandes volumes.
- Alternativa: se volume de pins for alto (>500), considerar usar Mapbox Symbol layer com `icon-color` expression ao invés de HTML markers custom. HTML markers são mais flexíveis mas menos performáticos.
- A legenda deve reagir em tempo real a mudanças — usar Zustand ou TanStack Query subscription para atualizar contagens ao adicionar/editar edifícios.
- Edifícios sem qualificação (sem registro em `edificios_qualificacoes` para o consultant atual) devem aparecer como Cinza (Não Visitado). Isso é importante para seed data (Story 1.7) — edifícios pré-carregados não terão qualificação até a consultora interagir.
- O percentual de cobertura é o principal indicador de progresso territorial. Deve ser destacado visualmente e atualizar-se automaticamente. Será usado na Story 2.7 (Expansão por Raio — desbloqueio ao atingir 80%).
