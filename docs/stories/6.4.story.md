# Story 6.4: Enriquecimento de Contatos do Anunciante

**Epic:** 6 — Busca Parametrica On-Demand com Enriquecimento de Contatos
**Status:** Done
**Priority:** Should
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero que o sistema extraia e enriqueca dados de contato (telefone, email, WhatsApp) dos anunciantes dos resultados de busca,
Para que eu tenha informacoes completas para abordar proprietarios diretamente na minha captacao.

## Acceptance Criteria

- [x] AC1: `normalizeContactData()` extrai contatos do raw Apify (nome, telefone, email, WhatsApp, CRECI) durante normalizacao.
- [x] AC2: Validacao CRECI via BuscaCRECI API (timeout 5s, fallback graceful). Se CRECI invalido, marca como possivelmente FISBO.
- [x] AC3: Enriquecimento endereco via ViaCEP (CEP -> logradouro completo, bairro, cidade).
- [x] AC4: Hook `useContactEnrichment` para enriquecimento on-demand individual e batch.
- [x] AC5: `POST /api/search/enrich-contact` endpoint server-side com autenticacao JWT.
- [x] AC6: ContactDataCard no resultado: icones tel/WhatsApp/email, mascaramento LGPD, botao "Revelar", badge CRECI.
- [x] AC7: LGPD: `lgpd_consent_origin` tracking em cada dado de contato. `fn_anonimize_contact_data` para direito ao esquecimento.
- [x] AC8: Batch enrichment: "Enriquecer todos (N)" com progress bar e rate limiting (1 req/s).
