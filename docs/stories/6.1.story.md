# Story 6.1: Schema, RPC e Tabelas para Busca Parametrica

**Epic:** 6 — Busca Parametrica On-Demand com Enriquecimento de Contatos
**Status:** Done
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero que o sistema tenha a infraestrutura de banco de dados para buscar anuncios com filtros parametricos e armazenar historico de buscas,
Para que minhas buscas nos portais sejam rastreadas, auditadas e os resultados filtrados com performance no servidor.

## Acceptance Criteria

- [x] AC1: Nova tabela `portal_searches` com colunas: `id UUID PK`, `consultant_id UUID FK auth.users NOT NULL`, `status search_status NOT NULL DEFAULT 'pending'`, `search_params JSONB NOT NULL`, `portals TEXT[] NOT NULL`, `results_count INTEGER DEFAULT 0`, `new_listings_count INTEGER DEFAULT 0`, `fisbo_count INTEGER DEFAULT 0`, `apify_run_ids JSONB`, `apify_cost_usd NUMERIC(8,4)`, `error_message TEXT`, `started_at TIMESTAMPTZ`, `completed_at TIMESTAMPTZ`, `created_at TIMESTAMPTZ DEFAULT now()`, `updated_at TIMESTAMPTZ DEFAULT now()`. Indexes em consultant_id e created_at DESC.
- [x] AC2: Nova tabela `portal_search_results` (junction search <-> scraped_listings). Colunas: `id UUID PK`, `search_id UUID FK portal_searches NOT NULL ON DELETE CASCADE`, `scraped_listing_id UUID FK scraped_listings NOT NULL`, `is_new BOOLEAN DEFAULT true`, `created_at TIMESTAMPTZ DEFAULT now()`. UNIQUE(search_id, scraped_listing_id).
- [x] AC3: Novas colunas em `scraped_listings`: `telefone_anunciante TEXT`, `email_anunciante TEXT`, `whatsapp_anunciante TEXT`, `nome_anunciante TEXT`, `creci_anunciante TEXT`, `contact_enriched_at TIMESTAMPTZ`, `lgpd_consent_origin TEXT`.
- [x] AC4: Novos enums: `search_status` ('pending', 'running', 'completed', 'failed', 'cancelled'). Novo valor em `tipo_feed`: 'busca_parametrica'.
- [x] AC5: Nova RPC `fn_scraped_listings_parametric(p_lat, p_lng, p_raio_metros, p_quartos_min, p_quartos_max, p_area_min, p_area_max, p_preco_min, p_preco_max, p_bairros, p_fisbo_only, p_portal, p_limit)` — busca server-side com PostGIS ST_DWithin + filtros parametricos. Retorna TABLE com colunas de scraped_listings + distancia_m. Todos parametros exceto lat/lng/raio sao opcionais (NULL = sem filtro).
- [x] AC6: Nova RPC `fn_anonimize_contact_data(p_listing_id)` — SET NULL em todos campos de contato de uma listing (LGPD direito ao esquecimento).
- [x] AC7: RLS policies: portal_searches CRUD own (consultant_id = auth.uid()), portal_search_results SELECT para authenticated.
- [x] AC8: Tipos TypeScript atualizados em types.ts com SearchStatus, PortalSearch, PortalSearchResult, campos de contato em ScrapedListing.
