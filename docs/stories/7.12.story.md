# Story 7.12: Proxy residencial BR no crawler para contornar 403 (Cloudflare)

**Epic:** 7 — Inteligencia de Prospeccao Automatizada Multi-Fonte
**Wave:** A (cobertura/anti-bloqueio)
**Status:** Done
**Priority:** Should (destrava cobertura de portais bloqueados)
**Estimated Complexity:** S-M (3-5 pts)

## User Story

Como sistema de prospeccao,
Quero rotear os crawlers de portais protegidos (ZAP/VivaReal) por proxy residencial BR,
Para contornar os bloqueios 403 do Cloudflare e ampliar a cobertura de anúncios ingeridos da região.

## Contexto

A pesquisa `docs/research/2026-06-15-fisbo-captacao-ingestao/report.md` (achados F5 média 2-1, F6 alta 3-0) confirmou: portais BR ficam atrás de Cloudflare e **bloqueiam IPs de datacenter por reputação de ASN no edge** (403/503); IP **residencial BR** atinge 85-99% de sucesso. O **Crawlee já aplica** fingerprints/headers realistas, rotação de user-agent e `SessionPool` por padrão — o que falta é fornecer o **pool de proxy residencial** via `ProxyConfiguration` (o usuário fornece os proxies).

`portal-crawler.ts` já aceita `proxyConfiguration?: ProxyConfiguration` (L85, repassado L179). Falta configurá-lo com grupo RESIDENTIAL + país BR para os alvos Cloudflare, conforme ADR-EPIC7-002 (Apify cloud Wave A; IPRoyal Wave B).

> ⚠️ **Refutado pela verificação adversarial (não usar como base):** retries agressivos NÃO derrotam 403; rotação de proxy SOZINHA não resolve; 403 não é primariamente fingerprinting. O fator decisivo é a **reputação do IP (residencial vs datacenter)** + o que o Crawlee já faz.

## Acceptance Criteria

- [x] AC1: Config de `ProxyConfiguration` com grupo `RESIDENTIAL` + `countryCode='BR'` (Apify, Wave A) aplicada aos crawlers de **portais protegidos** (ZAP/VivaReal). Fontes não-Cloudflare (ex.: MercadoLivre) seguem com o proxy atual/datacenter (custo menor). → `resolveProxySpec` + `buildProxyConfiguration` (proxy-config.ts).
- [x] AC2: Seleção de proxy **por alvo** — um mapa `portal → proxyTier` (residential | datacenter | none), não residencial global (residencial é mais caro; usar onde há bloqueio). → `PORTAL_PROXY_TIER` (zap/vivareal=residential; olx/mercadolivre=datacenter).
- [x] AC3: Não reinventar anti-bot: confiar no `SessionPool` + fingerprints default do Crawlee; apenas garantir `useSessionPool: true` e `persistCookiesPerSession: true` nos crawlers protegidos. → já são defaults em `PORTAL_CRAWLER_DEFAULTS`; teste afirma p/ alvo protegido.
- [x] AC4: Telemetria de bloqueio — registrar taxa de 403/503 por portal **antes/depois** (reusar telemetria do Epic 7 — `..._009_epic7_telemetry`) para medir o ganho real (meta: <15% de bloqueio nos alvos residenciais). → `isBlockStatus` + contador `requests_blocked`/`block_rate` no snapshot; anti-bot (`AntiBotDetectedError`) conta como bloqueio. `status_code` já persistido em `crawl_requests` (migration 009) — sem nova migration.
- [x] AC5: Env/config: variáveis para grupos de proxy (`APIFY_PROXY_GROUPS`, `APIFY_PROXY_COUNTRY`) documentadas; sem hardcode de credencial. Fallback gracioso se o proxy residencial não estiver disponível (loga e degrada para datacenter, não quebra). → `readProxyEnv` + fallback `degradedFrom` em `resolveProxySpec`; `.env.local.example` documentado.
- [x] AC6: Nota de custo no story/ADR — residencial tem custo/GB maior; estimar volume e gatilho de migração para IPRoyal self-host (Wave B) conforme ADR-EPIC7-002 (gatilho ≥50k págs/mês). → anotação "Story 7.12" no ADR-EPIC7-002.
- [x] AC7: Testes: mock de `ProxyConfiguration` confirma grupo/país por alvo; alvo não-Cloudflare não recebe residencial; fallback quando proxy indisponível. → `proxy-config.test.ts` (factory injetável).

- Scraping de novos portais (esta story é sobre destravar os já cobertos).

## Technical Notes

- **Mecanismo real (report):** Cloudflare classifica por **ASN/reputação de IP** no edge → datacenter cai. Residencial BR passa. Não é (só) fingerprint.
- Crawlee: fingerprints completos só em browser crawlers (Playwright/Puppeteer); HTTP crawlers usam `got-scraping` com `useHeaderGenerator`. Escolher o tipo de crawler conforme o alvo.
- Cloudflare é **alvo móvel** — instrumentar a telemetria (AC4) para detectar regressão de bloqueio cedo.
