## Technical Notes

- **Fonte de verdade:** O schema atual no Supabase remoto e a fonte. O dump captura o estado real.
- **PostGIS:** Extension `postgis` deve ser criada na migration 000 antes de qualquer tabela que use `geography`.
- **RLS:** Policies existentes devem ser incluidas no dump. Se nao existirem, isso e finding adicional.
- **Numeracao:** Usar timestamps para manter ordem (Supabase convention). Migrations existentes mantem seus timestamps.
- **Risco:** Se o schema remoto tem tabelas que o codigo nao usa (lixo), limpar antes do dump.
