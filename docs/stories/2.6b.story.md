# Story 2.6b: Checklist de Preparação V1→V2 (VETO PV #2)

**Epic:** 2 — Leads, Funil de Vendas & Metodologia RE/MAX
**Status:** Done
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero que o sistema me guie na preparação obrigatória antes de cada V2 com um checklist estruturado,
Para que eu nunca entre em uma V2 despreparada e transmita profissionalismo ao proprietário.

## Acceptance Criteria

- [x] AC1: Checklist criado automaticamente ao agendar V2 (Story 2.6, AC6). INSERT em `checklists_preparacao` (lead_id, consultant_id, tipo = 'preparacao_v2', data_v2 = agendamento.data_hora, todos os campos booleanos = false). Checklist acessível via: (a) detalhe do lead → botão "Checklist V2" (badge com progresso "3/5"), (b) agenda → agendamento V2 → link "Preparação", (c) notificação 24h antes.
- [x] AC2: 5 itens do checklist obrigatórios, exibidos como lista com checkboxes. Cada item com estado visual: pendente (fundo cinza claro #F3F4F6, checkbox vazio), completo (fundo verde claro #F0FDF4, checkbox verde #22C55E com checkmark). Itens:
  1. **ACM preparada** (`checklists_preparacao.acm_preparada`) — Sub-ação: "Gerar ACM" abre ACM Generator (Story 3.1). Se ACM já existe para o lead, mostra resumo: "R$ [valor]/m² | [N] comparáveis".
  2. **Dossiê montado** (`checklists_preparacao.dossie_montado`) — Sub-ação: "Gerar Dossiê" abre Dossiê Generator (Story 3.2). Se Dossiê já existe, mostra: "PDF pronto | [N] páginas".
  3. **Home Staging enviado** (`checklists_preparacao.home_staging_enviado`) — Sub-ação: "Gerar e Enviar WhatsApp" (detalhes no AC4).
  4. **Matrícula verificada** (`checklists_preparacao.matricula_verificada`) — Sub-ação: campo input para número da matrícula + botão "Consultar" (link externo para cartório online, MVP). Checkbox marcável manualmente.
  5. **Plano de Marketing rascunhado** (`checklists_preparacao.plano_marketing_rascunhado`) — Sub-ação: "Ver Plano" abre template com itens do plano de marketing (Story 4.8). Se plano não existe, mostra template genérico.
- [x] AC3: Barra de progresso visual no topo do checklist. Formato: "Progresso: [N]/5 [========>    ] [%]". Cores da barra: vermelho 0-40%, amarelo 40-80%, verde 80-100%. Countdown para data da V2: "V2 agendada: [data] às [hora] — Faltam: [N] dias". Countdown em vermelho se <24h e itens pendentes. Font tamanho: countdown 16px, progresso 14px.
- [x] AC4: Home Staging compartilhável via WhatsApp. Botão "Compartilhar via WhatsApp" na seção Home Staging do checklist. Preview do conteúdo antes de enviar: template com branding Luciana Borba, 3 regras de ouro de home staging, personalizável por tipologia do imóvel (residencial vs comercial). Deep link WhatsApp: `https://wa.me/{telefone}?text={mensagem_encoded}`. Telefone decriptado de `leads.telefone_encrypted` para gerar o link. Ao compartilhar com sucesso, checkbox `home_staging_enviado` marcado automaticamente. Template salvo/editável (Story 3.3 completa o feature).
- [x] AC5: Notificação 24h antes da V2 se itens pendentes. Push notification: "V2 com [Nome] amanhã às [hora]. [N] itens pendentes no checklist." Tap na notificação abre o checklist. Service Worker verifica `checklists_preparacao` WHERE `data_v2 BETWEEN NOW() AND NOW() + INTERVAL '25 hours'` e algum campo booleano = false. Se TODOS os itens completos, notificação positiva: "V2 com [Nome] amanhã — Preparação 100% completa!"
- [x] AC6: Checklist toggle — tap no checkbox alterna estado. UPDATE em `checklists_preparacao` SET `campo = NOT campo` WHERE `id = $1`. Animação: checkbox com spring animation (scale 0→1.2→1.0, 200ms) + ícone checkmark aparece. Cada toggle é salvo imediatamente (não precisa de botão "Salvar"). TanStack Query mutation com optimistic update na barra de progresso.
- [x] AC7: Checklist como gate visual (não bloqueante no MVP). O checklist NÃO bloqueia a V2 no MVP — é um guia visual. Badge amarelo "Preparação incompleta" no agendamento V2 se <100%. No futuro (Epic 3+), pode tornar-se bloqueante. VETO PV #2: "impossibilitar V2 sem preparação" — no MVP, impossibilitar = tornar impossível ESQUECER, não bloquear tecnicamente.

## Technical Notes

- **Database:** INSERT/UPDATE em `checklists_preparacao` (id, lead_id, consultant_id, tipo, acm_preparada, dossie_montado, home_staging_enviado, matricula_verificada, plano_marketing_rascunhado, data_v2). Index: `idx_checklists_lead` em `(lead_id)`. COMMENT na tabela explica o VETO PV #2.
- **Push Notifications:** Mesma infraestrutura da Story 2.6. Check periódico (a cada 1h) de checklists com `data_v2` próxima e itens pendentes.
- **WhatsApp Deep Link:** `https://wa.me/${telefoneFormatado}?text=${encodeURIComponent(mensagemHomeStaging)}`. Telefone: decriptar via `pgp_sym_decrypt`, formatar para internacional (+55...). Mensagem: template com variáveis (nome do proprietário, endereço, tipologia).
- **API:** Supabase client SDK. TanStack Query key: `['checklist', lead_id]`. Mutation individual por campo (toggle único, não form inteiro).
- **Components:** ChecklistV2 (tela principal), ChecklistItem (item com checkbox + sub-ação), ChecklistProgress (barra + countdown), HomeStageShareButton (WhatsApp), ChecklistNotification (push 24h).

## Dependencies

- Depends on: 2.6 (agendamento V2 cria checklist), 2.1 (leads com telefone para WhatsApp), 1.1 (infra, DB, auth)
- Blocks: 3.2 (Dossiê atualiza checklist), 3.3 (Home Staging completo atualiza checklist)

## File List

- `app/src/hooks/useChecklist.ts` — TanStack Query hooks: useChecklistByLead, useCreateChecklist (idempotent), useUpdateChecklistItem (optimistic toggle). Exports CHECKLIST_ITEMS config and countCompleted helper.
- `app/src/components/checklist/ChecklistV2.tsx` — Main checklist UI: auto-creates on mount, progress bar (red/yellow/green), 5 toggleable items with checkmark animation, 24h notification banner, integrates HomeStageShare.
- `app/src/components/checklist/HomeStageShare.tsx` — WhatsApp deep link share: 3 Home Staging rules (Neutralidade, Espaco Emocional, Higiene Visual), generates wa.me link with encoded message template.

## Notes

- **VETO PV #2 — Checklist V1→V2 impossibilita V2 sem preparação.** No MVP, "impossibilitar" = tornar impossível ESQUECER (notificações, badges, progresso visual), não bloquear tecnicamente. A consultora pode prosseguir mesmo com itens pendentes, mas o sistema deixa claro o que falta.
- **Princípio PV — Automação sugere, humano decide:** O checklist sugere ações (gerar ACM, enviar Home Staging), mas a consultora decide quando e como executar cada uma.
- Home Staging compartilhável via WhatsApp é MVP — template básico com 3 regras. Story 3.3 (Epic 3) completa com personalização por tipologia e branding completo.
- Sub-ações de ACM e Dossiê abrem telas de outros épicos (3.1, 3.2). Se esses épicos não estiverem implementados, sub-ações exibem "Em breve" com placeholder.
