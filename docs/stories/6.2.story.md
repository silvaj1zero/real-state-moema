# Story 6.2: API Route Busca Parametrica On-Demand via Apify

**Epic:** 6 — Busca Parametrica On-Demand com Enriquecimento de Contatos
**Status:** Done
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero disparar uma busca parametrica nos portais com meus filtros escolhidos,
Para que o sistema colete anuncios frescos que correspondam exatamente ao perfil que estou buscando para captacao.

## Acceptance Criteria

- [x] AC1: `POST /api/search/parametric` aceita body JSON com portals, params (quartos, area, preco, localizacao, tipo transacao) e consultant_id. Requer autenticacao JWT. Retorna `{ search_id, status: 'pending' }` imediatamente.
- [x] AC2: Processamento assincrono via `waitUntil`. Cria portal_searches, dispara Apify para cada portal, normaliza resultados, upserta em scraped_listings, insere em portal_search_results, atualiza status para 'completed'.
- [x] AC3: `buildParametricSearchInput(portal, params)` constroi input Apify com filtros do usuario mapeados por portal (ZAP, OLX, VivaReal).
- [x] AC4: Post-filtering server-side para filtros nao suportados nativamente pelo Actor Apify.
- [x] AC5: Rate limit 5 buscas/hora por consultant. Custo max $2/busca. Retorna 429 se excedido.
- [x] AC6: `GET /api/search/parametric/[searchId]` retorna status, metricas e resultados quando completed.
- [x] AC7: Evento intelligence_feed tipo='busca_parametrica' ao completar.
- [x] AC8: Graceful degradation: portal falha -> continua com outros. Todos falham -> status 'failed'.
