**Epic:** 9 — ACM: Paridade de Entrega Premium + Suporte a Apartamento
**Camada:** Renderização
**Status:** Done
**Priority:** Should (elevada após spike 9.0 — destravada e de entrega visível rápida)

> **Destravada pelo spike 9.0** (`docs/acm/9.0-data-audit.md`): a RPC viva já retorna `latitude`/`longitude` por comparável (1000/1000 não-nulo). O `staticMap.ts` (8.3a) **já suporta markers** (`MapMarker` com `label`, `color`, `size`) e o polígono do raio. Logo esta story é **camada de render/wiring**, não exige mudança de dado nem de RPC.

## User Story

Como Luciana (consultora),
Quero que o mapa do laudo/resumo mostre o **raio de análise + o alvo + os comparáveis** com o **Top 5 numerado** (como no laudo de referência),
Para dar credibilidade visual e rastreabilidade geográfica ao documento — hoje o mapa só mostra o alvo + o raio.

## Contexto

Comparação 1:1 (sessão 2026-06-17): o laudo de referência traz mapa com **raio de 1.000 m + alvo + Top 5 numerados (❶–❺)** e legenda; o gerador interno plota **só o alvo + raio** porque os pins de comparáveis foram omitidos por (suposta) ausência de lat/lng. O spike 9.0 confirmou que a RPC já entrega lat/lng. Falta montar o array de `markers` (com cor por faixa e rótulo numérico) a partir do ranking de aderência e passá-lo a `buildStaticMapUrl`.

## Acceptance Criteria

- [x] AC1: O mapa do **Laudo** (e do **Resumo**, mesmo gerador) plota: **alvo** (vermelho `COLORS.vermelho`, `size:'l'`), **Top 3** (dourado), **Top 4–5** (laranja), **demais comparáveis no raio** (azul), além do **polígono do raio**. → Laudo/Deck/Pacote já via 8.7; **Resumo wired nesta story**.
- [x] AC2: **Rótulos numerados 1–5** nos pins do Top 5 (via `MapMarker.label`), na ordem do ranking de aderência (`computation.ranking`). → `buildAcmMapMarkers` (label = rank+1).
- [x] AC3: A **legenda** do mapa no PDF reflete as faixas. → presente em `LaudoDocument`/`DeckDocument` (via model) e `ResumoDocument` (linha "● Imóvel-alvo ❶ Top 3 ❹ Top 4–5 ● Demais vendidos ■ Raio").
- [x] AC4: **Coords** = `latitude`/`longitude` da RPC por comparável; comparáveis sem lat/lng **omitidos** (não quebram); "demais" limitados por `maxOutros` (default **22**) p/ não estourar a URL da Static API.
- [x] AC5: **Degradação graciosa:** sem token → `buildStaticMapUrl` retorna `null`; sem comparáveis georreferenciados → só alvo + raio. Nunca quebra o PDF (ADR-EPIC8-001 / 8.3a AC8 preservado).
- [x] AC6: **Sem regressão:** 124/124 testes ACM verdes; 0 erros tsc em `src/`; 0 lint. Números/seções inalterados.
- [x] AC7: **Testes:** `buildAcmMapMarkers` (puro) cobre cor/label/ordem por faixa, omissão de NULL, teto e **n<5** (caso novo); "sem token" coberto em `staticMap.test.ts`.
- [x] AC8: `propertyType` (9.1) **não bloqueia** — o mapa é idêntico para apto e casa.

## Technical Notes

- `staticMap.ts` **não precisa mudar** (já aceita `markers: MapMarker[]` com `label`/`color`/`size`). O trabalho é no **view-model/sheet** que chama `buildStaticMapUrl` (hoje passa só `center`+`radius`+alvo).
- Atenção ao **comprimento da URL** da Static Images API: limitar o nº de pins "demais" (ex.: Top 10–15 mais próximos) e usar `circleSteps` enxuto. Documentar o teto escolhido (AC4).
- Cores já em `theme.ts` (`vermelho`, `dourado`, `azul`); Top 4–5 laranja — usar tom existente ou adicionar constante se necessário.
- Mapbox usa rótulo `pin-s-{label}` (0-9, a-z) — números 1–5 OK.
