# Story 2.7: Lógica de Expansão por Raio (80%)

**Epic:** 2 — Leads, Funil de Vendas & Metodologia RE/MAX
**Status:** InProgress
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero que o sistema calcule automaticamente minha cobertura territorial e sugira expansão quando atingir 80%,
Para que eu domine progressivamente minha micro-região sem pular etapas.

## Acceptance Criteria

- [x] AC1: Cobertura calculada automaticamente por raio. Fórmula: `cobertura = (edifícios com status != 'nao_visitado') / (total de edifícios no raio) * 100`. Query PostGIS para total no raio: `SELECT COUNT(*) FROM edificios e JOIN edificios_qualificacoes q ON q.edificio_id = e.id AND q.consultant_id = auth.uid() WHERE ST_DWithin(e.coordenadas::geography, ST_MakePoint($lng, $lat)::geography, $raio_metros)`. Edifícios "cobertos": WHERE `q.status != 'nao_visitado'`. Cálculo executado: (a) ao abrir o mapa, (b) ao mudar status de edifício, (c) via pull-to-refresh. Percentual exibido no header do mapa: "[Bairro] [%]%" (seção 2.2 do UX spec).
- [x] AC2: Notificação ao atingir 80% de cobertura no raio ativo. Quando cobertura >= 80% pela primeira vez, push notification: "Parabéns! Você cobriu 80% do raio [500m/1km/2km]. Hora de expandir!" + toast in-app com animação celebratória (confetti ou badge dourado). No Feed de Inteligência (Story 3.7): evento tipo "Raio 80% atingido", cor azul #003DA5, ações: "Expandir" (ativa próximo raio) e "Ver no Mapa". Threshold de 80% configurável (padrão: 80).
- [x] AC3: Próximo raio desbloqueado com seed data. Ao atingir 80% no raio 500m, raio 1km é desbloqueado. Ao atingir 80% no raio 1km, raio 2km é desbloqueado. Desbloqueio = (a) círculo do próximo raio se torna interativo no mapa (não mais translúcido/dimmed), (b) seed data carregado automaticamente para o novo raio (reutiliza lógica da Story 1.7 — API pública + Overpass para edifícios no raio expandido). Seed data entra como "Não Visitado" (cinza) com badge "auto". Botão manual "Expandir agora" no modal de 80% para trigger imediato. Se seed já executado anteriormente para aquele raio, não duplicar.
- [x] AC4: Sugestão de próximo bloco/rua. Ao desbloquear novo raio, sistema sugere a rua/bloco com maior potencial dentro do novo raio. Critérios de sugestão (em ordem de prioridade): (1) ruas com FISBOs não abordados, (2) ruas com mais edifícios pré-carregados (seed), (3) ruas com edifícios de alto padrão. Query: `SELECT e.endereco, COUNT(*) AS total, COUNT(*) FILTER (WHERE EXISTS(SELECT 1 FROM leads l WHERE l.edificio_id = e.id AND l.is_fisbo = true)) AS fisbos FROM edificios e WHERE ST_DWithin(e.coordenadas::geography, epicentro, novo_raio) AND NOT ST_DWithin(e.coordenadas::geography, epicentro, raio_anterior) GROUP BY e.endereco ORDER BY fisbos DESC, total DESC LIMIT 3`. Exibido como card: "Sugestão: R. [Nome] — [N] edifícios, [M] FISBOs" com botão "Ver no mapa" (centra o mapa na rua sugerida).
- [x] AC5: Dashboard de progresso por raio. Na tela Dashboard (Story 2.9), seção "Território" com cards por raio:
  - Raio 500m: barra de progresso [====>    ] 73%, cor verde (#22C55E) se >80%, amarelo se 50-80%, vermelho se <50%. "[N] edifícios / [M] total". Status: "Ativo" ou "Completo" (>=80%).
  - Raio 1km: mesma barra. Status: "Desbloqueado" ou "Bloqueado" (ícone cadeado cinza se raio anterior <80%).
  - Raio 2km: mesma barra.
  Cards KPI 165x80px, número principal 24px bold (percentual), label 12px. Tap em card: expande com lista de ruas e contagem por rua.
- [x] AC6: Raio ativo exibido no mapa com destaque. O raio em que a consultora está trabalhando atualmente é o que tem cobertura <80% mais próxima do epicentro. Círculo do raio ativo: borda 3px sólida, fill 8% opacity. Raios completos (>=80%): borda 1px tracejada, fill 3% opacity, badge "check" verde no centro. Raios bloqueados: borda 1px pontilhada cinza, ícone cadeado. Cores mantidas: 500m verde #22C55E, 1km amarelo #EAB308, 2km vermelho #DC1431.
- [x] AC7: Persistência do estado de raios. Estado de desbloqueio salvo em configuração do consultor (tabela `consultores` ou `configuracoes_consultor` — novo campo `raios_desbloqueados JSONB DEFAULT '{"500m": true, "1km": false, "2km": false}'`). Ao reabrir o app, estado restaurado. Seed data para novos raios marcado com flag para evitar re-execução.

## Technical Notes

- **Database:** Queries PostGIS com `ST_DWithin` para contagem de edifícios por raio. Novo campo em `consultores` ou tabela auxiliar para persistir estado de raios desbloqueados (JSONB). Seed data para novos raios usa mesma lógica da Story 1.7 (API pública → INSERT em `edificios` + `edificios_qualificacoes` com status 'nao_visitado').
- **PostGIS:** `ST_DWithin(e.coordenadas::geography, ST_MakePoint(lng, lat)::geography, raio_metros)` para filtro por raio. Performance: index GiST em `edificios.coordenadas` (criado na migration 001).
- **API:** Supabase client SDK + Edge Function para cálculo de cobertura (ou client-side com query agregada). TanStack Query key: `['cobertura', raio, epicentro]`. Cache com staleTime de 5min (cobertura não muda frequentemente).
- **Components:** CoverageBar (barra de progresso por raio), RadiusOverlay (círculos no mapa com estados), ExpansionModal (modal de 80% + seed + sugestão), NextBlockSuggestion (card de sugestão), RadiusDashboard (cards no dashboard).
- **Seed Data:** Reutiliza lógica da Story 1.7 (Google Places / OSM Overpass) com raio expandido. Verifica duplicatas via PostGIS `ST_DWithin(novo_edificio, existente, 50m)`.

## Notes

- **FR-004:** Expansão progressiva — ao atingir 80% de cobertura no raio atual, desbloqueia o próximo.
- **FR-005:** Sugestão de próxima rua/bloco com base em similaridade de m², perfil e FISBOs não abordados.
- O threshold de 80% é baseado na metodologia RE/MAX de domínio territorial — 80% garante que a consultora conhece a grande maioria dos edifícios antes de expandir.
- Raios são 500m, 1km, 2km a partir do epicentro (Rua Alvorada, Moema, CEP 04550-000).
- Seed data para novos raios é executado APENAS no desbloqueio — não duplica edifícios já existentes (verificação por proximidade PostGIS 50m).
