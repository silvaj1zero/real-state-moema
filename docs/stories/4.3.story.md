# Story 4.3: Progressão nos Clubes RE/MAX

**Epic:** 4 — Parcerias Ganha/Ganha & Escala
**Status:** Draft
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero visualizar meu progresso nos clubes de reconhecimento RE/MAX com barra de progresso e projeção,
Para que eu saiba onde estou, quanto falta para o próximo clube e me motive a atingir metas anuais.

## Acceptance Criteria

- [ ] AC1: Tela "Clubes RE/MAX" acessível via menu lateral → ícone troféu (#003DA5) ou via card resumo no dashboard principal. Barra de progresso visual mostrando a escala completa de clubes: Executive → 100% → Platinum → Chairman's → Titan → Diamond → Pinnacle. Clube atual destacado (preenchido), próximos em outline. Barra proporcional ao VGV acumulado vs threshold do próximo clube. Cor da barra: gradiente do azul #003DA5 ao dourado #FFD700.
- [ ] AC2: Thresholds dos clubes lidos da tabela `clubes_remax_thresholds` (seed data migration 004): Executive R$ 167.224, 100% R$ 334.448, Platinum R$ 836.120, Chairman's R$ 1.672.241, Titan R$ 2.508.361, Diamond R$ 3.344.482, Pinnacle R$ 6.688.963. Clube atual determinado por `consultant_settings.vgv_acumulado` comparado com thresholds. Se VGV < Executive → "Sem clube" com distância para Executive.
- [ ] AC3: Card informativo abaixo da barra com: "Clube atual: [nome]" (badge com cor do clube), "VGV acumulado: R$ [valor]" (formatado BR), "Próximo clube: [nome] — faltam R$ [diferença]" (destaque verde se < 20% restante, amarelo se 20-50%, padrão se > 50%). Percentual de progresso entre clube atual e próximo: `(vgv_acumulado - threshold_atual) / (threshold_proximo - threshold_atual) * 100`.
- [ ] AC4: Histórico mensal de VGV em gráfico de linha (últimos 12 meses). Eixo X: meses. Eixo Y: VGV acumulado. Linhas horizontais pontilhadas nos thresholds de cada clube (com label). Dados derivados de `comissoes` agrupados por `date_trunc('month', data_recebimento)` com soma cumulativa. Componente: VGVHistoryChart (Recharts/Chart.js).
- [ ] AC5: Projeção para próximo clube (classificação **Should**): estimativa baseada na média mensal dos últimos 3 meses. "No ritmo atual, você atinge [próximo clube] em [N meses]". Exibida em card secundário com ícone crystal ball. **Caveat obrigatório** (princípio PV): texto "Projeção estimada — sazonalidade e variação de mercado podem alterar o resultado" em fonte 12px, cor #757575, abaixo da projeção. Não exibir se menos de 3 meses de dados.
- [ ] AC6: Meta pessoal configurável: campo "Minha meta VGV anual" no topo da tela (input numérico R$, salvo em `consultant_settings.meta_vgv_anual`). Se definida, exibe segunda barra de progresso menor com `vgv_acumulado / meta_vgv_anual * 100`. Cores: verde se ritmo adequado (projeção >= meta), amarelo se apertado, vermelho se improvável. Botão "Editar meta" (ícone lápis).
- [ ] AC7: UPDATE em `consultant_settings.clube_remax_atual` sempre que VGV cruza um threshold (trigger aplicativo — não DB). Ao subir de clube, celebration modal: confete animado (canvas), nome do novo clube em dourado, botão "Compartilhar" (gera imagem para WhatsApp/Instagram com badge RE/MAX). Ao carregar a tela, verificar se clube armazenado difere do calculado e atualizar.
- [ ] AC8: Card resumo compacto reutilizável para dashboard principal (Story 2.9): "Clube: [atual] | VGV: R$ [valor] | Faltam R$ [diff] para [próximo]". 1 linha, 120px altura, tap abre tela completa de clubes. Query key: `['consultant_settings', consultant_id]` + `['clubes_thresholds']`.

## Technical Notes

- **Database:** Leitura de `consultant_settings` (colunas `clube_remax_atual`, `vgv_acumulado`, `meta_vgv_anual` — schema.sql). Leitura de `clubes_remax_thresholds` (seed data migration 004, 7 registros). UPDATE em `consultant_settings` para meta e clube atual. ALTER TABLE em `consultant_settings` para adicionar colunas `clube_remax_atual` (enum `clube_remax`), `vgv_acumulado` (NUMERIC(15,2) DEFAULT 0), `meta_vgv_anual` (NUMERIC(15,2) nullable), `percentual_relacionamento_default` (NUMERIC(5,2) DEFAULT 3.5) — se não existirem na migration 001.
- **Enum:** `clube_remax` (sem_clube, executive, cem_porcento, platinum, chairmans, titan, diamond, pinnacle) — criado em migration 004.
- **API:** Supabase client SDK. TanStack Query: `['clubes_thresholds']` (cache infinito — dados estáticos), `['consultant_settings', consultantId]`, `['comissoes_mensal', consultantId]` (para gráfico).
- **Cálculos:** Toda lógica de determinação de clube e projeção no client (funções puras testáveis). Projeção: `meses_restantes = (threshold_proximo - vgv_acumulado) / media_mensal_3m`.
- **Components:** ClubeProgressBar, ClubeInfoCard, VGVHistoryChart, ProjecaoCard, MetaEditor, ClubeResumoCompact, CelebrationModal.

## Notes

- **Princípio PV — Projeção como Should:** Projeções são estimativas, não promessas. O caveat de sazonalidade é obrigatório e deve ser visível. Mercado imobiliário tem ciclos — projeção linear é simplificação assumida.
- **Princípio PV — Meta pessoal:** A meta é do consultor, não da franquia. O sistema não impõe metas — apenas permite que a consultora defina a sua e acompanhe.
- Thresholds dos clubes são seed data na tabela `clubes_remax_thresholds` — se RE/MAX alterar valores, basta UPDATE na tabela (sem deploy).
- Celebration modal é detail de UX que aumenta engagement. Compartilhamento gera marketing orgânico.
