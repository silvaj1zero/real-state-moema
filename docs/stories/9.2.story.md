**Camada:** Renderização
**Status:** Done

**Executor sugerido:** @dev

> **Lacuna #1 da comparação:** referência = `ACM-Honduras629-CANONICO-validado.xlsx` com **7 abas** (Leia-me · Top 3 · Top 5 · Top 10 · Todos · Ofertas ativas · Terrenos), 21 colunas, fills por faixa, fonte colorida, hyperlinks, freeze panes, auto-filter, sínteses com merge. Interno = **só um CSV plano** (`buildComparaveisCsv`, 19 colunas). Esta story fecha essa distância **em TS (sem Python)**.

## User Story

Como Luciana (consultora),
Quero exportar do app uma **planilha XLSX canônica** com as mesmas abas, cores, links e sínteses da planilha de referência,
Para entregar/editar um material que o cliente abre no Excel — não um CSV cru.

## Contexto

A canônica de referência é o entregável que a corretora efetivamente manipula. O CSV atual não tem abas, cor, hyperlink nem as visões **Ofertas ativas** e **Terrenos**. A geração deve ser nativa TS (Opção C) usando uma lib JS de XLSX (`exceljs` recomendado por suporte a estilos/hyperlinks/freeze/auto-filter; `xlsx`/SheetJS já está no projeto para leitura mas tem estilo limitado — avaliar na implementação). Os números vêm de `AcmLaudoComputation` (8.2) + comparáveis da RPC; nada de recálculo na planilha.

## Acceptance Criteria

- [x] AC1: **7 abas** geradas: `Leia-me`, `Top 3`, `Top 5`, `Top 10`, `Todos`, `Ofertas ativas`, `Terrenos` — ordem canônica (teste trava a sequência).
- [x] AC2: **Colunas das abas de ranking** (18): Rank, SQL (ITBI), Endereço, Vendido?, Ano ref., Dorm, Suíte, Vagas, Área constr., Área terreno, Valor venda ITBI, Valor anúncio/pedido, R$/m² constr., R$/m² terreno, Distância, Status anúncio, Link anúncio, Aderência. Células sem dado ficam **vazias** (helper `blank`, nunca "nan"). *Nota: `Bairro`, `Programa:origem` e `Re-verif./Confere` da referência foram omitidos por não existirem no dado da RPC (entram quando 9.4/9.5 popularem).* 
- [x] AC3: **Formatação:** `fill` por faixa (Top 3 ouro / Top 5 laranja / demais gelo via `tierHighlight`), cabeçalho navy + fonte branca, **hyperlink** na coluna Link anúncio, `freeze` do cabeçalho (`views: frozen ySplit 1`), `autoFilter` na linha de cabeçalho.
- [x] AC4: **Aba `Leia-me`** com nota metodológica (ordenação 50/20/30, convenção dorm/suíte, status web, fonte) + **aviso de dado** (campos da metodologia dependem do sink 9.4).
- [x] AC5: **Aba `Ofertas ativas`** (endereço/área/pedido/R$/m²/uso/link) e **aba `Terrenos`** (input do consultor) — ambas com `emptyNote` "Nenhum registro no raio" quando sem dado. **`propertyType='apartamento'` omite a aba `Terrenos`** (teste cobre).
- [x] AC6: **Item no `AcmExportMenu`**: "Gerar Planilha (XLSX)" (`PlanilhaExportSheet`, loading + erro), gera `Blob` cliente, filename `acm-planilha-YYYY-MM-DD.xlsx`. CSV mantido.
- [x] AC7: **Degradação graciosa:** com campos 100% NULL (estado atual, spike 9.0) a planilha gera com colunas vazias; abas sem dado trazem "Nenhum registro no raio". Teste de comparáveis vazios passa.
- [x] AC8: **Testes:** `buildPlanilhaModel` (puro) + smoke `buildPlanilhaWorkbook` (assinatura ZIP "PK"), incluindo NULL e vazio. 7 testes novos; 132/132 ACM no total.
