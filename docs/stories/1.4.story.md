# Story 1.4: Qualificação e Card do Edifício

**Epic:** 1 — Foundation, Mapa & Registro de Campo
**Status:** Ready
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero tocar em um edifício no mapa e ver seus detalhes completos,
Para que eu tenha informações relevantes sobre cada condomínio e possa qualificá-lo em campo.

## Acceptance Criteria

- [ ] AC1: Tap em pin no mapa abre Bottom Sheet (BuildingCard) conforme frontend-spec seção 2.3. Três snap points: Peek (30%, 244px), Half (60%, ~487px), Full (95%). Handle de arraste (barra cinza 40x4px, área de toque 40px). Swipe up expande, swipe down contrai, tap fora fecha.
- [ ] AC2: Estado Peek exibe: nome do edifício (18px bold, truncado com ellipsis), badge FISBO (estrela vermelha) se aplicável, endereço (14px, cinza #6B7280), chip de status com cor (Cinza/Azul/Amarelo/Verde), data de cadastro, quick info chips (padrão, tipologia, nº unidades), botões de ação rápida (Editar, + Lead, FISBO).
- [ ] AC3: Estado Half exibe adicionalmente: dropdown "Abertura a Corretores" editável (enum `abertura_corretores`: 'zelador_amigavel', 'rigido', 'exige_autorizacao', 'desconhecido'), contador de oportunidades (campo `oportunidades_count` da `edificios_qualificacoes`), seção Leads com lista compacta (máx 3 visíveis, "Ver todos (N)" se mais), seção Informantes com lista compacta.
- [ ] AC4: Estado Full exibe adicionalmente: seção Notas (campo `notas` da `edificios_qualificacoes`, editável com botão "+ Adicionar nota"), seção Histórico (timeline de eventos — cadastro, mudanças de status, leads adicionados), seção Localização (mini mapa com pin, coordenadas, zona de raio), botões "Ver no mapa" e "Compartilhar".
- [ ] AC5: Campo "Abertura a Corretores" editável inline via dropdown. UPDATE na `edificios_qualificacoes` (campo `abertura_corretores`). Valores: Zelador amigável, Rígido, Exige autorização, Desconhecido. Salvamento automático ao selecionar (optimistic update).
- [ ] AC6: Contador de oportunidades editável (increment/decrement ou input numérico). Diferencia "placas" vs "anúncios" visualmente. UPDATE em `edificios_qualificacoes.oportunidades_count`. Exibe badge se `is_fisbo_detected = true`.
- [ ] AC7: Botão "Editar" (outlined azul, 36px) abre form de edição inline dentro do bottom sheet — permite alterar nome, endereço, tipologia, padrão, notas. UPDATE em `edificios` (dados base) e `edificios_qualificacoes` (dados privados). Salvamento com feedback visual.
- [ ] AC8: Notas livres — campo texto multilinha na `edificios_qualificacoes.notas`. Editável no estado Full do bottom sheet. Auto-save com debounce de 1s (optimistic update).
- [ ] AC9: Bottom sheet com animação suave (spring animation) em mobile. Swipe velocity detection para decidir snap point. Backdrop semi-transparente que escurece conforme expande.

## Technical Notes

- **Database:** SELECT com JOIN: `edificios` LEFT JOIN `edificios_qualificacoes` ON edificio_id AND consultant_id. UPDATE em `edificios_qualificacoes` para: abertura_corretores, oportunidades_count, notas, tipologia, padrao, status_varredura. UPDATE em `edificios` para: nome, endereco. Futuramente, seções Leads e Informantes virão dos JOINs com tabelas `leads` e `informantes_edificios` (Epic 2).
- **API:** Supabase client SDK com TanStack Query para cache e optimistic updates. Query key: `['building', edificio_id]`. Mutation com onMutate para optimistic update e onError para rollback.
- **UI Components:** BuildingCard/BottomSheet (3 snap points), BuildingHeader (nome, status, data), QuickInfoChips (padrão, tipologia), ActionButtons (Editar, +Lead, FISBO), AberturaCoorretoresDropdown, OpportunityCounter, NotesField (auto-save), LeadsList (compacta), InformantsList (compacta), MiniMap, Timeline.
- **PostGIS:** Nenhuma query nova. Coordenadas exibidas no card vêm do SELECT já feito.
- **Offline:** Dados em cache no TanStack Query + IndexedDB (Story 1.6). Edições offline via mutation queue (Story 1.6).

## Notes

- O Bottom Sheet é um dos componentes mais complexos do app. Considerar usar uma lib como `react-spring-bottom-sheet` ou implementar com Framer Motion para animações suaves e snap points.
- Seções Leads e Informantes estarão vazias no Epic 1 (serão populadas no Epic 2). Implementar empty states: "Nenhum lead cadastrado" com botão "+ Lead" (desabilitado ou stub).
- **Optimistic updates são essenciais** para a experiência de campo — a consultora não pode esperar confirmação do servidor para cada edição. TanStack Query fornece esse padrão nativamente.
- O campo `is_fisbo_detected` na `edificios_qualificacoes` será toggleável via botão "FISBO" (toggle vermelho quando ativo). Prepara para Epic 2 (Story 2.10 — Marcação Manual de FISBOs).
