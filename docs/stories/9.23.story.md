# Story 9.23 — UI in-app: ativar guard-rails, homogeneização e faixa H-3 nos export sheets ACM

**Epic:** 9 — ACM Paridade Premium
**Status:** Ready
**Priority:** Must (Classe S — a UI emite laudo no modelo pré-v5: sem guard-rail, sem deflação, valor-ponto)
**Executor sugerido:** @dev
**Estimativa:** M (1–2 dias)
**Origem:** gap declarado desde H-2 (`docs/acm/ROADMAP-ACM.md` H-2 "UI in-app ainda não expõe homogeneização/guard-rails" + §9b Wave 4) · decisões H-3 (`docs/acm/DECISOES-H3-LUCIANA-20260710.md`)

## Story

Como consultora, quero que o laudo/resumo/deck/didático gerado **pelo app** (export sheets) use os mesmos mecanismos v5 dos laudos offline — guard-rail anti-auto-referência, homogeneização FipeZap, deságio H-3 por estado A–F e headline em faixa — para que o PDF que eu gero em campo tenha a mesma defensabilidade do que o founder gera offline.

## Contexto (problema exato)

Cinco componentes chamam `computeLaudo` no client e TODOS passam o target mínimo, deixando os mecanismos v5 inertes:

| Componente | Chamada | O que falta |
|---|---|---|
| `app/src/components/acm/LaudoExportSheet.tsx` | linha ~122 | `target: { areaConstruida, areaTerreno }` apenas |
| `app/src/components/acm/PacoteExportSheet.tsx` | ~108 | idem |
| `app/src/components/acm/ResumoExportSheet.tsx` | ~107 | idem |
| `app/src/components/acm/EntregavelExportSheet.tsx` | ~118 | idem |
| `app/src/components/acm/PlanilhaExportSheet.tsx` | ~56 | idem |

O irônico: **o formulário já coleta** `endereco`, `vagas`, `pretendido`, `pedidoReal` (ver `LaudoExportSheet.tsx` ~148–167, usados só no `LaudoInput` de exibição) — os dados existem na tela e não chegam ao cálculo. Consequências concretas:

1. Guard-rail 9.8 inerte (sem `target.endereco/vagas/precoPretendido`) — o incidente Honduras 639 pode se repetir no in-app.
2. Sem `homogeneizacao` — comparáveis de 2023 entram a valor nominal (offline deflaciona via FipeZap, Stories 9.11/9.12).
3. Sem `estadoConservacao` — deságio H-3 (régua A–F, Story 9.14 + `ESTADO_DESAGIO_H3`) nunca aplica.
4. Sem `propertyType` — gate R5 (Story 9.17) inerte.
5. `avisos[]` (Story 9.15) e `autoReferenciasExcluidas` são computados mas **nunca mostrados** ao usuário antes do download.

## Acceptance Criteria

1. **AC1 — Guard-rail ativo:** os 5 sheets passam `target.endereco` (campo já coletado), `vagas` e `precoPretendido` ao `computeLaudo`. Quando houver `autoReferenciasExcluidas`, a sheet exibe contagem + endereços + motivos ANTES do botão de download (não bloqueia — informa).
2. **AC2 — Homogeneização por padrão:** `homogeneizacao` ligada com a série vendada (`FIPEZAP_SP_VENDA_RESIDENCIAL` / `FIPEZAP_SP_ULTIMA_COMPETENCIA` de `@/lib/acm/data/fipezapSpVendaResidencial`), exibindo a competência de referência na sheet; toggle explícito para desligar (default ON). Comparáveis sem `dataVenda` seguem o comportamento da 9.11 (sem ajuste, contados em `semAjuste`).
3. **AC3 — Ficha de estado A–F (H-3):** select opcional "Estado do imóvel (régua A–F)" no formulário → `target.estadoConservacao`; ausente → comportamento 9.14 (aviso `target_condition_unconfirmed`, faixa conservadora). Labels da régua conforme `DECISOES-H3-LUCIANA-20260710.md` (A 0% · B −5% · C −7,5% · D −10% · E −15% · F fora da régua).
4. **AC4 — Tipologia do alvo (R5 opt-in):** select opcional casa/apartamento → `propertyType`; documentar na sheet que em PROD a classificação dos comparáveis é parcial até a Story 9.4 (o gate 9.17 já emite `typology_r5_incomplete` nesse caso — só exibir).
5. **AC5 — Transparência pré-download:** a sheet exibe, após o cálculo e antes do download: headline em faixa no formato H-3 ("Mercado R$ X–Y (referência Z)") e a lista de `avisos[]` (ícone por severidade). Aplica-se no mínimo a Laudo, Resumo e Pacote.
6. **AC6 — Zero recálculo e consistência:** nenhum número novo inventado na UI — tudo vem do `AcmLaudoComputation` (ADR-EPIC8-001). Snapshot dos modelos inalterado quando os novos campos não são preenchidos (backward-compat: formulário vazio ≡ comportamento atual).
7. **AC7 — Testes:** testes de componente para (a) endereço preenchido → auto-referência sintética aparece na UI; (b) toggle homogeneização OFF → `homogeneizacao.aplicada === false`; (c) estado C selecionado → deságio −7,5% no computation; (d) formulário mínimo → paridade com snapshot atual. Suite `vitest run src/lib/acm src/components/acm` verde.
