# Story 2.8: Diagnóstico de Gargalos do Funil

**Epic:** 2 — Leads, Funil de Vendas & Metodologia RE/MAX
**Status:** Ready
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero visualizar a taxa de conversão entre cada etapa do funil e receber diagnóstico automático com ações corretivas,
Para que eu identifique onde estou perdendo leads e saiba exatamente o que fazer para melhorar.

## Acceptance Criteria

- [ ] AC1: Funil visual com taxas de conversão entre etapas. Tela acessível via Dashboard (Story 2.9) → tap na seção "Funil" ou via tab "Mais" → "Diagnóstico". Funil desenhado como barras horizontais empilhadas com formato de funil (largura proporcional à quantidade):
  - Contato [============================] 23 (100%)
  - V1     [====================]          12 (52%)
  - V2     [============]                   5 (42%)
  - Excl   [========]                       3 (60%)
  - Venda  [====]                           1 (33%)
  Cores: Contato cinza #6B7280, V1 azul #003DA5, V2 azul escuro #001D4A, Exclusividade dourado #D97706, Venda verde #22C55E. Entre cada barra: seta com taxa de conversão (font 14px bold). Tap em barra: navega para tab do funil correspondente (Story 2.2). Query: `SELECT etapa_funil, COUNT(*) FROM leads WHERE consultant_id = auth.uid() GROUP BY etapa_funil`.
- [ ] AC2: Conversão calculada entre etapas adjacentes. Fórmulas: Contato→V1 = (leads que passaram por V1) / (total que passaram por Contato), V1→V2, V2→Excl, Excl→Venda. Query via `funnel_transitions`: `SELECT from_etapa, to_etapa, COUNT(*) FROM funnel_transitions WHERE consultant_id = auth.uid() AND is_retrocesso = false AND created_at BETWEEN $start AND $end GROUP BY from_etapa, to_etapa`. Exibição: entre cada barra do funil, percentual com seta (verde se >50%, amarelo se 30-50%, vermelho se <30%).
- [ ] AC3: Matriz de diagnóstico RE/MAX automática. Para cada transição com conversão abaixo do threshold (default 40%), sistema exibe diagnóstico baseado na metodologia RE/MAX:
  | Transição | Diagnóstico | Ação Sugerida |
  |-----------|-------------|---------------|
  | Contato→V1 baixa | "Problema de abordagem ou script" | "Revise seus scripts de primeiro contato" + link para Biblioteca de Scripts filtrado por `etapa_funil = 'contato'` (Story 2.5) |
  | V1→V2 baixa | "Problema de rapport ou qualificação na V1" | "Foque nos campos V1 estruturados: motivação real e prazo" + link para lista de leads em V1 com campos V1 incompletos |
  | V2→Excl baixa | "Problema de fechamento ou objeção de exclusividade" | "Revise scripts de exclusividade" + link para Scripts filtrado por `categoria = 'objecao_exclusividade'` |
  | Excl→Venda baixa | "Pipeline lento ou mercado frio" | "Verifique leads em Exclusividade há mais de 30 dias" + link para funil filtrado |
  Card de diagnóstico: fundo amarelo claro #FFFBEB, ícone alerta, título (16px bold), descrição (14px), botão de ação (outlined azul #003DA5).
- [ ] AC4: Contagem de retrocessos. Seção dedicada "Retrocessos" no diagnóstico. Query: `SELECT from_etapa, to_etapa, COUNT(*) AS total, STRING_AGG(justificativa, ' | ') FROM funnel_transitions WHERE consultant_id = auth.uid() AND is_retrocesso = true AND created_at BETWEEN $start AND $end GROUP BY from_etapa, to_etapa ORDER BY total DESC`. Exibição: tabela/lista com transição (ex: "V2 → V1: 3 retrocessos"), justificativas mais comuns, indicador de impacto (vermelho se >2 retrocessos no período). Retrocessos contabilizados como "perda" na taxa de conversão, não como "avanço reverso".
- [ ] AC5: Filtro de período. Dropdown no header: "Hoje", "7 dias", "30 dias", "90 dias", "Personalizado" (date range picker). Default: 30 dias. Todas as métricas (conversão, retrocessos, diagnóstico) recalculadas ao mudar período. Query com `WHERE created_at BETWEEN $start AND $end`. Indicador visual do período selecionado no topo.
- [ ] AC6: Ações corretivas com links diretos. Cada diagnóstico inclui botão de ação que navega diretamente para o recurso relevante: "Ver scripts" → Biblioteca de Scripts (Story 2.5) com filtro, "Ver leads paradas" → Funil (Story 2.2) com filtro por etapa e tempo, "Ver informantes" → Lista de informantes com leads pendentes. Botão: filled azul #003DA5, 36px altura, ícone seta. Deep links preservam contexto (filtros pré-aplicados).
- [ ] AC7: Resumo executivo no topo. Card destaque com: "Seu funil converte [X]% do Contato à Venda" (font 20px bold, cor por resultado: verde >10%, amarelo 5-10%, vermelho <5%), "Principal gargalo: [Transição] ([%]%)" (font 16px), "Retrocessos no período: [N]" (font 14px, vermelho se >0). Background gradiente azul/branco, 120px altura.

## Technical Notes

- **Database:** SELECT agregado em `leads` (contagem por etapa_funil) e `funnel_transitions` (contagem por from_etapa/to_etapa, filtro por is_retrocesso e período). Nenhuma tabela nova — usa dados existentes de Story 2.1 (leads) e Story 2.2 (funnel_transitions).
- **API:** Supabase client SDK. Queries agregadas podem ser pesadas — considerar Supabase Edge Function ou view materializada para performance. TanStack Query key: `['diagnostico', consultant_id, periodo]`. StaleTime: 5min.
- **Components:** FunnelDiagnostic (tela principal), FunnelVisual (barras de funil com conversão), DiagnosticCard (card de diagnóstico com ação), RetrocessoSection (contagem e detalhes de retrocessos), PeriodFilter (dropdown de período), DiagnosticSummary (card resumo executivo).
- **Diagnóstico Rules Engine:** Lógica client-side simples (if-else por transição e threshold). Não requer ML. Threshold de 40% configurável. Rules: `if (conversao < threshold) → show diagnostic card with action`.

## Dependencies

- Depends on: 2.1 (leads), 2.2 (funnel_transitions com retrocessos), 2.5 (scripts para links de ação), 1.1 (infra, DB)
- Blocks: 2.9 (dashboard exibe resumo do diagnóstico)

- Flow: Dashboard → tap Funil → Diagnóstico completo → Cards com ações → Deep links para Scripts/Funil/Informantes
- Components: barras horizontais empilhadas (funnel shape), cards de diagnóstico com ações

## Notes

- **FR-012:** Diagnóstico de gargalos do funil com ação corretiva sugerida.
- **FR-033:** Matriz de diagnóstico de gargalos automática com ação sugerida.
- **Princípio PV — Retrocesso contabilizado no diagnóstico:** Guardrail 3 da Story 2.2. Retrocessos são visíveis no diagnóstico, não escondidos. Isso incentiva a consultora a tratar a causa raiz (objeção não resolvida) em vez de apenas mover leads para frente.
- A matriz de diagnóstico RE/MAX é baseada na experiência prática: Contato→V1 falha = problema de script/abordagem, V1→V2 falha = falta de rapport, V2→Excl = objeção de exclusividade não contornada.
- Thresholds de conversão são heurísticos iniciais (40% para alerta). Podem ser ajustados com base nos dados reais da Luciana ao longo do tempo.
