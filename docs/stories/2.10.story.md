# Story 2.10: Marcação Manual de FISBOs

**Epic:** 2 — Leads, Funil de Vendas & Metodologia RE/MAX
**Status:** Done
**Priority:** Must
**Estimated Complexity:** S

## User Story

Como consultora RE/MAX,
Quero marcar manualmente edifícios onde detectei proprietários vendendo por conta própria (FISBO),
Para que eu priorize prospecção nesses imóveis e visualize oportunidades no mapa.

## Acceptance Criteria

- [x] AC1: Botão "FISBO" no card do edifício (Story 1.4, estados Half/Full). Botão toggle: outlined vermelho #DC1431 quando inativo, filled vermelho #DC1431 com texto branco quando ativo. Ícone estrela. 36px altura. Ao ativar: UPDATE em `leads` (se lead existente) SET `is_fisbo = true`, ou cria flag no edifício via campo futuro. Toggle com animação scale (1.0→1.2→1.0, 200ms). Haptic feedback ao marcar.
- [x] AC2: Badge FISBO no pin do mapa. Quando edifício tem pelo menos 1 lead com `is_fisbo = true`, pin exibe estrela vermelha (#DC1431) no canto superior direito, 12x12px. Query: JOIN `edificios` com `leads` WHERE `leads.is_fisbo = true AND leads.edificio_id = edificios.id`. Componente MapPin com prop `isFisbo: boolean` (seção 3.1 do UX spec).
- [x] AC3: Filtro rápido "Apenas FISBOs" no mapa. Toggle no painel de filtros (ícone filtro no header do mapa). Quando ativo: apenas pins com badge FISBO visíveis, demais ocultos com opacity 0.2. Badge numérico no ícone de filtro quando filtro ativo. Query com WHERE clause: `EXISTS (SELECT 1 FROM leads WHERE leads.edificio_id = edificios.id AND leads.is_fisbo = true)`.
- [x] AC4: Contagem de FISBOs por raio. Na legenda do mapa (barra recolhível, 36px colapsada), exibir "F:[N]" ao lado dos contadores de status. Query PostGIS: `SELECT COUNT(DISTINCT e.id) FROM edificios e JOIN leads l ON l.edificio_id = e.id WHERE l.is_fisbo = true AND l.consultant_id = auth.uid() AND ST_DWithin(e.coordenadas, epicentro, raio_metros)`. Contagem separada por raio (500m, 1km, 2km) exibida ao expandir a legenda.
- [x] AC5: Atalho para criar lead a partir de FISBO. Ao marcar "FISBO" em edifício que ainda NÃO tem lead, diálogo: "FISBO detectado! Deseja cadastrar um lead?" com botões "Cadastrar Lead" (abre form da Story 2.1 com `is_fisbo = true` pré-marcado e `origem = 'fisbo_scraping'`) e "Apenas marcar" (marca flag sem criar lead). Se já existe lead no edifício, toggle `is_fisbo` diretamente.
- [x] AC6: Lista de FISBOs acessível via busca global ou filtro no funil (Story 2.2). Na busca global, opção "Filtrar: FISBOs" retorna todos leads com `is_fisbo = true`. No funil, chip "FISBO" nos filtros mostra apenas leads FISBO em cada etapa. Contagem total de FISBOs no Dashboard (Story 2.9) seção territorial.
- [x] AC7: Marcação FISBO é reversível. Toggle off no botão FISBO do card do edifício: UPDATE `leads.is_fisbo = false`. Badge some do pin com animação fade-out (300ms). Confirmação não necessária para desmarcar (ação leve).

## Technical Notes

- **Database:** UPDATE em `leads` SET `is_fisbo = true/false` WHERE `edificio_id = $1 AND consultant_id = auth.uid()`. Se múltiplos leads no edifício, marca o lead mais recente ou todos (decisão: marcar todos). Query PostGIS para contagem por raio usa `ST_DWithin(coordenadas::geography, ST_MakePoint(lng, lat)::geography, raio)`.
- **API:** Supabase client SDK. TanStack Query mutation com optimistic update: pin do mapa atualiza badge imediatamente. Query key invalidation: `['edificios', 'mapa']`, `['leads', 'fisbo']`.
- **Components:** FisboToggle (botão no card do edifício), MapPin (atualizado com prop isFisbo), FisboFilter (no painel de filtros do mapa), FisboCounter (na legenda do mapa).
- **Index:** `idx_leads_fisbo` em `leads(is_fisbo) WHERE is_fisbo = true` para queries rápidas de filtro FISBO.

## Notes

- **FR-010:** Identificação e destaque de FISBOs como prioridade de prospecção.
- FISBO = "For Sale By Owner" — proprietário vendendo sem corretor. São a principal oportunidade de prospecção para a consultora RE/MAX.
- A marcação manual é MVP. A Story 3.4 (agente de varredura) automatizará a detecção via scraping de portais (ZAP, OLX, VivaReal).
- Campo `leads.is_fisbo` default false. Story 2.10 permite marcação manual; Story 3.4 marca automaticamente via scraping.
