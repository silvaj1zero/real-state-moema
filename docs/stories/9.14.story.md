## Story

Como consultora (Luciana Borba), quero declarar o **estado de conservação do imóvel-alvo** e ver o valor de mercado com **cenários de deságio 0 / −7,5% / −15% explícitos** (não um Capex Score B de 15% escondido), para que o laudo seja defensável em contraditório e eu possa calibrar a tese (conservador / provável / agressivo) na V2.

## Acceptance Criteria

1. **AC1 — Ficha do alvo (input tipado):** `AcmTarget` (ou tipo adjacente) aceita, opt-in:
   - `estadoConservacao?: 'A' | 'B' | 'C' | 'D' | null` (escala a validar na H-3)
   - `anoConstrucao?: number | null`
   - `reformasRelevantes?: string | null` (texto curto)
   - Sem ficha → comportamento legado documentado + **aviso** se `avisos[]` (9.15) existir
2. **AC2 — Deságio declarado separado do Score:** o desconto de até 15% **não** se chama mais apenas `CAPEX_BY_SCORE.B` no fluxo de headline. Novo mecanismo:
   - `cenariosDesagio: { conservador: 0.15, provavel: 0.075, agressivo: 0 }` (defaults; override por input)
   - `valorMercadoPorCenario` derivado da mediana × área × (1 − d)
   - Score AAA–B permanece classificação de **mercado** (R$/m²), não substituto de vistoria
3. **AC3 — Default sensato por estado (elicitável):** tabela default (ajustável pós H-3):
   | Estado | Cenário default |
   |--------|-----------------|
   | A (novo/reformado integral) | agressivo (0%) |
   | B (conservado / pronto morar) | provável (−7,5%) |
   | C (necessita ajustes) | conservador (−15%) |
   | D (obsolescência / reforma pesada) | conservador + flag “fora da régua simples” |
   | null | não aplica default silencioso: expõe **os três** cenários; não escolhe −15% sozinho
4. **AC4 — Laudo / computation:** `AcmLaudoComputation` expõe `desagioTratado` (cenários + default aplicado + motivo). Sec. 1 e capa do laudo mostram faixa **técnica** e nota de arbítrio. Residual e fatores de liquidez **não** misturam com este deságio.
5. **AC5 — Regressão:** fixture Honduras (sem ficha) permanece dentro da tolerância atual **ou** documenta delta explícito se o default legado Score B for reenquadrado como “conservador” — teste de migração obrigatório.
6. **AC6 — Caso 132:** com estado B (conservado), default −7,5% documentado; teste de fumaça com números da ordem do handoff 09-Jul (não byte-idêntico ao Capex cego).

## Dev Notes (implementação — @dev, Opus 4.8, 2026-07-09)

**Escopo entregue vs portão H-3.** O *mecanismo* foi implementado por inteiro; os *defaults da régua A–D* estão marcados como **provisórios até a H-3** (Art. IV — não inventar a régua sem a Luciana). A tabela `ESTADO_CENARIO_PROVISORIO` e o `origemDefault: 'ficha-provisoria-pre-H3'` carregam essa pendência de forma explícita, e a capa imprime "Régua A–D e defaults PROVISÓRIOS — pendentes de H-3". Todos os números usados (0 / −7,5% / −15% e o mapeamento A→agressivo/B→provável/C,D→conservador) são rastreáveis à AC2/AC3 desta story.

**Decisões de design:**
- **Deságio de estado é eixo SEPARADO do Score de mercado (AC2).** `CAPEX_BY_SCORE.B` permanece como classificação de mercado (R$/m²); o deságio de condição vira `desagioTratado` — camada explícita `valorMercado × (1 − d)` para os três cenários. Não mistura com residual nem com fatores de liquidez (AC4).
- **Sem ficha → sem default silencioso (AC3).** `cenarioAplicado = null`; a capa expõe os três cenários e reporta faixa conservadora; o aviso `target_condition_unconfirmed` (9.15) continua. `valorMercado`/headline **inalterados** — o deságio é aditivo e inerte sem ficha (AC5: Honduras dentro da tolerância, números-âncora intactos).
- **Três preços obrigatórios na capa (veredito §2):** técnico provável · comercial de captação · estratégico de anúncio — nunca colapsados em um só.
- **Ficha declarada confirma o estado** → `estadoAlvoConfirmado` derivado automaticamente, silenciando o aviso da 9.15.
