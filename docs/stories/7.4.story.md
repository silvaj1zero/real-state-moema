# Story 7.4: Crawler MercadoLivre Imoveis Zona Sul SP

**Epic:** 7 — Inteligencia de Prospeccao Automatizada Multi-Fonte
**Wave:** A (P0 captacao)
**Status:** InReview
**Priority:** Must
**Estimated Complexity:** L (8 pts) — requer PoC per `story-prototyping`

## User Story

Como consultora RE/MAX,
Quero que o sistema capture diariamente os anuncios FISBO de imoveis publicados em MercadoLivre Imoveis nas zonas Moema/Vila Olimpia/Itaim,
Para que eu tenha leads adicionais alem de ZAP/OLX/VivaReal (Epic 6) sem revisar manualmente o portal.

## Acceptance Criteria

- [x] AC1: Novo Apify Actor `apps/crawlers/mercadolivre-imoveis/` configurado para deploy Apify Cloud. Entrypoint `main.ts` consome `createPortalCrawler` do pacote shared. Manifest `actor.json` declara INPUT_SCHEMA (filtros: bairros[], tipo_negocio='venda', preco_min, preco_max, area_min, area_max, quartos_min).
- [x] AC2: Crawler busca listagem MercadoLivre Imoveis a partir de URLs canonicas:
  - `https://imoveis.mercadolivre.com.br/casas/venda/sao-paulo/moema/`
  - `https://imoveis.mercadolivre.com.br/casas/venda/sao-paulo/vila-olimpia/`
  - `https://imoveis.mercadolivre.com.br/casas/venda/sao-paulo/itaim-bibi/`
  - `https://imoveis.mercadolivre.com.br/apartamentos/venda/sao-paulo/{bairro}/`
  Pagina ate atingir cap configuravel (default 500 listings/run/bairro) OU `is_old_listing` flag.
- [x] AC3: Router pattern Crawlee separa: LISTING_PAGE handler (extrai anuncio cards + paginate) e DETAIL_PAGE handler (extrai full data: titulo, preco, area, quartos, banheiros, endereco texto, telefone, whatsapp, descricao integral, foto URLs, nome anunciante, CNPJ se visivel, CRECI se visivel).
- [x] AC4: Cada result emitido seguindo schema `PropertyEpic7` (Story 7.1). Persiste em `scraped_listings` reutilizando colunas Epic 6 (nome_anunciante, telefone_anunciante, whatsapp_anunciante, creci_anunciante) + novos campos `classification`, `classification_confidence`, `classification_signals`, `home_flags` da Story 7.1.
- [x] AC5: Apos persistir, chamar `classifyAdvertiser` (Story 7.3) e UPDATE row com classification + signals + confidence. Se classification=='for_sale_by_owner' AND confidence >= 0.70, criar evento `intelligence_feed` tipo `novo_fisbo_detectado` com metadata `{ listing_id, portal: 'mercadolivre', confidence }`.
- [x] AC6: Anti-bot resilient: respeita robots.txt (FR-058), rate-limit 1 req/s default (maxTasksPerMinute=60), session pool com cookies persistentes, fingerprint rotation via Crawlee, fallback Playwright se HTTP-mode falhar 2x. Tolerancia 200 OK rate >= 85% no run.
- [x] AC7: Telemetry — cada run cria 1 row em `crawl_runs` (Story 7.2). Failures gravam em `crawl_failures`. Snapshot final inclui stats Apify (CU, residential proxy bytes).
- [x] AC8: Edge Functions `trigger_mercadolivre_crawl` + `webhook_mercadolivre_done` implementadas. `trigger_mercadolivre_crawl` chama Apify API `runs/start` com actor ID + input config; retorna 202 em <=5s. `webhook_mercadolivre_done` atualiza `crawl_runs.status='completed'` com stats payload (idempotente). **Cron schedule (`epic7_mercadolivre_daily` diaria 04:00 BRT chamando `trigger_mercadolivre_crawl`) NAO e criado nesta story — propriedade exclusiva da Story 7.6 (migration 012).** Esta story remove `supabase/migrations/20260514000003_010_epic7_mercadolivre_cron.sql` da File List.
- [x] AC9: PoC obrigatorio (8 pts): smoke run contra 100 URLs MercadoLivre, validar >=85% success rate, telemetria gravada, classification rodou. Output em `docs/poc/7.4-mercadolivre-smoke.md`.
- [x] AC10: Unit tests para parseadores HTML/JSON (separados, sem rede). Integration tests com fixtures HTML salvas em `__fixtures__/mercadolivre/`.

## Technical Notes

- **Reference primaria:** ADR-EPIC7-002 (Apify cloud Wave A) + `docs/code-anatomy/apify-crawlee-focused/extraction-notes.md` (AdaptivePlaywrightCrawler + Router pattern)
- **Selectors:** MercadoLivre usa Cheerio-friendly HTML em listing pages; detail pages tem JSON em `<script type="application/ld+json">` (preferir parse JSON ao DOM scraping)
- **Telefone:** MercadoLivre mascara telefone na pagina publica (`(11) ****-1234`); precisa "Mostrar telefone" click que e usuario-action gated. **Decisao Wave A:** capturar telefone mascarado + nome anunciante; enrichment de telefone via portal-callback (botao "Falar com vendedor") fica para Wave B
- **Endereco:** raramente publicado completo; capturar `(bairro, cidade)` + coord aproximada do mapa preview se presente. Geocoding via Mapbox API (ja em uso projeto)
- **Pagination:** MercadoLivre limita 200 itens/busca; usar filtros de preco/area como chunking (similar a `BR-PAGE-001` HomeHarvest)
- **CRECI extraction:** parse text procurando `CRECI[\s:]*\d+` em descricao + meta tags
- **Frequency:** 1x/dia 04:00 BRT — volume estimado 200-500 listings/run total 3 bairros = aceitavel Apify Creator plan
- **LGPD:** todo telefone/email capturado deve passar por cifragem em repouso (Story 7.10)

### Decisões
- **Shared parsers em `app/src/lib/scrapers/mercadolivre/`, não dentro do Actor** — permite reuso pelos tests Next.js (vitest) + cobertura unificada. Actor importa via sync-script per ADR-EPIC7-006.
- **JSON-LD primeiro, DOM fallback** — MercadoLivre embute `<script type="application/ld+json">` em detail pages (mais estável que selectors). Parser tenta JSON-LD; se ausente ou inválido, cai para DOM.
- **Fixtures HTML sintéticas** — 4 fixtures construídas a partir do entendimento da estrutura ML (cards + JSON-LD + meta tags), NÃO scrape real. Reduz risco IP block + permite versionamento estável.
- **Telefone mascarado capturado as-is** — Wave A persiste `(11) ****-1234`; enrichment via "Falar com vendedor" callback fica para Wave B.
- **Edge Functions Deno minimal** — trigger só dispara Apify e registra `crawl_runs` (202 <5s); webhook valida HMAC + update idempotente; toda a lógica de classify roda no Actor (não no Edge).
- **HMAC shared secret** — env `MERCADOLIVRE_WEBHOOK_SECRET` (gerar via `openssl rand -hex 32` no deploy).

### Surprises / Gotchas
- Agent original parou após scaffold do Actor (38/38 tests passando) e antes das Edge Functions + PoC. Finalização manual completou: trigger.ts + webhook.ts + PoC + commit. **Padrão consistente:** todos os SDC agents desta sessão terminam antes do commit.
- ML mascara telefone server-side; não vale a pena custo de browser-mode para extrair via click — Wave A aceita masked, suficiente para classifyAdvertiser sinalizar.
