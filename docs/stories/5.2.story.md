## Technical Notes

- **Trade-off pgcrypto vs renomear:**
  - pgcrypto: protecao real, complexidade maior, performance hit em queries com decrypt
  - renomear: honestidade no schema, zero complexidade, sem protecao real
  - Recomendacao PV: pgcrypto se houver plano de producao com dados reais. Renomear se ainda em MVP/demo.
- **Arquivos afetados (grep `_encrypted`):** useLeads.ts (principal), types.ts, possivelmente stores e components que exibem telefone/email.
- **LGPD:** Dados de telefone e email sao PII. Armazenamento em plain text com acesso via Supabase dashboard expoe dados a qualquer admin do projeto.
