## Story

Como consultora, quero que o laudo **classifique automaticamente** se o preço pretendido/anúncio está **acima, alinhado ou abaixo** da referência ACM, com texto de tese pronto, para eu não usar sempre a narrativa de “deságio” quando o correto for “não cortar”.

## Acceptance Criteria

1. **AC1 — Função pura `classificarTeseComercial(ref, pret|anuncio) → { tese, deltaPct, label }`:**
   - `acima` se preço ≥ ref × 1,05  
   - `alinhado` se dentro ±5%  
   - `abaixo` se preço ≤ ref × 0,95  
   - limiares configuráveis; defaults documentados
2. **AC2 — Computation / model:** campo `teseComercial` usando `headline.referencia.valorMercado` vs `precoPretendido` / `precoPedidoReal` (prioridade: anúncio real se ambos).
3. **AC3 — Capa e Sec. 1:** badge + 1 frase:
   - acima → tese deságio / realismo de mercado  
   - alinhado → tese execução / diferenciação  
   - abaixo → tese subprecificação / **não cortar preço** (132)
4. **AC4 — Testes:** 3 casos sintéticos + 132-like (anúncio 1,495M vs ref ~1,77M → `abaixo`).
5. **AC5 — Sem preço comercial:** tese `indefinida` + sem badge (não inventa).
