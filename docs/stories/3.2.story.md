# Story 3.2: Dossiê/Showcase V2 (PDF)

**Epic:** 3 — Inteligência, ACM & Agentes de Automação
**Status:** Ready for Review
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero gerar um Dossiê PDF profissional que compile ACM, Plano de Marketing, histórico e branding para apresentar na V2,
Para que eu transmita autoridade e profissionalismo ao proprietário com um documento personalizado e compartilhável.

## Acceptance Criteria

- [ ] AC1: Botão "Gerar Dossiê" no card do lead (outlined azul #003DA5, 36px altura, ícone documento). Também acessível via checklist V2 (Story 2.6b, AC2 item 2 — sub-ação "Gerar Dossiê"). Botão visível apenas se lead está em etapa >= "V1 Realizada" no funil. Se Dossiê já existe para o lead, botão muda para "Ver Dossiê" (solid azul) com badge "v[N]" indicando versão.
- [ ] AC2: PDF gerado via React-PDF client-side (biblioteca `@react-pdf/renderer`). Geração inteiramente no browser — NENHUMA chamada server-side para renderização PDF. Componentes React-PDF: Document, Page, View, Text, Image, StyleSheet. Output: blob PDF renderizado client-side, exibido em preview antes de salvar. Tempo máximo de geração: <5s para dossiê com 6 páginas. Loading indicator: spinner + "Gerando Dossiê..." durante renderização.
- [ ] AC3: Seções do Dossiê compiladas automaticamente a partir de dados existentes. Estrutura do PDF (6 páginas MVP): (1) **Capa** — logo RE/MAX + branding Luciana Borba (nome, foto, CRECI, telefone) + endereço do imóvel + data, (2) **ACM** — snapshot dos dados de `dossies.acm_snapshot` JSONB: média preço/m², mediana, tendência, tabela top-5 comparáveis (se ACM Story 3.1 gerada), ou placeholder "ACM em preparação" se não existir, (3) **Plano de Marketing** — dados de `dossies.plano_marketing` JSONB: estratégia de divulgação, canais, timeline (template default se não personalizado), (4) **Histórico de Resultados** — dados de `dossies.historico_resultados` JSONB: resultados anteriores da consultora na região (vendas realizadas, tempo médio, satisfação), (5) **Estratégia de Parceria** (Could) — proposta de parceria ganha/ganha com benefícios para o proprietário, (6) **Contato** — dados de contato + QR code para agendamento.
- [ ] AC4: Branding personalizado da Luciana Borba no Dossiê. Cores primárias: azul RE/MAX #003DA5 e vermelho #CC0000. Header em cada página: logo RE/MAX esquerda + "Luciana Borba | Consultora RE/MAX Galeria" direita. Footer: CRECI + telefone + email. Fonte: Helvetica (built-in React-PDF). Foto da consultora na capa: carregada de Supabase Storage bucket `branding` path `{consultant_id}/foto-perfil.jpg`. Se foto não existir, placeholder com iniciais.
- [ ] AC5: Dossiê editável antes de salvar. Tela de preview exibe PDF renderizado com seções editáveis via bottom sheet. Tap em seção do preview abre editor: (a) ACM — readonly (gerado pela Story 3.1), (b) Plano de Marketing — textarea editável com template pré-preenchido, (c) Histórico — campos editáveis (nº vendas, tempo médio, depoimentos), (d) Estratégia de Parceria — textarea editável. Ao editar, PDF re-renderiza em tempo real (debounce 500ms). Botão "Salvar Dossiê" persiste versão final.
- [ ] AC6: Dossiê salvo no Supabase Storage e referenciado no banco. INSERT em `dossies` (id, lead_id, consultant_id, titulo, pdf_url, acm_snapshot, plano_marketing, historico_resultados, versao). PDF blob uploaded para Storage bucket `dossies` path `{consultant_id}/{lead_id}/dossie-v{versao}.pdf`. Campo `pdf_url` recebe URL pública do Storage. `versao` auto-incrementa: `SELECT COALESCE(MAX(versao), 0) + 1 FROM dossies WHERE lead_id = $1`. Index `idx_dossies_lead` em `(lead_id)`.
- [ ] AC7: Dossiê compartilhável. Botão "Compartilhar" (dropdown): (a) "WhatsApp" — deep link `https://wa.me/{telefone}?text={mensagem}` com link para PDF público no Storage, (b) "Email" — abre app de email nativo com PDF como link, (c) "Copiar Link" — copia URL pública do PDF para clipboard, (d) "Download" — força download do blob PDF local. Toast "Dossiê compartilhado!" com haptic feedback.
- [ ] AC8: Ao salvar/gerar Dossiê, atualiza automaticamente checklist V2 (Story 2.6b). UPDATE `checklists_preparacao` SET `dossie_montado = true` WHERE `lead_id = $1` AND `tipo = 'preparacao_v2'`. Se checklist não existe (V2 não agendada), não faz nada (graceful). TanStack Query invalidation de `['checklist', lead_id]`.

## Technical Notes

- **Database:** INSERT/UPDATE em `dossies` (id, lead_id, consultant_id, titulo, pdf_url, acm_snapshot JSONB, plano_marketing JSONB, historico_resultados JSONB, versao INTEGER, created_at, updated_at). Trigger `trg_dossies_updated_at`. Index `idx_dossies_lead` em `(lead_id)`.
- **React-PDF:** `@react-pdf/renderer` para geração client-side. Componentes: `<Document>`, `<Page size="A4">`, `<View>`, `<Text>`, `<Image>`, `<StyleSheet>`. Font registration: Helvetica (built-in). Renderização via `pdf(<MyDocument />).toBlob()`. Preview via `<PDFViewer>` ou custom preview com `URL.createObjectURL(blob)`.
- **Storage:** Supabase Storage bucket `dossies`. Policy: consultant dono pode ler/escrever. URLs públicas para compartilhamento (signed URLs com expiração 30 dias ou public bucket).
- **API:** Supabase client SDK. TanStack Query key: `['dossie', lead_id]`. Mutation para INSERT com upload para Storage.
- **Components:** DossieGenerator (tela principal), DossiePreview (preview PDF), DossieEditor (editor de seções), DossieCoverPage (React-PDF capa), DossieAcmPage (React-PDF ACM), DossieMarketingPage (React-PDF plano), DossieHistoryPage (React-PDF histórico), DossieContactPage (React-PDF contato), DossieShareMenu (dropdown compartilhar).
- **Offline:** Geração PDF funciona offline (React-PDF é client-side). Upload para Storage requer conexão — queued até reconectar.

## Notes

- **Princípio PV — React-PDF client-side:** Geração PDF é inteiramente no browser para capacidade offline. A consultora pode gerar Dossiê em campo sem conexão. Upload para Storage ocorre quando conectar.
- **Princípio PV — Graceful degradation:** Se ACM (Story 3.1) não foi gerada, seção ACM exibe placeholder. Dossiê funciona sem ACM — seção é opcional.
- Versioning permite histórico de dossiês por lead — cada geração cria nova versão sem deletar anteriores.
- A seção "Estratégia de Parceria" é Could-have — implementar como template estático no MVP, editável manualmente.
- Branding personalizado é diferencial competitivo — o proprietário vê profissionalismo e autoridade da consultora.
