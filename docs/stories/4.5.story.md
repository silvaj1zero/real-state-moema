# Story 4.5: Integração My RE/MAX

**Epic:** 4 — Parcerias Ganha/Ganha & Escala
**Status:** Draft
**Priority:** Should
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero sincronizar dados com a plataforma My RE/MAX (CRM oficial),
Para que eu não precise manter dados duplicados entre os dois sistemas e tenha consistência nas informações de leads e vendas.

## Acceptance Criteria

- [ ] AC1: Fase de investigação (BLOQUEANTE): antes de qualquer implementação, investigar a disponibilidade de API My RE/MAX. Checklist de investigação: (a) consultar documentação oficial My RE/MAX, (b) contatar suporte técnico RE/MAX Brasil, (c) verificar se existe API pública ou SDK, (d) documentar endpoints disponíveis (se houver), (e) registrar limitações (rate limits, scopes). Resultado salvo em `docs/architecture/integrations/myremax-api-investigation.md`. **Se API indisponível (provável), seguir caminho CSV/PDF.**
- [ ] AC2: Caminho API (se disponível — classificação **Should**): configuração de credenciais My RE/MAX em `consultant_settings` ou variável de ambiente segura. Tela "Integrações" → "My RE/MAX" → "Conectar" com OAuth flow ou API key input. Sync bidirecional: (a) leads do sistema → My RE/MAX, (b) leads do My RE/MAX → sistema. Mapeamento de campos configurável (similar a Story 4.4 column mapper). Sync manual via botão "Sincronizar agora" + sync automático (cron diário — Should).
- [ ] AC3: Caminho CSV/PDF (se API indisponível — classificação **Must**): tela "Integrações" → "My RE/MAX" → "Exportar para My RE/MAX". Export CSV com campos compatíveis com import do My RE/MAX (nome, telefone descriptografado temporariamente para export, email, endereço, etapa funil mapeada para campos My RE/MAX). Export PDF: relatório formatado com lista de leads ativos, pipeline status, últimas atividades. Botão "Exportar CSV" e "Exportar PDF" (ícones download).
- [ ] AC4: Upload manual de dados My RE/MAX: tela "Integrações" → "My RE/MAX" → "Importar do My RE/MAX". Upload CSV exportado do My RE/MAX. Mesmo fluxo de mapeamento de colunas da Story 4.4 (reutilizar componente ColumnMapper). Match com leads existentes por nome/telefone. Opção: atualizar existentes ou criar novos. Leads importados com `origem = 'myremax'`.
- [ ] AC5: Log de sincronizações: tabela `sync_logs` (a criar se necessário) ou campo JSON em `consultant_settings`. Cada operação de export/import registra: timestamp, direção (export/import), tipo (csv/pdf/api), registros processados, registros com erro, status (sucesso/parcial/falha). Tela "Integrações" → "My RE/MAX" → tab "Histórico" com lista cronológica de syncs. Máximo 50 registros mantidos (FIFO).
- [ ] AC6: Segurança no export: ao exportar CSV com dados descriptografados (telefone, email), modal de aviso: "Este arquivo contém dados pessoais. Trate conforme LGPD. Não compartilhe publicamente." Checkbox "Entendo e aceito" obrigatório antes de download. Arquivo CSV não armazenado no servidor — gerado client-side e baixado diretamente. Log do export registrado (AC5).
- [ ] AC7: Mapeamento de etapas do funil: tabela de correspondência entre etapas do sistema (contato, v1, v2, exclusividade, venda, perdido) e etapas do My RE/MAX (a investigar — possivelmente: Prospect, Qualified, Listing, Closed, Lost). Configurável na tela de integrações. Default: mapeamento 1:1 onde possível, "Outros" para etapas sem correspondência.

## Technical Notes

- **Investigação API:** Primeiro deliverable é o documento de investigação. Se API existe: documentar auth flow, endpoints, rate limits, data model. Se não existe: documentar evidência (emails, screenshots do suporte) e seguir caminho CSV.
- **Database:** Possível nova tabela `sync_logs` (id, consultant_id, integration TEXT, direction TEXT, type TEXT, records_processed INT, records_error INT, status TEXT, details JSONB, created_at). Ou armazenar em `consultant_settings` como JSONB array (simpler for v1).
- **Export CSV:** Papa Parse (unparse) para gerar CSV client-side. Dados descriptografados em memória apenas durante geração — não persistir. jsPDF para PDF.
- **Import:** Reutilizar componentes da Story 4.4 (CapteiUploader, ColumnMapper, ImportPreview) com configuração diferente.
- **API (se existir):** Supabase Edge Function como proxy para API My RE/MAX (evitar CORS, proteger credenciais). TanStack Query: `['sync_myremax', consultantId]`.
- **Components:** MyRemaxIntegration (tela principal), ExportPanel, ImportPanel, SyncHistory, FunnelMapper, APIConnectionForm (se API disponível).

## Dependencies

## Notes

- **Princípio PV — Investigação bloqueante:** A API My RE/MAX provavelmente NÃO existe ou não é acessível para consultores individuais. A investigação documenta isso formalmente e justifica o caminho CSV. Não assumir API sem evidência.
- **Princípio PV — CSV fallback:** Integrações SEMPRE têm fallback CSV. O caminho CSV/PDF é o Must, API é Should. Mesmo que API exista, CSV deve funcionar como alternativa.
- **LGPD (NFR-003):** Export de dados descriptografados é operação sensível. Aviso + log obrigatórios. Dados nunca persistidos descriptografados no servidor.
- Reutilização de componentes da Story 4.4 (Captei) reduz esforço significativamente. ColumnMapper, ImportPreview e ImportReport são genéricos.
- Esta é a última story na sequência do Epic 4 (4.5 é executada por último) — depende de multi-tenant estar ativo.
