**Epic:** 1 — Foundation, Mapa & Registro de Campo
**Status:** Done
**Priority:** Must

## User Story

Como consultora RE/MAX,
Quero que o sistema esteja configurado e deployado,
Para que eu possa acessar o app via URL no meu celular e começar a trabalhar em campo.

## Acceptance Criteria

- [x] AC1: Projeto Next.js 16+ inicializado com App Router, TypeScript strict, Tailwind CSS e shadcn/ui. Monorepo structure conforme definido na arquitetura. ESLint + Prettier configurados.
- [x] AC2: Supabase projeto criado com PostgreSQL 16+ e extensões `postgis` e `pgcrypto` habilitadas. Tabelas do núcleo territorial criadas: `consultores`, `epicentros`, `edificios`, `edificios_qualificacoes` com todos os campos do ERD (incluindo `geography(Point, 4326)` para coordenadas). Enums criados: `status_varredura`, `tipologia_edificio`, `padrao_edificio`, `abertura_corretores`, `origem_edificio`.
- [x] AC3: Supabase Auth configurado com autenticação por email/senha. Row Level Security (RLS) habilitada nas tabelas `edificios_qualificacoes` e `epicentros` com policy `consultant_id = auth.uid()`. Tabela `edificios` com RLS permissiva para leitura (futuramente pública).
- [x] AC4: PWA configurada com `manifest.json` (nome "RE/MAX Moema", tema azul #003DA5, ícone RE/MAX), Service Worker base via Workbox 7.x com `precacheAndRoute` para App Shell (~2 MB). Configuração de `next-pwa` ou equivalente no Next.js.
- [x] AC5: App instalável no celular — meta tags para iOS (`apple-mobile-web-app-capable`) e Android (`display: standalone` no manifest). Testado em Safari iOS e Chrome Android via "Adicionar à tela inicial".
- [x] AC6: Deploy na Vercel configurado com CI/CD via GitHub Actions. Pipeline: lint → typecheck → test → build → deploy. Branch `main` deploys para produção, PRs geram preview deployments.
- [x] AC7: Tela de login responsiva mobile-first conforme frontend-spec (seção 2.1). Campos email/senha com altura mínima 48px (touch target), botão "Entrar" full-width 56px cor #003DA5, link "Esqueci senha". Shake animation em erro, loading spinner no botão, transição fade para Mapa Principal após login.
- [x] AC8: State management configurado: Zustand para estado global + TanStack Query (React Query) 5.x para server state e cache. Provider configurado no layout root.
- [x] AC9: Indexes geoespaciais criados: `idx_edificios_coordinates` (GIST), `idx_epicentros_coordinates` (GIST). Indexes de lookup por consultor: `idx_edificios_qual_consultant`, `idx_edificios_qual_edificio`, `idx_edificios_qual_status`.

## Technical Notes

- **Database:** Tabelas `consultores` (id, email, nome, telefone_enc, franquia, regiao_foco, avatar_url, timestamps), `epicentros` (id, consultant_id FK, label, coordinates geography, raio_ativo_m, is_active, timestamps), `edificios` (id, nome, endereco, endereco_normalizado, coordinates geography Point 4326, bairro, cep, cidade, estado, origem enum, seed_source, verificado bool, created_by FK, timestamps), `edificios_qualificacoes` (id, edificio_id FK, consultant_id FK, tipologia enum, padrao enum, status_varredura enum, abertura_corretores enum, oportunidades_count, notas, is_fisbo_detected, timestamps, UNIQUE(edificio_id, consultant_id))
- **API:** Supabase Client SDK para operações CRUD diretas com RLS. Sem API Routes customizadas nesta story.
- **UI Components:** Tela de Login (seção 2.1 do frontend-spec) — Logo RE/MAX centrado 120x80px, campos 48px, botão 56px, shake animation, loading spinner
- **PostGIS:** `CREATE EXTENSION IF NOT EXISTS postgis;` + `CREATE EXTENSION IF NOT EXISTS pgcrypto;`
- **Offline:** Service Worker base (Workbox precache) — apenas App Shell cacheado nesta story. Cache completo na Story 1.6.
- **PWA:** manifest.json, meta tags iOS/Android, Workbox config, ícone RE/MAX 192x192 e 512x512

### Concerns

- **RLS deferred:** AC3 specifies RLS policies on `edificios_qualificacoes` and `epicentros`, but policies are deferred to Story 5.2. For single-user MVP this is acceptable, but must be tracked.
- **pgcrypto vs uuid-ossp:** AC2 specifies `pgcrypto` but migration uses `uuid-ossp`. Functionally equivalent for UUID generation. Not a blocker.
- **Table naming:** AC2 specifies `consultores` table but implementation uses `consultant_settings`. Schema is consistent with rest of codebase.
