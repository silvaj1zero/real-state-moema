# Story 3.3: Home Staging Automatizado

**Epic:** 3 — Inteligência, ACM & Agentes de Automação
**Status:** Done
**Priority:** Should
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero gerar e compartilhar um guia de Home Staging personalizado por tipologia do imóvel com branding profissional,
Para que o proprietário prepare o imóvel para visitas e venda, demonstrando meu cuidado e expertise antes da V2.

## Acceptance Criteria

- [x] AC1: Tela "Home Staging" acessível a partir do card do lead (botão "Home Staging" — outlined verde #22C55E, 36px altura, ícone casa com estrela). Também acessível via checklist V2 (Story 2.6b, AC2 item 3 — sub-ação "Gerar e Enviar WhatsApp"). Ao abrir, exibe preview do guia Home Staging para o lead atual.
- [x] AC2: Template base com "3 Regras de Ouro" do Home Staging. Conteúdo default (editável pela consultora): (1) **Despersonalize** — "Retire fotos pessoais, objetos religiosos e itens muito específicos. O comprador precisa se imaginar morando ali.", (2) **Ilumine** — "Abra todas as cortinas, acenda todas as luzes. Ambientes claros parecem maiores e mais convidativos.", (3) **Organize** — "Guarde itens do dia-a-dia (sapatos, roupas, louças). Cada ambiente deve mostrar seu melhor uso." Cada regra exibida em card com ícone (casa, lâmpada, caixa), título bold 18px, texto regular 14px, cor de fundo branco com borda esquerda colorida (azul, amarelo, verde).
- [x] AC3: Personalização por tipologia do imóvel. Selector no topo: "Apartamento" / "Casa" / "Comercial" / "Cobertura" (chips, single-select — default baseado em `edificios.tipologia` vinculado ao lead). Cada tipologia adiciona dicas específicas abaixo das 3 regras de ouro: (a) **Apartamento** — "Valorize a varanda: plantas, mesa pequena. Garagens: limpas e organizadas.", (b) **Casa** — "Jardim: gramado aparado, piscina limpa. Fachada: pintura em dia.", (c) **Comercial** — "Deixe o espaço neutro: sem logos ou decoração específica do negócio anterior.", (d) **Cobertura** — "Terraço é o diferencial: móveis de área externa, iluminação indireta." Dicas salvas em tabela de configuração ou hardcoded no MVP.
- [x] AC4: Branding da Luciana Borba no guia. Header: logo RE/MAX + "Luciana Borba | Consultora RE/MAX Galeria". Footer: "Preparado especialmente para [Nome do proprietário] — [Endereço do imóvel]". Dados herdados de `leads.nome` e `edificios.endereco`. Cores: azul RE/MAX #003DA5, vermelho #CC0000. Fonte: system font (San Francisco / Roboto). Preview exibe o guia como card renderizado.
- [x] AC5: Compartilhamento via WhatsApp. Botão "Enviar via WhatsApp" (verde WhatsApp #25D366, 48px altura, ícone WhatsApp). Ao clicar: (a) gera imagem do guia (html2canvas ou React-PDF single page), (b) abre deep link WhatsApp `https://wa.me/{telefone}?text={mensagem}`. Mensagem: "Olá [Nome], preparei um guia de Home Staging para o seu imóvel na [Endereço]. Seguindo essas 3 dicas simples, seu imóvel vai se destacar nas visitas! 🏠✨ [link para PDF ou texto inline]". Telefone: decriptado de `leads.telefone_encrypted` e formatado +55XXXXXXXXXXX. Se telefone não cadastrado, exibe alerta: "Cadastre o telefone do lead para compartilhar via WhatsApp."
- [x] AC6: Compartilhamento via PDF com branding. Botão "Gerar PDF" (outlined azul). Gera PDF single-page via React-PDF client-side com: logo, 3 regras de ouro, dicas por tipologia, dados do proprietário, dados da consultora. PDF salvo em Supabase Storage bucket `home-staging` path `{consultant_id}/{lead_id}/home-staging.pdf`. URL compartilhável.
- [x] AC7: Ao enviar Home Staging com sucesso, atualiza automaticamente checklist V2 (Story 2.6b). UPDATE `checklists_preparacao` SET `home_staging_enviado = true` WHERE `lead_id = $1` AND `tipo = 'preparacao_v2'`. Toast "Home Staging enviado!" com haptic feedback. Se checklist não existe, não faz nada (graceful).
- [x] AC8: Template editável pela consultora. Botão "Editar Template" (ícone lápis) abre editor: cada regra de ouro com título e texto editáveis, dicas por tipologia editáveis. Alterações salvas per-consultant (não globais). Botão "Restaurar Padrão" reverte ao template default. Salvamento em `localStorage` no MVP (Supabase em futuro).

## Technical Notes

- **Database:** Leitura de `leads` (nome, telefone_encrypted, edificio_id) e `edificios` (endereco, tipologia). Checklist update em `checklists_preparacao`. Storage bucket `home-staging` para PDFs gerados. Templates personalizados em `localStorage` no MVP.
- **React-PDF:** Reutiliza infraestrutura de React-PDF da Story 3.2 para geração de PDF single-page. Componentes: HomeStageDocument, HomeStagePage, HomeStageRule, HomeStageFooter.
- **WhatsApp:** Deep link `https://wa.me/${telefoneFormatado}?text=${encodeURIComponent(mensagem)}`. Telefone decriptado via `pgp_sym_decrypt`. Formato internacional: +55 + DDD + número.
- **API:** Supabase client SDK para leitura de lead/edifício. TanStack Query key: `['home-staging', lead_id]`.
- **Components:** HomeStageScreen (tela principal), HomeStagePreview (preview do guia), HomeStageRule (card de regra), HomeStageTypologyTips (dicas por tipologia), HomeStageShareWhatsApp (botão WhatsApp), HomeStageEditor (editor de template).
- **Offline:** Preview e geração PDF funcionam offline (React-PDF client-side). Compartilhamento WhatsApp requer conexão.

## Notes

- **Princípio PV — Automação sugere, humano decide:** Template default com 3 regras de ouro, mas consultora pode personalizar texto e dicas. Sistema facilita, não impõe.
- **Princípio PV — Graceful degradation:** Se telefone não cadastrado, guia funciona como PDF/preview — WhatsApp é canal preferencial, não único.
- Story 2.6b (checklist V2) já implementa compartilhamento Home Staging básico via WhatsApp (AC4). Esta story completa o feature com personalização por tipologia, PDF com branding e editor de template.
- Templates hardcoded no MVP. Migração para tabela de configuração em futuro (consultora gerencia múltiplos templates).
- Imagem do guia para WhatsApp pode usar html2canvas para screenshot do componente React como alternativa a React-PDF.
