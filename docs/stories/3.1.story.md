# Story 3.1: ACM Semi-Automatizada (VETO PV #3)

**Epic:** 3 — Inteligência, ACM & Agentes de Automação
**Status:** Ready
**Priority:** Must
**Estimated Complexity:** XL

## User Story

Como consultora RE/MAX,
Quero gerar uma Análise Comparativa de Mercado (ACM) com dados reais de comparáveis na micro-região,
Para que eu apresente ao proprietário uma avaliação profissional fundamentada em evidências, diferenciando preço de anúncio vs preço de venda real.

## Acceptance Criteria

- [ ] AC1: Tela "ACM" acessível a partir do card do lead (botão "Gerar ACM" — outlined azul #003DA5, 36px altura, ícone gráfico de barras). Também acessível via checklist V2 (Story 2.6b, AC2 item 1 — sub-ação "Gerar ACM"). Ao abrir, tela exibe o endereço de referência (herdado do edifício do lead) com mapa centrado no ponto + raio de 500m visualizado como círculo translúcido azul #003DA5 com opacidade 0.15.
- [ ] AC2: Busca de comparáveis no raio de 500m via `fn_comparaveis_no_raio(lat, lng, consultant_id, 500)`. Resultados exibidos em tabela responsiva com colunas: Endereço (TEXT, truncado 30 chars com tooltip), Área m² (NUMERIC(8,2)), Preço (NUMERIC(12,2) formatado R$ com separador de milhar), Preço/m² (NUMERIC(10,2) calculado), Tipo (chip: "Anúncio" cinza #9CA3AF / "Venda Real" verde #22C55E — mapeado de `acm_comparaveis.is_venda_real`), Fonte (chip: "Manual" azul / "Scraping" laranja / "Captei" roxo / "Cartório" verde — enum `fonte_comparavel`), Distância (metros, 1 decimal). Ordenação padrão por distância crescente. Coluna "Tipo" com ícone: anúncio = olho, venda real = handshake.
- [ ] AC3: Cálculos automáticos exibidos em cards de resumo acima da tabela. 4 cards lado a lado (grid 2x2 em mobile): (a) **Média Preço/m²** — `AVG(preco_m2)` formatado R$ X.XXX,XX/m² — fundo branco, borda esquerda azul 4px, (b) **Mediana Preço/m²** — `MEDIAN(preco_m2)` — mesmo estilo com borda verde, (c) **Tendência** — variação % entre comparáveis mais antigos vs mais recentes (baseado em `data_referencia`) — seta para cima verde ou seta para baixo vermelha com %, (d) **Total Comparáveis** — contagem com breakdown "N manual | N scraping". Cálculos separados para "Anúncio" e "Venda Real" quando ambos existem — toggle para alternar entre os dois conjuntos ou "Todos".
- [ ] AC4: Diferenciação explícita entre preço de anúncio e preço de venda real. Toggle visual no topo da ACM: "Anúncio" / "Venda Real" / "Todos" (chips selecionáveis). Quando filtrado por "Venda Real", cards de resumo mostram badge "Dados reais de venda" em verde. Quando filtrado por "Anúncio", badge "Preços de anúncio — expectativa do mercado" em amarelo. Tabela filtra conforme seleção. Tooltip explicativo: "Preço de anúncio reflete a expectativa do vendedor. Preço de venda real reflete o valor efetivamente transacionado. A diferença média em Moema é de 10-15%."
- [ ] AC5: Formulário de input manual para cadastro de comparáveis. Botão "+ Comparável" (FAB verde #22C55E, ícone plus) abre bottom sheet com campos: Endereço (input TEXT, autocomplete com edifícios cadastrados — query `edificios` ILIKE), Área m² (input NUMERIC, teclado numérico), Preço R$ (input NUMERIC com máscara monetária), É venda real? (toggle boolean — default false = preço anúncio), Data de referência (date picker, default hoje), Notas (textarea opcional). INSERT em `acm_comparaveis` com `fonte = 'manual'`, `consultant_id = auth.uid()`, `coordinates` herdado do edifício selecionado ou geocodificado. Campo `preco_m2` calculado automaticamente (preco / area_m2) antes do INSERT.
- [ ] AC6: Comparáveis de scraping (Story 3.4) integrados automaticamente. Quando `scraped_listings` com `is_active = true` existem no raio, botão "Importar [N] anúncios de portais" (outlined laranja) aparece. Ao clicar, INSERT em `acm_comparaveis` com `fonte = 'scraping'` e `scraped_listing_id` FK. Importação NÃO é automática — consultora decide quais importar (checkbox por listing). Se nenhum scraping disponível, seção não aparece — ACM funciona 100% com dados manuais.
- [ ] AC7: ACM exportável. Botão "Exportar" (dropdown): (a) "Copiar Resumo" — copia para clipboard texto formatado com média, mediana, tendência e nº de comparáveis, (b) "Incluir no Dossiê" — salva snapshot JSONB em `dossies.acm_snapshot` (Story 3.2), (c) "CSV" — download de arquivo CSV com todos os comparáveis. Toast "ACM exportada!" com haptic feedback.
- [ ] AC8: Recálculo por bairro/micro-região. Selector no topo da ACM: "Raio 500m" (default) / "Moema" / "Vila Olímpia" / "Itaim Bibi" / "Customizado [input metros]". Ao trocar, refaz query `fn_comparaveis_no_raio` com novo raio ou filtra por `acm_comparaveis.bairro`. Cálculos automáticos (AC3) recalculados instantaneamente. Loading skeleton durante query.
- [ ] AC9: **VETO PV #3 — ACM funciona em modo manual-only.** Se zero dados de scraping existirem no sistema, ACM funciona normalmente com comparáveis manuais. Tela nunca exibe mensagem de erro tipo "Sem dados — aguardando scraping". Em vez disso, se poucos comparáveis (<3), exibe sugestão: "Adicione mais comparáveis para uma análise mais precisa." com botão "+ Comparável". Scraping enriquece quando disponível, mas ACM NUNCA depende dele. Teste de aceitação: deletar TODOS os registros de `scraped_listings` e verificar que ACM funciona perfeitamente com dados manuais.

## Technical Notes

- **Database:** SELECT/INSERT em `acm_comparaveis` (id, consultant_id, edificio_referencia_id, endereco, coordinates, area_m2, preco, preco_m2, is_venda_real, fonte, scraped_listing_id, data_referencia, notas, created_at, updated_at). Enum `fonte_comparavel`: manual, scraping, captei, cartorio. Function `fn_comparaveis_no_raio(lat, lng, consultant_id, raio_metros)` retorna comparáveis com distância. Index GIST em `acm_comparaveis.coordinates`.
- **API:** Supabase client SDK. TanStack Query key: `['acm', lead_id, raio]`. Mutation para INSERT comparável manual com optimistic update na tabela. Invalidation de query ao importar de scraping.
- **Cálculos:** Média e contagem via SQL aggregate. Mediana calculada client-side (sorted array, middle value) ou via SQL `PERCENTILE_CONT(0.5)`. Tendência: comparáveis com `data_referencia` no último mês vs mês anterior — `(avg_recente - avg_antigo) / avg_antigo * 100`.
- **Components:** AcmScreen (tela principal), AcmSummaryCards (4 cards de resumo), AcmTable (tabela comparáveis), AcmFilterToggle (Anúncio/Venda Real/Todos), AddComparableSheet (bottom sheet formulário), AcmExportMenu (dropdown exportação), AcmRadiusSelector (selector raio/bairro).
- **Offline:** Comparáveis manuais cadastráveis offline via mutation queue (Story 1.6). ACM com dados locais funciona offline. Sync ao reconectar.

## Notes

- **VETO PV #3 — ACM manual-first:** ACM funciona com ZERO dados de scraping. Consultora cadastra comparáveis manualmente. Scraping (Story 3.4) enriquece quando disponível, mas NUNCA é dependência. Teste de aceitação: sistema sem scraping = ACM 100% funcional.
- **Princípio PV — Scraping é acelerador, não dependência:** A arquitetura de ACM trata scraping como fonte enriquecedora opcional. O fluxo primário é sempre manual-first.
- **Princípio PV — Graceful degradation:** Se scraping morre (API bloqueada, Apify falha), ACM continua operando com dados manuais sem qualquer degradação visível para a consultora.
- Diferenciação preço anúncio vs venda real é um diferencial competitivo da Luciana — dados de cartório (venda real) são muito mais valiosos que dados de portais (anúncio).
- Exportação CSV permite importação futura em planilhas para consultores que ainda usam workflow manual.
