# Story 6.7: UI "Quem e o dono?" (Dossie de Proprietario)

**Epic:** 6 — Busca Parametrica On-Demand com Enriquecimento de Contatos
**Status:** Ready
**Priority:** Should
**Estimated Complexity:** S

## User Story

Como consultora RE/MAX,
Quero clicar num botao "Quem e o dono?" dentro do card do edificio,
Para receber na mesma tela um dossie com o nome do proprietario, matricula e contatos ja enriquecidos.

## Acceptance Criteria

- [ ] AC1: Botao "Quem e o dono?" no componente `BuildingCard` visivel apenas se `edificio.tipo_residencial IS TRUE`. Icone de pessoa + tooltip "Consulta cartorio (R$0,28)". Estados: `idle`, `loading`, `success`, `error`, `forbidden` (rate limit) e `budget_exceeded`.
- [ ] AC2: Ao clicar, abre `OwnerLookupModal`: estado inicial mostra spinner + "Consultando cartorio..." e dispara `useOwnerLookup.mutate({ edificio_id })`. Se ja houver cache (`cache_hit=true`), modal nao chama API e mostra payload existente imediatamente com badge "Dados de cache (X dias)".
- [ ] AC3: Modal renderiza 3 secoes quando `status=success`: (a) **Proprietario** — nome, CPF/CNPJ mascarado, data da matricula, cartorio, data da ultima transacao; (b) **Contatos enriquecidos** — junta dados de `scraped_listings.nome_anunciante / telefone_anunciante / email_anunciante` do Epic 6 via `fn_enriched_contacts_by_edificio(p_edificio_id)` (criar nesta story) com deduplicacao por telefone; (c) **Proxima acao** — botao "Captar lead" que abre `CaptarLeadModal` (Story 6.5) pre-preenchido com nome do proprietario e melhor contato.
- [ ] AC4: Estados de erro:
  - `status=not_found` -> mensagem amigavel "Cartorio nao localizou matricula para este endereco" + link para GeoSampa.
  - `status=failed` -> "Erro ao consultar cartorio. Tente novamente." + botao retry.
  - `forbidden` -> "Limite de 30 consultas/hora atingido. Tente novamente apos HH:MM."
  - `budget_exceeded` -> "Orcamento mensal de consultas atingido (R$X de R$Y). Aguarde proximo mes ou ajuste em Configuracoes."
- [ ] AC5: LGPD UX: rodape fixo do modal com micro-copy "Dados obtidos de cartorios publicos via ARISP. Uso restrito a prospeccao profissional. [Esquecer dados]". Link dispara `POST /api/owners/:id/forget` com confirmacao; sucesso remove entrada da tabela e retira do cache.
- [ ] AC6: Historico de lookups: nova aba "Proprietarios" na tela `mais/` lista os ultimos 50 lookups do consultor com filtro por edificio e status. Reusa componentes de tabela existentes.
- [ ] AC7: Dashboard de consumo: no mesmo componente do AC6, header com cards "Consultas no mes", "Custo acumulado", "Cache hits / misses". Dados via RPC `fn_owner_lookup_stats(p_consultant_id)` (criada nesta story) agregando `owner_lookups`.
- [ ] AC8: Analytics leve: disparar `intelligence_feed` tipo `owner_lookup_aberto` quando modal abre (ajuda measuring adoption). Nao disparar em cache hits silenciosos se flag `?quiet=1` estiver presente.
- [ ] AC9: Acessibilidade: modal segue pattern dos modais existentes (focus trap, escape fecha, aria-label). Botao "Quem e o dono?" com `aria-label="Consultar proprietario via cartorio (custa R$0,28)"`.

## Dependencies

- Depends on: 6.6 (endpoint + tabela + hook), 6.4 (colunas de contato em scraped_listings), 6.5 (CaptarLeadModal)
- Blocks: nenhum (terminal)

## Out of Scope

- Edicao manual do nome/matricula pelo consultor — seria "override" que exige decisao do PO sobre trilha de auditoria.
- Exportacao PDF do dossie — avaliar se emerge demanda real.
- Integracao com CRM externo (RD, Hubspot).

## Technical Notes

- **Composicao de dados:** endpoint `/api/owners/lookup` (Story 6.6) ja retorna o proprietario. O modal complementa com `fn_enriched_contacts_by_edificio` chamado via `createClient().rpc(...)` no componente server-side se possivel, ou via hook dedicado `useEnrichedContactsByEdificio`.
- **Rate / budget UI:** response do endpoint inclui `rate_remaining, rate_reset_at, budget_used, budget_limit`; modal lê os headers/payload para mostrar estado sem nova request.
- **Cache freshness:** badge "Dados de cache (X dias)" calcula `(now() - created_at).days` baseado no payload retornado (Story 6.6 AC6).
- **Error taxonomy:** status mapping codificado em `app/src/lib/owner-lookup-errors.ts` para ser testavel.

## Risks

| Risco | Mitigacao |
|---|---|
| Consultora dispara por engano e gasta budget | AC1 tooltip mostra custo + confirmacao implicita via modal (segundo clique). Cache elimina double-charge. |
| Dados do proprietario vazarem na URL | Modal nao navega, state mantido em memoria; logs mascaram CPF. |
| UX confusa com 3 secoes muito densas | Cada secao colapsavel; default: Proprietario expandido, Contatos colapsado. |
