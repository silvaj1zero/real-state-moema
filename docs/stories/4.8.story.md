# Story 4.8: Plano de Marketing Ativo por Imóvel

**Epic:** 4 — Parcerias Ganha/Ganha & Escala
**Status:** InProgress
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero que ao fechar exclusividade de um imóvel seja criado automaticamente um checklist de ações de marketing,
Para que eu execute sistematicamente todas as ações necessárias para promover o imóvel e nada fique esquecido.

## Acceptance Criteria

- [x] AC1: Ao mover lead para `etapa_funil = 'exclusividade'` (Story 2.2), sistema auto-cria registro em `marketing_plans` com `lead_id` FK e `consultant_id = auth.uid()`. Todos os campos booleanos iniciam como `false`. Toast: "Plano de Marketing criado para [endereço]!" com haptic. Se plano já existe para o lead, não duplicar (UNIQUE constraint ou check antes de INSERT).
- [x] AC2: Tela "Plano de Marketing" acessível via card do lead (estado Exclusividade/Venda) → botão "Marketing" (ícone megafone, azul #003DA5). Checklist visual com itens agrupados em seções: **Portais** (ZAP Imóveis, OLX, VivaReal), **Redes Sociais** (Instagram, Facebook), **Produção** (Fotos profissionais, Tour virtual), **Presencial** (Placa física, Safari/Open House). Cada item é um card 64px com checkbox, label e área para evidência.
- [x] AC3: Cada item do checklist é checkável com registro de data: ao marcar checkbox, UPDATE no campo booleano correspondente (`publicar_zap = true`) e campo de data (`publicar_zap_data = now()`). Para itens com URL (ZAP, OLX, VivaReal, Tour virtual): campo de input para colar URL da publicação (`publicar_zap_url`, etc.). URL salva como evidência de que a ação foi executada. Campos mapeados 1:1 com colunas da tabela `marketing_plans`.
- [x] AC4: Barra de progresso visual no topo: "N de M ações concluídas" com percentual. Barra horizontal (altura 8px) com gradiente verde. Cards de contagem: Portais (N/3), Redes (N/2), Produção (N/2), Presencial (N/2). Cálculo: soma dos campos booleanos `true` / total de campos booleanos (9 itens fixos). Cor: verde se >= 80%, amarelo se 50-79%, vermelho se < 50%.
- [x] AC5: Lembretes automáticos: se plano tem itens pendentes após 3 dias da criação, push notification "Você tem N ações de marketing pendentes para [endereço]". Repetir a cada 3 dias enquanto houver pendências. Respeita `consultant_settings.notificacoes_push`. Implementação via Supabase Edge Function (cron) ou notificação local com scheduling.
- [x] AC6: Safari linkado: item "Safari/Open House" no checklist, ao marcar, verifica se existe `safari_event_id` vinculado ao `marketing_plans`. Se não existe, botão "Criar Safari" que redireciona para Story 4.7 com `lead_id` pré-preenchido. Se já existe, exibe link para o evento com status. Campo `safari_event_id` FK nullable em `marketing_plans`.
- [x] AC7: Template customizável (classificação **Should**): tela de configuração com lista de itens do checklist. Possibilidade de reordenar, ativar/desativar itens padrão. Salvo em `consultant_settings` (campo JSON `marketing_template` — a ser adicionado) ou em tabela auxiliar. Para v1, template fixo com os 9 itens da AC2 é suficiente.
- [x] AC8: Alimenta Dossiê (Story 3.2): dados do plano de marketing acessíveis via join `marketing_plans.lead_id = dossies.lead_id`. Dossiê exibe seção "Ações de Marketing" com checklist readonly (ícones verdes/cinza) e links para evidências (URLs dos portais, fotos). Não requer implementação adicional se Dossiê já faz o join — apenas garantir que dados estão corretos.

## Technical Notes

- **Database:** Tabela `marketing_plans` (migration 004). Colunas: id, lead_id (FK), consultant_id (FK), publicar_zap (bool), publicar_zap_data (date), publicar_zap_url (text), publicar_olx (bool + data + url), publicar_vivareal (bool + data), postar_instagram (bool + data), postar_facebook (bool + data), fotos_profissionais (bool + data), tour_virtual (bool + url), placa_fisica (bool), safari_planejado (bool), safari_event_id (FK nullable), notas, created_at, updated_at. Índice: `idx_marketing_plans_lead`.
- **Trigger aplicativo:** Auto-criação ao mudar `etapa_funil` para 'exclusividade' — implementar no handler de transição do funil (Story 2.2), não como DB trigger (flexibilidade).
- **API:** Supabase client SDK. TanStack Query: `['marketing_plan', leadId]`. Mutation para UPDATE individual de cada campo (otimistic update no checkbox).
- **Components:** MarketingPlanChecklist, MarketingProgressBar, ChecklistItem (checkbox + date + url), MarketingSection (agrupador), ReminderConfig.
- **Notificações:** Push via Supabase Edge Function com cron ou via Web Push API com service worker scheduling.

## Notes

- **Princípio PV — Sistematização:** O plano de marketing é a sistematização do processo RE/MAX. Cada imóvel exclusivo DEVE ter todas as ações executadas. O checklist garante que nada é esquecido.
- **Princípio PV — Evidência:** URLs e datas servem como evidência para o proprietário (Dossiê) de que a consultora está trabalhando ativamente o imóvel. Isso é fundamental para manter a exclusividade.
- Template fixo na v1 é aceitável — customização como Should para não atrasar entrega.
- Os 9 itens do checklist cobrem os canais mais relevantes para mercado imobiliário em São Paulo (ZAP, OLX, VivaReal, Instagram, Facebook, fotos, tour, placa, Safari).
