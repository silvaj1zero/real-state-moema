# Story 1.2: Mapa Interativo com Epicentro e Raios Concêntricos

**Epic:** 1 — Foundation, Mapa & Registro de Campo
**Status:** Done
**Priority:** Must
**Estimated Complexity:** L

## User Story

Como consultora RE/MAX,
Quero ver um mapa centrado na minha região com círculos de raio,
Para que eu possa visualizar meu território de atuação e me orientar em campo.

## Acceptance Criteria

- [x] AC1: Mapa Mapbox GL JS (via `react-map-gl` 3.x) renderizado em tela cheia como view principal do app, conforme seção 2.2 do frontend-spec. Mapbox token configurado via variável de ambiente `NEXT_PUBLIC_MAPBOX_TOKEN`.
- [x] AC2: Epicentro padrão configurado na Rua Alvorada, Moema (CEP 04550-000, coordenadas aproximadas: -23.5988, -46.6658). Epicentro salvo na tabela `epicentros` com `consultant_id`, `coordinates` (geography Point 4326), `raio_ativo_m` (default 500), `is_active = true`.
- [x] AC3: Três círculos concêntricos renderizados via `@turf/circle` como GeoJSON layers no Mapbox: 500m (borda verde #22C55E, fill 5% opacity), 1km (borda amarela #EAB308, fill 5% opacity), 2km (borda vermelha #DC1431, fill 5% opacity).

- [x] AC6: Mapa responsivo e performático em mobile. Gestos suportados: pinch zoom, pan, long press. Touch targets mínimos de 44x44px para controles. Performance <2s para render inicial (NFR-004).

- [x] AC8: Header bar do mapa (44px) conforme frontend-spec: indicador offline à esquerda (aparece apenas quando offline, badge laranja), nome do bairro + percentual de cobertura no centro, botão filtro (ícone funil) + botão layers (ícone camadas) à direita.

## Technical Notes

- **Database:** Tabela `epicentros` (consultant_id, coordinates geography Point 4326, raio_ativo_m, is_active). Query PostGIS Q1 para buscar edifícios dentro de raio: `ST_DWithin(e.coordinates, epicentro.coordinates, :raio_metros)`.
- **API:** Supabase client SDK para CRUD de epicentros. Mapbox GL JS token via env var. Mapbox Tiles API para rendering.
- **UI Components:** MapView (Mapbox GL fullscreen), RadiusCircles (@turf/circle GeoJSON layers), GPSPin (ponto azul pulsante), HeaderBar (44px com offline/cobertura/filtro/layers), BottomTabBar (56px com 5 tabs), LayersPanel (toggle raios), EpicenterMoveConfirm (dialog).
- **PostGIS:** Raios renderizados client-side via Turf.js, NÃO via PostGIS. PostGIS usado apenas para filtrar dados (ST_DWithin). Clustering configurado no source Mapbox com `cluster: true`, `clusterMaxZoom: 14`, `clusterRadius: 50`.
- **Offline:** Map tiles pre-cache será implementado na Story 1.6. Nesta story, o mapa requer conexão para carregar tiles.

## Notes

- **Performance crítica:** O mapa é a tela principal e precisa carregar em <2s (NFR-004). Usar lazy loading para componentes não-essenciais. Map tiles via CDN Edge (Vercel).
- O epicentro padrão (Rua Alvorada) será usado apenas na primeira configuração. Depois, a consultora pode mover via long press.
- Mapbox GL JS já possui suporte nativo a clustering via `supercluster` — usar ao invés de solução server-side (volume esperado: 500-2000 pins em Moema, gerenciável no client).
- Zoom levels para clustering: <13 clusters grandes, 13-14 clusters menores, >=15 pins individuais.
- A porcentagem de cobertura no header será calculada pela query PostGIS Q2 (edifícios qualificados / total no raio). Inicialmente mostrará 0% até haver edifícios cadastrados.
