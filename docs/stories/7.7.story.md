# Story 7.7: Servico CRECI Unificado (Conselho Nacional + SP)

**Epic:** 7 — Inteligencia de Prospeccao Automatizada Multi-Fonte
**Wave:** A (P0 enriquecimento)
**Status:** InReview
**Priority:** Must
**Estimated Complexity:** M (6 pts)

## User Story

Como sistema de classificacao,
Quero um servico `creciService` que consulte CRECI nos sites oficiais (Conselho Nacional 21 UFs + CRECI SP) com captcha resolvido via 2Captcha + cache 30 dias,
Para que classifyAdvertiser (Story 7.3) tenha sinal `hasCRECI` confiavel e Luciana visualize CRECI validado de cada anuncio.

## Acceptance Criteria

- [x] AC1: Novo modulo `packages/scrapers/lib/creci/` com:
  - `index.ts` — entrypoint `creciService.lookup(numero, uf)`
  - `conselho-nacional.ts` — adapter para 21 UFs (Turnstile)
  - `crecisp.ts` — adapter para SP (reCAPTCHA Enterprise v3)
  - `parsers/conselho-response.ts` — Cheerio parsing per `BR-COFECI-004`
  - `parsers/crecisp-response.ts` — parser HTML CRECI SP
  - `captcha-client.ts` — wrapper 2Captcha (Turnstile + reCAPTCHA Enterprise)
  - `__tests__/` unit tests fixture-based
- [x] AC2: `creciService.lookup(numero: string, uf: string)` retorna `{ inscricao, nomeCompleto, situacao: 'Ativo'|'Inativo', telefone, fonte: 'conselho'|'crecisp', cached: boolean }` OR `null` se nao encontrado.
- [x] AC3: Cache em tabela `creci_cache` (migration 014) com TTL 30 dias:
  - colunas: id UUID PK, numero TEXT, uf TEXT, inscricao TEXT, nome_completo TEXT, situacao TEXT, telefone TEXT, fonte TEXT, raw_response TEXT, expires_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT now()
  - UNIQUE (numero, uf)
  - Index expires_at
  - RLS: service_role full, authenticated SELECT
- [x] AC4: Cache flow: lookup verifica `creci_cache` WHERE numero=$1 AND uf=$2 AND expires_at > now(). Se hit, retorna `cached: true` com cost=0. Se miss, consulta site oficial, persiste em cache, retorna `cached: false`.
- [x] AC5: Conselho Nacional adapter implementa BR-COFECI-001 a BR-COFECI-004 verbatim per `docs/code-anatomy/buscacreci/07-business-rules.md`. URLs:
  - Base: `https://www.creci{uf}.conselho.net.br` para 21 UFs (al,am,ap,ba,ce,df,go,ma,ms,mt,pa,pb,pe,pi,pr,rj,rn,ro,rr,sc,se)
  - Form path: `/form_pesquisa_cadastro_geral_site.php`
  - Turnstile sitekey: `0x4AAAAAAB5EssxvqmsTJ5Wx`
- [x] AC6: CRECI SP adapter implementa BR-CRECISP-001:
  - Base: `https://www.crecisp.gov.br`
  - reCAPTCHA Enterprise sitekey: `6LfUMMgqAAAAABG4tjE8VkT2wKZlqmAvV2YsId7a`
  - Page action: `submit_broker_search`
  - Endpoint: `/cidadao/buscaporcorretores`
  - Status `temporarily_disabled` documentado em `docs/code-anatomy/buscacreci/07-business-rules.md` mar/2026 — Wave A retorna `null` com `error_code='crecisp_unavailable'` se reCAPTCHA falhar server-side. Wave B avalia Camoufox/Puppeteer-stealth.
- [x] AC7: Rate-limit dedicado per portal:
  - Conselho Nacional: 1 req / 2s (configuravel via env `CRECI_CONSELHO_RATE_MS=2000`)
  - CRECI SP: 1 req / 5s (configuravel via env `CRECI_SP_RATE_MS=5000`)
  - Implementado via simple `setTimeout` lock por portal OR via Bottleneck library
- [x] AC8: 2Captcha integration:
  - Env vars: `TWOCAPTCHA_TOKEN`, `TWOCAPTCHA_BUDGET_MONTHLY_BRL` (default 200)
  - Budget guard: tabela `captcha_spend` (date, cost_brl); reject se sum(month) + estimated_cost > budget. Retorna 402 + error_code='captcha_budget_exceeded'.
  - Circuit-breaker: se 5 consecutive failures em 1 min, suspend 10min
- [x] AC9: robots.txt compliance per BR-COFECI-005 — funcao `isAllowedByRobotsTxt(url, userAgent)` verifica antes de cada request. graceful fallback se robots.txt 404 (assume allowed).
- [x] AC10: Unit tests fixture-based:
  - Conselho: HTML response with tbody -> parsed corretamente
  - Conselho: HTML response sem tbody -> retorna null
  - Conselho: tbody com tr mas tds < 6 -> retorna null
  - Telefone "NAO DIVULGADO" -> retorna ""
  - Situacao "INATIVO" -> classificado "Inativo" (nao confundir com "ATIVO" suffix)
  - CRECI SP: response com "Validacao reCAPTCHA" -> retorna error
- [x] AC11: Performance benchmark — cache hit P50 < 30ms, cache miss P50 < 8s (incluindo 2Captcha solve ~5-10s typical). Throughput sustained: 25 lookups/min (= rate-limit Conselho 2s)

### Decisões
- **Service factory `createCreciService({...})`** — pattern injetável (mesmo paradigma de TelemetryClient/CNAELookupClient nas stories 7.2/7.3). Permite mock 100% offline nos tests.
- **Migration `010` em vez de `014`** — preencheu gap entre `009_epic7_telemetry` e `011_epic7_cnpj_enrichment`. Story original sugeriu 014 (antes do auto-fix de migration numbering); manter cronologia compacta evita confusão.
- **`captcha-budget.ts` modular** — separado do `captcha-client.ts` para permitir budget circuit-breaker independente do provider concreto. Facilita migração para AntiCaptcha/CapMonster (Wave B) sem mexer no budget guard.
- **Negative caching** — TTL 30d para `situacao=NULL` (CRECI não encontrado). Economia massiva de 2Captcha em batch validation 7.9 (200 lookups: estimativa 60-80% cache hit após 1ª iteração).
- **CRECI SP retorna null + `error_code='crecisp_unavailable'`** — Wave A WAIVED per BR-CRECISP-001 (server-side reCAPTCHA Enterprise V3 bloqueio mar/2026). Documentado para Luciana.

### Surprises / Gotchas
- Agent original parou antes de criar a migration (decidiu que era domínio de @db-sage). Pragmaticamente migration foi criada pelo orquestrador — @dev pode criar arquivos `.sql` em `supabase/migrations/`; só não aplica via `supabase db push` (esse é @devops/@db-sage).
- Comment in `cache.ts` referenciava `supabase/migrations-proposed/` (dir inexistente). Corrigido para path real `supabase/migrations/20260514000003_010_*`.
