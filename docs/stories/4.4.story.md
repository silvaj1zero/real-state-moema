# Story 4.4: Integração Captei

**Epic:** 4 — Parcerias Ganha/Ganha & Escala
**Status:** InProgress
**Priority:** Must
**Estimated Complexity:** M

## User Story

Como consultora RE/MAX,
Quero importar dados de prospecção da plataforma Captei via CSV para o meu sistema,
Para que leads digitais capturados pela Captei sejam incorporados ao funil com match geográfico aos edifícios do mapa.

## Acceptance Criteria

- [x] AC1: Tela "Importar Captei" acessível via menu lateral → "Integrações" → "Captei" (ícone upload, azul #003DA5). Upload de arquivo CSV via drag-and-drop ou botão "Selecionar arquivo" (aceita .csv e .xlsx). Limite 5MB. Validação: arquivo não vazio, extensão válida, encoding UTF-8 (com fallback para ISO-8859-1 detecção automática). Preview das primeiras 5 linhas após upload.
- [x] AC2: Mapeamento de colunas: após upload, tela com 2 colunas — "Coluna do CSV" (esquerda, readonly) e "Campo do sistema" (direita, dropdown). Campos mapeáveis: nome do proprietário → `leads.nome`, telefone → `leads.telefone_encrypted`, email → `leads.email_encrypted`, endereço/edifício → matching PostGIS, tipo imóvel → `leads.tipologia`, valor pretendido → informativo. Sistema tenta auto-detectar mapeamento por nome de coluna (heurística: "nome" → nome, "tel"/"telefone"/"phone" → telefone, etc.). Usuária pode ajustar manualmente.
- [x] AC3: Match com edifícios via PostGIS: coluna de endereço do CSV é geocodificada e comparada com `edificios.coordinates` usando `ST_DWithin(edificio.coordinates, geocoded_point, 100)` (raio 100m). Se match encontrado: lead vinculado ao edifício (`leads.edificio_id`). Se múltiplos matches: exibir lista para seleção manual. Se nenhum match: lead criado sem edifício (campo `edificio_id` null), marcado com ícone alerta "Sem edifício" para revisão posterior.
- [x] AC4: Preview antes de importar: tela com tabela mostrando cada linha do CSV mapeada para os campos do sistema. Colunas: nome, telefone (mascarado), endereço original, edifício match (nome ou "Sem match"), status (Novo/Duplicata/Erro). Botão "Importar N leads" (verde #28A745) e "Cancelar" (outline). Contagem: "N novos | M duplicatas | P erros".
- [x] AC5: Detecção de duplicatas: antes de importar, comparar cada linha com leads existentes por: (a) telefone (decrypt e compare), (b) nome + edifício (fuzzy match com `similarity()` do pg_trgm, threshold 0.7). Duplicatas marcadas em amarelo no preview com tooltip "Lead similar encontrado: [nome existente]". Opção por linha: "Importar mesmo assim" / "Pular" (checkbox). Default: pular duplicatas.
- [x] AC6: Leads importados com `origem = 'digital_captei'` (adicionar ao enum `origem_lead` se necessário) e `etapa_funil = 'contato'`. Telefone e email criptografados via `pgp_sym_encrypt` antes do INSERT em `leads`. Cada lead criado com `consultant_id = auth.uid()`. Batch INSERT com transação — se uma linha falhar, pular e continuar (log de erros).
- [x] AC7: Relatório pós-importação: modal com resumo — "Importados: N | Duplicatas puladas: M | Erros: P". Lista de erros expansível (linha do CSV + motivo). Botão "Ver leads importados" redireciona para lista de leads filtrada por `origem = 'digital_captei'` e data de hoje. Log salvo em `console.log` para debug.
- [x] AC8: API REST como **Should**: endpoint Supabase Edge Function `/api/captei/import` que aceita JSON com array de leads (formato Captei API). Auth via API key. Mesmo processamento da importação CSV (match, duplicatas, INSERT). Para v1, apenas CSV upload é obrigatório. API REST planejada para quando Captei disponibilizar webhook/API.

## Technical Notes

- **Database:** INSERT em `leads` (tabela existente, migration 002). Novos valores de enum `origem_lead` se necessário ('digital_captei'). Geocoding: pode usar API externa (Google Geocoding, Nominatim) ou geocoding client-side para converter endereço → coordenadas. PostGIS match: `SELECT id, nome FROM edificios WHERE ST_DWithin(coordinates, ST_SetSRID(ST_MakePoint(lng, lat), 4326), 0.001)` (aprox 100m).
- **CSV parsing:** Biblioteca Papa Parse (browser) para parsing CSV. Para .xlsx: SheetJS (xlsx) para converter para array. Detecção encoding: TextDecoder com tentativa UTF-8, fallback Latin-1.
- **API:** Supabase client SDK para batch INSERT. TanStack Query mutation com progress tracking. Query key invalidation: `['leads']` após importação.
- **Components:** CapteiUploader (drag-drop zone), ColumnMapper (mapeamento visual), ImportPreview (tabela com status), ImportReport (modal resumo), GeoMatchIndicator.
- **Fallback CSV:** Este é o modo primário (princípio PV — integrações com fallback CSV). API REST é enhancement.

## Notes

- **Princípio PV — CSV fallback:** Integrações SEMPRE têm fallback CSV. A Captei pode não ter API aberta, então CSV é o modo primário, não o fallback. API REST é o enhancement futuro.
- **Princípio PV — Match geográfico:** O match com edifícios é o diferencial — leads da Captei ganham contexto territorial ao serem vinculados ao mapa de edifícios. Sem match, o lead existe mas sem vínculo geográfico.
- Geocoding de endereços pode ter custo (Google API) ou ser impreciso (Nominatim). Para v1, match por nome de rua/edifício via `ILIKE` no `edificios.endereco` é alternativa mais simples ao PostGIS geocoding.
- Detecção de duplicatas por telefone requer decrypt de todos os telefones existentes — pode ser lento com muitos leads. Considerar hash index para otimização futura.
