# Story 5.6: ADR de Arquitetura + Housekeeping de Repositorio

**Epic:** 5 — Correção de Divida Tecnica (Audit PV)
**Status:** Done
**Priority:** Could
**Estimated Complexity:** S
**Audit Finding:** F4 (HIGH), F8 (LOW)

## User Story

Como equipe de desenvolvimento,
Queremos que decisoes arquiteturais estejam documentadas como ADRs e que o repositorio nao tenha arquivos de backup expostos,
Para que novos desenvolvedores entendam POR QUE certas decisoes foram tomadas e nao haja risco de commitar credenciais acidentalmente.

## Acceptance Criteria

- [x] AC1: ADR criado em `docs/architecture/adr-001-api-routes-vs-edge-functions.md`. Contexto, decisao, alternativas, consequencias documentados.
- [x] AC2: Template ADR criado em `docs/architecture/adr-template.md`. Formato: Title, Date, Status, Context, Decision, Alternatives, Consequences.
- [x] AC3: Patterns de backup adicionados ao `.gitignore`: `*.backup.*`, `*.backup`, `.env.backup*`. Arquivos de backup existentes nao estao tracked (confirmado — untracked no git status).
- [ ] AC4: Limpar arquivos de backup do working directory — aguardando confirmacao do usuario.

## Technical Notes

- **ADR format:** Lightweight ADR (Michael Nygard). Nao precisa ser formal — 1 pagina e suficiente.
- **Backup files atuais:** `.env.backup.1775886867736`, `.env.backup.1775888336179`, `AGENTS.md.backup.*` — todos untracked mas visiveis no git status.
- **Risco real:** `.env.backup` contem credenciais. Se alguem fizer `git add .` sem prestar atencao, vaza segredos.
