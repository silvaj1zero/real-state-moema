## User Story

Como engenheiro de operacao,
Quero a orquestracao Wave A toda em pg_cron + Supabase Edge Functions + status-tables auto-healing,
Para que o pipeline ETL determinista rode sem LangGraph/Prefect, mantenha observabilidade nativa e se recupere de stalls automaticamente.

### Decisions

1. **pgmq Path A adopted (per @data-engineer PoC).** Migration 013 enables `CREATE EXTENSION IF NOT EXISTS pgmq;` and creates queue `epic7_long_tasks` directly. Fallback `epic7_task_queue` table NOT created — would be additive in a future story if pgmq deprecation occurs.

2. **Cron idempotence via `DO $$ ... IF EXISTS unschedule ... END $$;` blocks.** `cron.unschedule` raises if the job is missing, so a guard against `cron.job` is required. Pattern is now established for any future Epic 7 schedule migrations (but per AC1, no other migration is allowed to call `cron.schedule()`).

3. **`classify-anuncio` boundary contract versioning via `source` field.** Wave A returns `source: 'deterministic_v1'` with null classification fields. Wave B will return `source: 'langgraph_subgraph_v1'`. Consumers MUST switch on `source` rather than null-check fields — this is the durability contract per ADR-EPIC7-003 + AppFolio Realm-X precedent.

4. **Pure-logic mirror in `app/src/lib/edge-contracts/`.** The Deno Edge Function at `supabase/functions/classify-anuncio/index.ts` cannot be imported by vitest (Deno URL imports). Created a runtime-agnostic TypeScript mirror at `app/src/lib/edge-contracts/classify-anuncio.ts` that the Edge Function explicitly references in its header comment. Drift between the two is a contract violation. The OpenAPI spec is the authoritative shape.

5. **`_shared/run-lifecycle.ts` created but NOT consumed yet.** Story 7.4 owns `trigger_mercadolivre_crawl` + `webhook_mercadolivre_done`; refactor to use this shared helper is left to a follow-up story to respect 7.4 file ownership.

6. **`net.http_post` for cron HTTP dispatch.** pg_cron jobs use Supabase's `pg_net` extension (assumed available — present in all standard Supabase projects) to call Edge Functions. Service role key + base URL injected via session settings (`current_setting('app.service_role_key', true)`), to be wired by @devops during deploy.

### Summary

Story 7.6 is structurally sound. The two cross-story risks the @po flagged in the prior validation cycle (cron single-ownership and `fn_mark_stale_runs` ownership) are resolved cleanly in code:

- **AC1 cron single-ownership — CONFIRMED.** Migration 012 is the sole Epic 7 cron writer. The only other `cron.schedule` user in the repo is a pre-existing Epic 3 geosampa migration with disjoint scope.
- **AC2 fn_mark_stale_runs single owner — CONFIRMED.** Defined exclusively in 013. Migration 009 contains only consumer comments — no function definition.
- **AC6 classify-anuncio boundary contract — APPROVED.** `source` enum is the version anchor; tests assert null fields + enum exactly; OpenAPI 3.1 spec is authoritative; pure-logic mirror at `app/src/lib/edge-contracts/classify-anuncio.ts` enables vitest validation without Deno imports. 12/12 contract tests PASS (total suite 432 PASS).

### Cron Pattern Findings

- Idempotent `DO $$ ... IF EXISTS unschedule ... END $$; SELECT cron.schedule(...)` pattern is correct (avoids `cron.unschedule` raise-on-missing).
- All 5 cron expressions are valid 5-field syntax: `0 7 * * *`, `0 5 20 * *`, `0 6 25 * *`, `0 4 1 1,4,7,10 *`, `*/5 * * * *`.
- Timezone documented as UTC with BRT conversions in comments — auditable.
- Functions referenced in cron bodies (`fn_mark_stale_runs`, `fn_retry_failed_runs`) resolve at job tick (lazy), not at migration apply — safe since 013 timestamp > 012 timestamp ensures both applied before first tick.
- `SECURITY DEFINER` + `SET search_path = public` correctly applied on both healing functions.

### Boundary Contract Approval

`classify-anuncio` Wave A boundary is exactly what Wave B will need: consumers branch on `source: 'deterministic_v1' | 'langgraph_subgraph_v1'` rather than null-checking fields. The Deno Edge Function + pure-logic mirror + OpenAPI 3.1 + 12 contract tests form a tight three-way contract — drift between any two is detectable. AppFolio Realm-X precedent honored.
