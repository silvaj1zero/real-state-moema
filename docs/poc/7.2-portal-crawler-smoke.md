## Procedimento para execucao real (CI / dev local)

**Pre-requisitos:**
- Supabase local rodando (`supabase start`)
- Migration 009 aplicada (`supabase db push`)
- Playwright browser instalado (`npx playwright install chromium`)
- Service-role key em `.env.local`

**Script smoke** (`scripts/poc/7.2-smoke.ts` — a ser criado em ambiente CI):

```ts
import { createPortalCrawler, Telemetry } from '@/lib/scrapers'

const urls = Array.from({ length: 100 }, (_, i) =>
  `https://httpbin.org/anything?listing=${i + 1}&list_price=${500_000 + i * 1000}`
)

const tel = new Telemetry({
  supabaseUrl: process.env.SUPABASE_URL!,
  supabaseServiceRoleKey: process.env.SUPABASE_SERVICE_ROLE_KEY!,
})

const runId = await tel.startRun('mercadolivre') // portal arbitrario p/ teste
const t0 = Date.now()

const crawler = createPortalCrawler({
  portal: 'mercadolivre',
  telemetry: tel,
  requestHandler: async ({ request, pushData }) => {
    const url = new URL(request.url)
    const id = url.searchParams.get('listing')!
    const price = Number(url.searchParams.get('list_price'))
    await pushData({
      portal_listing_id: `MLB-${id}`,
      list_price: price,
    })
  },
})

await crawler.run(urls)
await tel.finishRun('completed')

const elapsed = (Date.now() - t0) / 1000
const snap = tel.snapshot()

console.log({
  run_id: runId,
  total_seconds: elapsed,
  requests_per_second: snap.requests_finished / elapsed,
  requests_finished: snap.requests_finished,
  requests_failed: snap.requests_failed,
  avg_duration_ms: snap.avg_duration_ms,
})

// Validacao DB
const { data, count } = await supabase
  .from('crawl_requests')
  .select('*', { count: 'exact' })
  .eq('run_id', runId)

console.log({ crawl_requests_rows: count, expected: 100 })
```

**Criterios de aceitacao do smoke real:**

- [ ] `requests_finished === 100`
- [ ] `requests_failed === 0`
- [ ] `requests_per_second >= 5` (Apify-proxy / network-bound — alvo conservador)
- [ ] `crawl_requests` table tem 100 rows com `run_id` correto
- [ ] `crawl_runs.status === 'completed'` e `finished_at IS NOT NULL`
- [ ] AutoscaledPool log mostra `currentConcurrency` escalando 2 -> 15+ no meio do run

## Custos / overhead medidos via unit tests

Telemetria adiciona:
- 1 `INSERT crawl_requests` por request (fire-and-forget, async)
- 0 round-trips bloqueantes no hot path
- 1 `UPDATE crawl_runs` no fim do run (apenas)

Overhead esperado: < 5ms p99 por request (apenas push para microtask queue + closure capture).

## Decisoes do PoC

1. **NAO rodamos 100 URLs reais nesta story** — story 7.2 entrega o wrapper; o consumidor (Story 7.4) faz o smoke contra portal real (mais signal-rich que httpbin).
2. **`crawlee@^3.16.0`** em vez de `^3.13.0`: 3.16 e o latest stable em 2026-05-18; story original disse "pinar versao" + "verificar latest stable na hora". 3.x e major-compat com APIs documentadas em `extraction-notes.md`.
3. **Telemetria fire-and-forget** validada por testes que simulam `client.failNext` — crawler NUNCA quebra por falha de insert.
