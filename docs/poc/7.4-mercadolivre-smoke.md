# PoC 7.4 — MercadoLivre Imóveis Crawler Smoke

**Date:** 2026-05-19
**Story:** 7.4 — Crawler MercadoLivre Imóveis Zona Sul SP
**Status:** Doc-only runbook (real execution deferred to staging — risco IP block para dev local + Apify CU cost)

## Goal

Validar end-to-end o Actor `mercadolivre-imoveis` em smoke run controlado:
1. Trigger via Edge Function `trigger_mercadolivre_crawl` → Apify runs/start
2. Actor crawla ~100 URLs MercadoLivre Moema (cap configurável)
3. Parser extrai PropertyEpic7 + AdvertiserSignals
4. `classifyAdvertiser` aplica heurística → UPDATE row
5. FISBO confidence ≥ 0.70 → INSERT `intelligence_feed`
6. Webhook `webhook_mercadolivre_done` finaliza crawl_run

## Pré-requisitos

- Env vars no Supabase (config staging):
  - `APIFY_TOKEN` (paid plan ≥ Creator)
  - `APIFY_ACTOR_MERCADOLIVRE_ID` (após deploy do Actor)
  - `MERCADOLIVRE_WEBHOOK_SECRET` (HMAC compartilhado)
  - `EDGE_FUNCTION_BASE_URL` (`https://{ref}.supabase.co/functions/v1`)
- Supabase migrations aplicadas: 008 (schemas Story 7.1), 009 (telemetry Story 7.2), 010 (CRECI cache 7.7), 011 (CNPJ 7.5)
- Apify Actor publicado: rodar `npm run deploy` em `apps/crawlers/mercadolivre-imoveis/` (após `apify login`)
- Edge Functions deployadas: `supabase functions deploy trigger_mercadolivre_crawl webhook_mercadolivre_done`

## Procedure (executar em staging, ~20 min)

### Step 1 — Deploy Actor

```bash
cd apps/crawlers/mercadolivre-imoveis
./scripts/sync-shared.sh    # copia app/src/lib/scrapers/ -> src/shared/
npm install
npm run build               # tsc
apify login --token $APIFY_TOKEN
apify push                  # publica no Apify Cloud
```

Anotar o Actor ID retornado → configurar em `APIFY_ACTOR_MERCADOLIVRE_ID`.

### Step 2 — Deploy Edge Functions

```bash
cd ../../..  # repo root
supabase functions deploy trigger_mercadolivre_crawl
supabase functions deploy webhook_mercadolivre_done
supabase secrets set \
  APIFY_TOKEN=$APIFY_TOKEN \
  APIFY_ACTOR_MERCADOLIVRE_ID=<actor-id> \
  MERCADOLIVRE_WEBHOOK_SECRET=$(openssl rand -hex 32) \
  EDGE_FUNCTION_BASE_URL=https://<ref>.supabase.co/functions/v1
```

### Step 3 — Smoke run (curl)

```bash
# Trigger smoke com cap baixo (100 URLs, só Moema)
curl -X POST "https://<ref>.supabase.co/functions/v1/trigger_mercadolivre_crawl" \
  -H "Authorization: Bearer $SUPABASE_SERVICE_ROLE_KEY" \
  -H "content-type: application/json" \
  -d '{
    "bairros": ["moema"],
    "tipo_negocio": "venda",
    "max_listings_per_bairro": 100
  }'

# Expected: 202 { run_id, apify_run_id, status: "running" }
```

### Step 4 — Monitor

```bash
# Apify dashboard: monitorar logs do Actor em tempo real
open "https://console.apify.com/actors/<actor-id>/runs"

# OU via API: poll status
APIFY_RUN_ID=...
curl "https://api.apify.com/v2/actor-runs/$APIFY_RUN_ID?token=$APIFY_TOKEN" | jq '.data.status'

# Supabase: monitorar crawl_runs
psql $DATABASE_URL -c "SELECT id, status, requests_finished, requests_failed FROM crawl_runs WHERE portal='mercadolivre' ORDER BY started_at DESC LIMIT 1;"
```

### Step 5 — Validação pós-run

```sql
-- 1. crawl_runs concluído
SELECT status, requests_finished, requests_failed,
       (requests_finished::float / NULLIF(requests_finished + requests_failed, 0)) AS success_rate
FROM crawl_runs
WHERE portal='mercadolivre'
ORDER BY started_at DESC LIMIT 1;
-- Expect: status='completed', success_rate >= 0.85 (AC6)

-- 2. Listings persistidos
SELECT count(*), count(DISTINCT classification) AS classes
FROM scraped_listings
WHERE source_portal='mercadolivre' AND created_at > now() - interval '1 hour';
-- Expect: count ~80-100, classes >= 2

-- 3. FISBO detectados
SELECT count(*) FROM intelligence_feed
WHERE event_type='novo_fisbo_detectado' AND created_at > now() - interval '1 hour';
-- Expect: > 0 (Zona Sul historicamente tem ~5-15% FISBO em ML)

-- 4. Classification distribuição
SELECT classification, count(*), avg(classification_confidence)
FROM scraped_listings
WHERE source_portal='mercadolivre' AND created_at > now() - interval '1 hour'
GROUP BY classification ORDER BY count DESC;
```

## Security envvars required (post-QA fixes 2026-05-19)

Após fixes SEC-001 + SEC-002 (gate `2794411`), o deploy staging exige 2 secrets adicionais:

| Env var | Onde configurar | Como gerar | Função |
|---|---|---|---|
| `MERCADOLIVRE_WEBHOOK_SECRET` | Supabase Secrets + Apify Actor webhook config | `openssl rand -hex 32` | Apify Actor calcula HMAC-SHA256 do body do webhook usando este secret (`webhookKey`). Edge Function `webhook_mercadolivre_done` valida via `x-webhook-signature` header com constant-time compare. |
| `MERCADOLIVRE_TRIGGER_SECRET` | Supabase Secrets + GUC `app.mercadolivre_trigger_secret` (lido pelo cron job) | `openssl rand -hex 32` | Edge Function `trigger_mercadolivre_crawl` exige `x-trigger-secret` header. Cron job `epic7_mercadolivre_daily` (migration 017) envia o header via `cron.alter_job`. |

```bash
# Geração + configuração one-shot
WEBHOOK_SECRET=$(openssl rand -hex 32)
TRIGGER_SECRET=$(openssl rand -hex 32)

# Supabase Edge Functions
supabase secrets set \
  MERCADOLIVRE_WEBHOOK_SECRET=$WEBHOOK_SECRET \
  MERCADOLIVRE_TRIGGER_SECRET=$TRIGGER_SECRET

# Postgres GUC (lido pela migration 017 ao reconfigurar cron)
psql $DATABASE_URL -c "ALTER DATABASE postgres SET app.mercadolivre_trigger_secret = '$TRIGGER_SECRET';"
psql $DATABASE_URL -f supabase/migrations/20260519000001_017_epic7_trigger_secret_header.sql

# Apify Actor webhook config (Apify dashboard ou apify CLI)
# Configurar webhook 'ACTOR.RUN.SUCCEEDED' com URL + secret = $WEBHOOK_SECRET
```

**Sem essas vars o trigger retorna 500 `trigger_secret_unset` e webhook retorna 500 `webhook_secret_unset` — fail-closed por design.**
